In MoEngage, users are categorized as registered, and anonymous users. Each user is identified with a unique user ID generated and associated with the users in the system with their mobile number and email address. A user who logs in to your app or website would have parameters such as Mobile Number, Email Address, or Custom Identifier (the ID generated by the app). For each user, MoEngage also generates an identifier, the MoEngage ID. You can search for the user using any of the parameters mentioned above. For more information on these user types, refer to [Users in MoEngage](https://help.moengage.com/hc/en-us/articles/12088416349460-User-Merging).

We will achieve this use case by creating segments,

1. To find registered users in the system.
2. To find anonymous users in the system.

## Steps to find a user who logged in to the system

You can search using a mobile number, email address, or MoEngage ID from the search bar. In this example, we will use MoEngage ID.

1. On the left navigation menu in the MoEngage dashboard, click **Segments**, and then click **Search Users**.
2. In the search bar, search using Name, Email, Mobile Number, MoEngage ID, or ID.
3. Enter one of the parameters in the search box and press Enter. This search presents a list of users matching your input with the parameters.
4. Click the user you want to view, and the user profile of the respective user will be displayed.  
    <img alt="Search-ezgif.com-loop-count.gif"  src="/images/moengage_a65d9e.png"  />   
     
   <Info>
Search by MoEngage ID is not supported for a user base below 10k.
</Info>
5. The same functionality is also present on Create Segments page, click **Segments**, and then click **Create Segment**. For more information on creating segments, refer [here](https://help.moengage.com/hc/en-us/articles/206169646-Create-Segments).
6. With Admin and Managers access, you can also provide instructions for their team members on how to look up their users, as shown below:  
    <img alt="search.gif"  src="/images/moengage_62f4d4.png"  />


## Steps to find anonymous users

Anonymous users typically have only the MoEngage ID. You can, however, search for them using the advertising identifiers or web push tokens depending on the platform on which the user is available.

Perform the following steps to identify users on the respective platforms as mentioned below:

Android Users iOS Users Web Users

<Tip>
Ensure you have Google Advertising ID (GAID) enabled during integration for users who are not logged in.
</Tip>

### Search a user with GAID

1. On the left navigation menu in the MoEngage dashboard, click **Segments**, and then click **Create segment.**
2. Click the **Filter users by** option, and then click the **User property** tab.
3. Add the filter criteria with user attributes, *Google Advertising ID (Android)*
4. Select the GAID from the dropdown **Select Option.**
5. Click **Show count.** <img alt="GAID_USer.gif" src="/images/moengage_9ed167.png"  />
6. From the Query result, click on the user icon in **Sample users** to open the User Profile. **<img alt="GAID.png" src="/images/moengage_569d5b.png" />**

You can identify iOS users who are not logged in using IDFA (Identifiers For Advertisers). To use IDFA, your users should opt in for IDFA tracking in MoEngage.

<Info>
- IDFA can be tracked only if the AdSupport framework is added to your iOS project.
- If IDFA tracking is opted out, IDFA will not be tracked. For more information about IDFA tracking, refer to the [opt-out capability in MoEngage](https://developers.moengage.com/hc/en-us/articles/4404181384980-Compliance).
- Apps for getting an iOS device's IDFA are available on the Apple App Store. For more information, refer to the [Sample App](https://apps.apple.com/in/app/the-identifiers/id564618183).
</Info>

### Search a user with IDFA

1. On the left navigation menu in the MoEngage dashboard, click **Segments**, and then click **Create segment.**
2. Click the **Filter users by option**, and then click the **User attributes** tab.
3. Add the filter criteria with user attributes, *Advertising Identifiers(iOS and Windows)*
4. Select the IDFA number from the dropdown **Select Option.** Click **Show count.** <img alt="IDFA.gif" src="/images/moengage_f67efe.png"  />

### Web push token for laptops, desktops, and Tablets

You can find users who subscribed to web push notifications on the MoEngage dashboard using the web push notifications token. Perform the following steps to obtain the token for laptops, desktops, and tablets:

**Obtain a web push notifications token:**  
The user must have accepted the request to receive the web push notifications. If the notification request was denied previously, the user must do the following:

1. Open the website.
2. Click the Security Icon[] on the left side of the address bar.
3. Select site settings \> Notifications and change it from Block to Ask.
4. Reload the page and accept to receive the Web Push notification.

**View the user 's web push token:**

1. Install the MoEngage SDK extension for the Chrome browser using this link.
2. Open your website.
3. Click on the MoEngage extension.
4. Copy the push token present in the Web Push tab. (It may take a few seconds to get the push token information. If unavailable, reload the website and click the extension after some time.)

<img alt="Webpush.png" src="/images/moengage_4b3aa5.png" />

### Web push token for Mobile devices

1. Perform the following steps to know the push token for mobile devices:
2. Connect the phone to the laptop, and open the site in Chrome.
3. Open this link:*chrome://inspect/#devices* on the laptop. You’ll see your device listed with the sites opened on your phone.
4. Inspect the website being tested by right-clicking and selecting Inspect. <img alt="image__3___1_.png" src="/images/moengage_bd6a43.png"  />
5. Navigate to the applications tab from Inspect, select local storage \> MOE data, and fetch the push token.

### Searching user with Web Push Token

Perform the following steps to search for a user on the Segmentation page using the Web Push token:

1. On the left navigation menu in the MoEngage dashboard, click **Segments**, and then click **Create segment**.
2. Enter the following criteria: User Behavior, Has Executed Subscribed to WebPush in the last 3 days.
3. From the time attribute list, select the time duration during which the user must have subscribed to the web push.
4. Click **+ attribute** and filter for *MOE\_WEB\_PUSH\_TOKEN* (this is the event attribute).  
   Enter your web push token into the options field and click **Show count**.

To search for a user on the Segmentation page using the Web Push token, do the following:

1. Navigate to *Segment -\> Create Segment*on the MoEngage Dashboard.
2. Select the *Filter users by* option.
3. In the *User behavior* tab, select the following from the event drop-down list:  
   *Has Executed Subscribed to WebPush*.
4. From the time attribute drop-down list, select the time duration in which the user must have subscribed to the web push.
5. Click *Add attribute* and filter for **MOE\_WEB\_PUSH\_TOKEN** (this is the event attribute)
6. Copy and paste your web push token into the filter box
7. Click *Show count*.  
    <img alt="Webpush.gif" src="/images/moengage_90d442.png"  />
8. In the **Query result** section, click on the user icon in **sample users** to open the User Profile. <img alt="WPuser.png" src="/images/moengage_5e8abd.png"  />
