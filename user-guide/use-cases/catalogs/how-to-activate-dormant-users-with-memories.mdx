# Introduction

The efforts to acquire new customers should not be nullified by losing customers. The trend goes from customers coming on the website or app to becoming dormant and getting lost. Retaining dormant customers thus plays a crucial role.

The users should be engaged by multiple campaigns or offers that help them realize the value of the brand they had intended to be associated with. One of the most effective ways of capturing the attention of the customer is by showcasing the memories they have spent with the brand. These can be either the last video the user watched or the last purchase the user made.

In this article, we will create a Catalog, a Recommendation, and a Push campaign to target users based on their past experience, such as users who have made purchases in the last 30 days.

<Tip>
The time range can differ depending on your retention period. If you need a longer retention period of more than 30 days, contact your account manager.
</Tip>

# Create a Catalog

In this section, let us create a Catalog:

1. Go to **Content** \> **Catalogs** from the left navigation. The Catalogs page is displayed.
2. Click **+ Create Catalog**. The Create catalog pop-up window is displayed.
3. In the **Enter catalog name** field, enter a name. For example, "skuwise\_post\_purchase\_journey".
4. Click **Submit**. The newly-created catalog is listed on the Catalogs page.  
    <img alt="create catalogs.png" src="/images/moengage_d8f4cf.png" />
5. Click the catalog name link or the icon next to it. The Create feed pop-up window is displayed
6. Upload a catalog with fields such as ID, product image, price, product name, and description. For more information, refer to [Catalogs](https://help.moengage.com/hc/en-us/articles/17456587985428).

# Create a Recommendation

In this section, let us map the user action and set the recommendation criteria.

## Map the User Action

1. Go to **Content** \> **Recommendations** from the left navigation. The Recommendations page is displayed.
2. Click **Map user actions**.
3. Click the event drop-down list under a User Action bucket and choose the relevant action from the drop-down menu.  
    <img alt="cat 2.png" src="/images/moengage_d3ade6.png" />

## Set the Recommendation Criteria

1. Go to **Content** \> **Recommendations** from the left navigation. The Recommendations page is displayed.
2. Click **+ Create recommendation**. The Create recommendation page is displayed.
3. Under **Filter by user action and item attributes**, click the **User Actions** model.
4. Click **Next**. It takes you to the second step "Create recommendation". Enter the following details:  
   - **Recommendation name**: Enter a name for the recommendation. For example, "product purchased". This will be the name through which you will refer the output of this recommendation model.
   - **Recommendation description**: Enter a description for the recommendation. This description will help you understand what the model is aimed at.
   - **Catalog**: Select the "skuwise\_post\_purchase\_journey" catalog.  
      <img alt="cat3.png" src="/images/moengage_3c76ca.png" />
5. Click **Create** to save the changes and run the model.

# Create a Push Campaign

In this section, let us create a Push notification for our use case.

## Step 1: Target Users

1. Navigate to the sidebar on the left and click **Engage** \> **Campaigns** and click **+ Create campaign** or click **+ Create new** \> **Campaign**.
2. Under **Outbound**, select **Push** and then select the delivery type.  
    <img alt="5.png"  src="/images/moengage_b4cc38.png"  />   
   You are taken to the first step "Target users" of defining your campaign.
3. Enter the following details:
   1. **Team**: Select a team if your organization has teams enabled for your account. For example, "UseCases". For more information, refer to [Teams in MoEngage](https://help.moengage.com/hc/en-us/articles/360028586211).
   2. **Campaign name**: Enter a name for the campaign. For example, "Activate Dormant Users".
   3. **Campaign tags**: Select the required campaign tags.  
       <img alt="2.png"  src="/images/moengage_462fce.png"  />
4. In the **Target Audience** section, select the target audience as shown in the image below:  
    <img alt="Screenshot 2025-02-03 at 6.37.29 PM.png" src="/images/moengage_953391.png" />
5. In the **Target Platforms** section, select **Android**.
6. Click **Next** to move to the second step "Content" where you can define the content that will go into your Push campaign.

## Step 2: Content

1. Select the template that you like to go with. For our example, select **Basic notification**. <img alt="image-20231129-213115 (1).png" src="/images/moengage_cb6a06.png" />
2. Enter the required content for the campaign. For example:
   - Message title
   - Message
   - Message summary
3. Make use of the Catalog to display the memory images.  
    <img alt="cat 5.png" src="/images/moengage_5d1f80.png" />   
   <Info>
   Carousels work best when there are multiple memories in hand. To set them up, you can use the data to be fetched from the product Catalog and displayed to the user.
   </Info>
4. Click **Next** to move to the third step "Schedule and goals" where you can define the schedule and goal of your campaign.

## Step 3: Schedule and Goals

1. In the **Send campaign** section, define when you want to start and end your campaign.
2. In the **Conversion goals** section, add conversion goals that you like to track.
3. Define **Delivery controls** and **Advanced** settings based on your requirements. For more information, refer to [Create Push Campaigns](https://help.moengage.com/hc/en-us/articles/206541583).  
    <img alt="4.png" src="/images/moengage_72cd0b.png" />
4. Click **Publish**.

# Conclusion

In this use case, we created a Catalog, a Recommendation, and a Push campaign to target users using their past experience such as users who have made purchases before 30 days.
