title: "Audit Logs"
---

Audit logs are a central place to record a trail of all actions and changes, such as user logins and configurations, that occur within your MoEngage workspace. They help you understand what changes were made, when they were made, and who made them.

The following are some examples of audit logs:

- A user logging into the account
- A user editing the account
- A user creating a content block
- A user editing an alert

<Info>
  - Only admins can view and download the audit logs.
  - Audit logs will track the events after you enable the audit logs feature for your workspace.
  - The **Teams** is only visible when the feature is enabled for your account. (This specifies the team to which the user belongs when the particular action is performed.)
  - No user can modify or update audit logs.
</Info>

# View Audit Logs

To view audit logs, perform the following steps:

1. On the left navigation menu in the MoEngage UI, navigate to **Settings**\> **Account** \> **Audit logs**. The Audit logs page appears. <img src="/images/moengage_b58d43.png" />

On the Audit logs page, the data is displayed in a tabular format, with columns containing details related to each activity performed by the user or team.

| Columns  | Description                                                                                                                                                                                   |
| -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Time     | This specifies the timestamp by capturing the specific day, month, and year when a particular action is performed.                                                                            |
| Teams    | This specifies the team to which the user belongs when a particular action is performed.                                                                                                      |
| Entity   | This specifies the type of event when the particular action is performed.                                                                                                                     |
| Activity | This specifies the summary when the particular action is performed.                                                                                                                           |
| Action   | This specifies the type of task performed, such as CREATE, UPDATE, DELETE, EXPORT, INVITE, Campaign duplicated, Campaign resumed, Campaign archived, Campaign unarchived, or Campaign paused. |
| Actor    | This specifies the email ID of the user who performed the particular action.                                                                                                                  |
| IP       | This specifies the IP address of the system the user used to perform the particular action.                                                                                                   |

# Filter Audit Logs

MoEngage allows you to search and sort audit logs within the workspace based on your selected criteria. You can filter the audit logs in two ways:

### Search Logs

1. On the left navigation menu in the MoEngage UI, navigate to **Settings** \> **Account** \> **Audit logs**. The Audit logs page appears.
2. In the **Search logs by** box, type the keyword, such as the Team, Entity, Activity, Action, Actor, or IP.
3. Press **Enter**.

   ![](/images/moengage_67fc03.png)

### Filter Logs

1. Click the  <Icon icon="bars-filter" /> Filter icon.
2. On the Filter dialog box, enter the following details:

   | Field               | Description                                                                        |
   | ------------------- | ---------------------------------------------------------------------------------- |
   | **Team(s)**         | Select the team to which the user belongs when the particular action is performed. |
   | **Action**          | Select the type of task performed.                                                 |
   | **Actor**           | Select the email ID of the user who performed the particular action.               |
   | **Date time Range** | Select the date range in which a particular action is performed.                   |

   ![](/images/moengage_ada366.png)
3. Click **Apply**.

The audit logs are displayed based on your selected criteria.

# Download Audit Logs

MoEngage allows you to export and download the audit log reports in CSV files. To download them, perform the following steps:

1. On the left navigation menu in the MoEngage dashboard, navigate to **Settings** \> **Account** \> **Audit logs**. The Audit logs page appears. <img src="/images/moengage_b58d43.png" />
2. [Filter the audit logs](#h_01JNQD9P6SMHC9MDAVHYBMQHVP) if required.
3. Click **Export audit logs** in the upper-right corner. <img src="/images/moengage_4ccc83.png" />\
   The audit log report is successfully sent to your email ID.

<Info>
  - You can export and download the audit log reports in CSV files using the following file naming convention: WorkspaceName_ExportID_YYYY-MM-DD.
  - Audit logs are available for the last 90 days only.
</Info>
