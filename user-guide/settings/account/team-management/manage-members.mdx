This feature enables you to invite a member to a workspace in MoEngage and define each member's level of access to the workspace.

# View Members

To view members, perform the following steps:

1. On the left navigation menu in the MoEngage dashboard, click **Settings** \> **Account** \>**Team management**.
2. On the Team management page, you are on the **Members** tab by default.

   ![](/images/moengage_6b6145.png)

On the Team management page, the data is displayed in a tabular format, with columns containing details related to each member.

| Columns   | Description                                                                                                                                                                                                                                                                                                                                                                                            |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Member(s) | This specifies the name and email ID of the member.                                                                                                                                                                                                                                                                                                                                                    |
| Status    | This specifies the member's status, such as Joined, Pending, or Access revoked. <ul><li>The **Pending** status means the member has received an invitation but has not joined the workspace</li><li>The **Joined** status means the member has received an invitation and joined the workspace</li><li>The **Access revoked** status means the member's access is revoked from the workspace</li></ul> |
| Role      | This specifies the role of the member, such as Admin, Marketer, or Analyst.                                                                                                                                                                                                                                                                                                                            |
| Action    | This enables admins to perform actions such as granting access, revoking access, or editing the role of a member.                                                                                                                                                                                                                                                                                      |

# Invite Members

<Info>
  - Members with the **Create & Manage** permission level in the **Team Members** component can invite other members to the workspace.
  - You can add a maximum of 100 users to a single workspace.
  - If a member is already part of a workspace, their role cannot be changed.
  - You can assign roles in bulk to members without affecting the existing members.
</Info>

To invite members to a workspace, perform the following steps:

1. On the **Members** tab, click + **Invite Member** in the upper-right corner.

   ![](/images/moengage_f23481.png)

   The Invite members pop-up window appears.

   ![Screenshot 2025-04-20 at 7.23.24 AM.png](/images/moengage_f6b186.png)
2. In the **Email addresses** box, type the email address of the member you want to invite.

   <Info>
     You can add multiple email addresses, separate them with a comma, and press Enter.
   </Info>
3. Click **Next**. The Invite members pop-up window is updated.
4. Under **Assign Role and teams**, enter the following details:

   | Field     | Description                                                                      |
   | --------- | -------------------------------------------------------------------------------- |
   | Member(s) | Edit the member's name, which is populated based on the email address you added. |
   | Role      | Select the role for an individual member.                                        |
   | Edit role | Change the role of all members.                                                  |

   ![](/images/moengage_923e8f.png)
5. Click **Invite**.

# Filter Members

You can filter the members in two ways:

### Search Members

1. On the left navigation menu in the MoEngage dashboard, click **Settings** \> **Account** \>**Team management**. The **Team management** page appears.
2. In the **Search members(s)** box, type the name or email ID of the member. <img src="/images/moengage_aeebe3.png" />

### Filter Members

1. Click the **Filter members** icon.
2. On the Filter dialog box, enter the following details:

   | Field  | Description                                                 |
   | ------ | ----------------------------------------------------------- |
   | Role   | Select the role of the member.                              |
   | Status | Select the status of the member, such as Joined or Pending. |

   ![](/images/moengage_6d49bf.png)
3. Click **Apply**.

## Edit a Member’s Role

<Info>
  The **Edit role** option is displayed for members whose status is **Pending** or **Joined**.
</Info>

To edit a member’s role, perform the following steps:

1. On the **Members** tab, click the ellipsis icon adjacent to the member you want to edit under the **Action** column.

   ![](/images/moengage_d1872a.png)
2. Click **Edit role**. <img src="/images/moengage_2f879e.png" /> The **Edit role(s)** pop-up window appears.
3. In the **Select Role** list, select the desired role.
4. Click **Done**.

   ![](/images/moengage_e23666.png)

The member role is now assigned successfully.

<Info>
  After the role change, the member will be automatically logged out of the dashboard.
</Info>

## Revoke a Member’s Access

<Info>
  - The **Revoke access** option is displayed for members whose status is **Pending** or **Joined**.
  - When user access is revoked, any data created by that user, such as campaigns, reports, or segments, is not deleted from the system.
</Info>

To revoke a member’s access to a workspace, perform the following steps:

1. On the **Members** tab, click the ellipsis icon adjacent to the member you want to revoke access from, under the **Action** column. <img src="/images/moengage_d1872a.png" />
2. Click **Revoke access**. <img src="/images/moengage_d9c629.png" /> The **Revoke access** pop-up window appears, prompting for confirmation to revoke the member's access.
3. Click **Revoke**.

   ![](/images/moengage_f18a46.png)

The member's access has been successfully revoked.

## Grant Access to a Member

<Info>
  The **Grant access** is displayed for members whose status is **Access revoked**.
</Info>

To grant a member access to a workspace, perform the following steps:

1. On the left navigation menu in the MoEngage dashboard, click **Settings** \> **Account** \> **Team Management** \> **Members**.
2. On the **Members** tab, click the ellipsis icon adjacent to the member you want to grant access to, under the **Action** column.

   ![](/images/moengage_d1872a.png)
3. Click **Grant access**.

   ![](/images/moengage_ed7571.png)

The member's access has been successfully granted.

## Delete a Member

<Info>
  - The **Delete user** option is displayed for members whose status is **Pending**, **Joined**, or **Access Revoked**. Deleting a member permanently removes their profile from the workspace. Any data created by that user, such as campaigns, reports, or segments, remains in the system.
  - You can delete a member only if they have a role with permissions less than or equal to your role. For example, only a user with the Admin role can delete another user with the Admin role.
</Info>

To delete a member, perform the following steps:

1. On the **Members** tab, click the ellipsis icon adjacent to the member you want to delete, under the **Action** column.

   ![](/images/moengage_d1872a.png)
2. Click **Delete user**. <img src="/images/moengage_83b0e8.png" />\
   The **Delete User**pop-up window appears, prompting for confirmation to delete the member.
3. Click **Delete**.

   ![](/images/moengage_605998.png)

   You deleted the member successfully.

# Bulk Edit Members

This feature enables you to perform bulk edits for multiple users. The bulk edit actions available in the **Action** column vary based on the members' statuses.

### Members with the Same Status

When you select one or more members of the same status, you can perform the following bulk edits.

| Status         | Options        | Description                                                                          |
| -------------- | -------------- | ------------------------------------------------------------------------------------ |
| Pending        | Edit           | This enables you to edit the role of any member.                                     |
|                | Resend invite  | This enables you to invite the particular member to join the workspace again.        |
|                | Export Members | This enables you to export members in a CSV file format.                             |
|                | Revoke access  | This enables you to revoke the member's access and remove the user from the account. |
|                | Delete user    | This enables you to permanently remove a member from the workspace.                  |
| Joined         | Edit           | This enables you to edit the role of any member.                                     |
|                | Export Members | This enables you to export members in a CSV file format.                             |
|                | Revoke access  | This enables you to revoke the member's access and remove the user from the account. |
|                | Delete user    | This enables you to permanently remove a member from the workspace.                  |
| Access revoked | Grant access   | This enables you to grant access to the members.                                     |
|                | Export Members | This enables you to export members in a CSV file format.                             |
|                | Delete user    | This enables you to permanently remove a member from the workspace.                  |

### Members with Different Statuses

When you select one or more members of different statuses, you can perform the following bulk edits.

| Status                     | Options                                                                                 |
| -------------------------- | --------------------------------------------------------------------------------------- |
| Joined and Pending         | <ul><li>Edit</li><li>Export Members</li><li>Revoke access</li><li>Delete user</li></ul> |
| Joined and Access revoked  | <ul><li>Edit</li><li>Export Members</li><li>Delete user</li></ul>                       |
| Pending and Access revoked | <ul><li>Edit</li><li>Export Members</li><li>Resend invite</li><li>Delete user</li></ul> |
| All members on a page      | <ul><li>Edit</li><li>Export Members</li><li>Resend invite</li><li>Delete user</li></ul> |

You can perform the following bulk edits on team members in the **Action** column:

- [Edit Roles of Members](/user-guide/settings/account/team-management/manage-members#edit-roles-of-members)
- [Export Members](/user-guide/settings/account/team-management/manage-members#export-members)
- [Revoke Access](/user-guide/settings/account/team-management/manage-members#revoke-access)
- [Grant Access](/user-guide/settings/account/team-management/manage-members#grant-access)
- [Resend Invite](/user-guide/settings/account/team-management/manage-members#resend-invite)
- [Delete Members](/user-guide/settings/account/team-management/manage-members#delete-members)

# FAQ

**How to recover Admin access and assign it to a new user if the original Admin account is deactivated?**\
Suppose your organization’s sole admin account has been deactivated, and you no longer have access to it. In that case, you can still recover admin access and assign new admin privileges by following these steps:

- Contact your organization’s IT administrator or domain owner to temporarily reactivate and grant access to the email account of the workspace's admin.
- Once the account is reactivated, use the Forgot Password option on the MoEngage login page to reset the password for that admin user account.
- After resetting the password, log in to the account and assign admin access to an existing user within your workspace or invite a new user with Admin access. This new admin will then have access to the MoEngage dashboard for that project.
- _(Optional)_ Revoke the former admin's access from the workspace.

To prevent similar issues in the future, MoEngage recommends assigning admin roles to at least two users. This ensures that if one admin leaves or becomes unavailable, the other can still manage the workspace and perform necessary administrative tasks.
