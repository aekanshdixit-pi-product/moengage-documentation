title: "Authorization Configuration with OAuth 2.0"
---

MoEngage allows you to configure and centrally manage OAuth 2.0 authentication services that fetch and refresh access tokens. Once configured, these authentication settings can be easily selected from a dropdown menu when setting up various services, such as Custom Connectors or Content APIs.

As a marketer, while creating a custom connector campaign (in step 2 Content) or configuring a custom SMS connector, you can use OAuth 2.0 as authentication. For more information, refer to [Create a Connector Campaign](/user-guide/campaigns-and-channels/connectors/create/create-a-connector-campaign) and [Configure Custom Connector (Service Provider)](/user-guide/campaigns-and-channels/sms-mms-and-rcs/sms/getting-started-with-sms/configure-custom-connector-for-sms).

This article explains how to configure and manage OAuth 2.0 settings.

# Add an OAuth 2.0 Configuration

1. Navigate to the MoEngage Dashboard and select **Settings** \> **Advanced settings** \> **Authorization configuration**. The Authorization configuration page is displayed.
2. Click **\+ Add OAuth config**. The Setup Authorization page is displayed. <img src="/images/moengage_9eb1f2.png" alt="setup OAuth.png" />

Adding an OAuth 2.0 configuration involves the following steps:

1. [OAuth details](#h_01HR13VH6DE2FRY5RF2E1E81HB): Define the target audience for the campaign.
2. [Response settings](#h_01HR13W8NGX8KFR48S5BDKRRTZ): Define the content of the connector campaign to be sent or the custom SMS connector.

## Step 1: OAuth Details

On the OAuth details tab, enter the following details:

1. **OAuth name**: Enter a name for OAuth configuration. These settings would be available for selection during campaign creation.
2. **Grant type**: Currently Client Credential grant type is supported. If you are using any other grant types, for further assistance, please contact your MoEngage Customer Success Manager (CSM) or the Support team.
3. **Token type**: Select a token type. Supported values are:
   - Opaque: Select this option if the token issuer uses Opaque tokens. Here the response from the issuer will contain the expiry time alongside the opaque access token.
   - JWT: If you select this option, the access token and the expiry time is encoded in the response as JSON Web Token (JWT) in Step 2 Response settings. You must decode the response to get the expiry time.
4. Under **API details**,    provide the details required to fetch the access token. Please check the API documentation of the OAuth service you plan to implement.
5. **API URL**: Enter the API URL.
6. **Method**: Select the API method. The following are the supported methods and available fields for each method:
   | Method | Supported Fields |
   | --- | --- |
   | GET | - URL parameters - Headers |
   | POST | - URL parameters - Headers - Body type |
   | PUT | - URL parameters - Headers - Body type |
   | DELETE | - URL parameters - Headers - Body type |
7. **URL parameters**: Click **\+ KV pair** and enter the parameters for the API.
8. **Headers**: Click **\+ KV pair** and enter the headers for the API.
9. **Body type**: Configure the body for your requests using the following options:

     </Tab>
     <Tab title="Tab">
     </Tab>
     <Tab title="Tab">
     </Tab>
   </Tabs>
   JSON format. Click **\+ KV pair** to add more JSON content. <img src="/images/moengage_46e3db.png" alt="OAuth JSON.png" /> Create the body using the raw text format. <img src="/images/moengage_e7c385.png" alt="OAuth Raw.png" />
10. Click **Send test & continue** to test the configuration and move to the second step "Response settings".

## Step 2: Response Settings

On the Response settings tab, enter the following details:

1. OAuth delivery response Opaque JWT If you have selected "Opaque" from the **Token type** drop-down list in Step 1 OAuth details, you will see only one **OAuth delivery response** field. The OAuth delivery response has a key with an expiry time, which is not encoded. You can also see when was the response received. <img src="/images/moengage_7ec5d3.png" alt="Opaque.png" /> Select an expiry option from the **Expiry** drop-down list and select a value from the second drop-down list. The values on the **Expiry** drop-down list depend on the kind of response received. <img src="/images/moengage_69e003.png" alt="Expiry.png" /> If you have selected "JWT" from the Token type drop-down list in Step 1 OAuth details, you will see two **OAuth delivery response** fields. <img src="/images/moengage_3a65c2.png" alt="JWT.png" /> In the first **OAuth delivery response** field, the response has a key with an expiry time, but the expiry time is encoded. You can also see when was the response received. From the **Access token** drop-down list, select a key that has the expiry time and click **Decode token type**. After the token type is decoded, you will receive one more response in the second **OAuth delivery response** field. Select an expiry option from the **Expiry** drop-down list and select a value from the second drop-down list. The values on the **Expiry** drop-down list depend on the kind of response received. <img src="/images/moengage_69e003.png" alt="Expiry.png" />
2. **Preferred prefix**: Select the preferred prefix of the OAuth setting.\
   Marketers can create custom prefixes in addition to the default options. These prefixes can be used when creating a custom connector or configuring a custom SMS connector, and can be overridden at a campaign level.
3. **Preferred error codes**: In the **OAuth error code** field, enter the preferred error code of the Oauth setting for an authorization failure because of an expired token. This error code can be used to reattempt the connector campaign or custom SMS connector.
4. Click **Save**. Now this Oauth setting is active. You can now use this Oauth configuration while [creating a custom connector campaign](/user-guide/campaigns-and-channels/connectors/create/create-a-connector-campaign) or [configuring a custom SMS connector](/user-guide/campaigns-and-channels/sms-mms-and-rcs/sms/getting-started-with-sms/configure-custom-connector-for-sms).

# Manage OAuth Settings

1. Navigate to the MoEngage Dashboard and select **Settings** \> **Advanced settings** \> **Authorization configuration**. The Authorization configuration page is displayed. You can see the existing OAuth settings configured. <img src="/images/moengage_a27134.png" alt="OAuth settings.png" />
2. To see where a specific OAuth setting is used, click the View list for the required setting. A list of associated Content APIs, PII configurations, and Connector Campaigns is displayed. <img src="/images/moengage_0fa906.png" alt="viewlist.gif" />
   <Info>
     - A "Failed" OAuth setting means your OAuth configuration is unable to fetch a response. You must edit your OAuth configuration and send test again. Do not use failed OAuth settings while creating connector campaigns or configuring custom SMS connectors.
     - Only configurations with an "Active" status will be available for selection in dropdowns when setting up connectors or campaigns.
     - If an active OAuth configuration expires, its status will change to "Failed." When you edit a campaign or connector that uses this expired configuration, an error message will be displayed, and you will need to select a new, active OAuth configuration.
   </Info>
3. To edit an OAuth setting, click the ellipsis icon for the required OAuth setting and click **Edit**.
4. To delete an OAuth setting, click the ellipsis icon for the required OAuth setting and click **Delete**. If there are any campaigns associated with the OAuth setting, you will not be able to delete it. A warning is displayed with the list of associated campaigns. <img src="/images/moengage_a7c2c0.png" alt="cannot delete.png" />

# Support Matrix

You can use the OAuth 2.0 configurations you set up here to authenticate requests in the following features:

- **Connector Campaigns**: Secure your connector campaigns. For more information, refer to [Create a Connector Campaign.](/user-guide/campaigns-and-channels/connectors/create/create-a-connector-campaign)
- Custom Connectors: Secure your custom SMS campaigns.. For more information, refer to [Configure Custom Connectors for SMS](/user-guide/campaigns-and-channels/sms-mms-and-rcs/sms/getting-started-with-sms/configure-custom-connector-for-sms).
- **Content APIs**: Authenticate API calls used for real-time personalization. For more information, refer to [Add Content APIs](/user-guide/settings/advanced-settings/add-a-content-api).
- **PII Tokenized Sending**: Secure the API used to fetch customer PII during campaign dispatch. For more information, refer to [PII Tokenized Sending](/user-guide/data/privacy/personal-data-or-personally-identifiable-information-pii/pii-tokenized-sending).
