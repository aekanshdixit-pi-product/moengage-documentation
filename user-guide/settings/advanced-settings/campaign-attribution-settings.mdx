# Configure Attribution Settings

To configure the attribution settings:

1. Navigate to **Settings** \-> **Advanced settings** \-> **Conversion and attribution**.
2. The following fields are available:
   | Field Name | Description |
   | --- | --- |
   | **Default Attribution Model** | Choose the attribution model for your campaigns and flows by selecting one of the following: <ul><li>**All Interactions**</li><li>**Last Interaction**</li></ul> For more information about how each of these models works, refer to [All Interactions model](/user-guide/campaigns-and-channels/campaign-management-and-reports/conversion-and-attribution#all-interactions-model) and [Last Interaction model](/user-guide/campaigns-and-channels/campaign-management-and-reports/conversion-and-attribution#how-the-all-interactions-model-works). |
   | **Conversion view** | Each attribution model has certain attribution types associated with it. Based on the attribution model chosen, you get the following options in this dropdown list: <ul><li>The All Interactions model has the following options: **View Through Attribution**\*,\* **Click Through Attribution**\*,\* and **In Session Attribution**.</li><li>The Last Interaction model has the following options: **Click Through Conversion** and **Total Conversions**.</li></ul> |
3. Click **Save**. You are prompted for confirmation about changing the attribution model or type. Click **Done**. You get a success message, as shown in the following image. <img src="/images/moengage_cbdc3a.png" alt="SuccessMessage (1).png" />

## Switching Between Attribution Models

You can switch between the All Interactions and Last Interaction attribution models directly in **Settings** \> **Advanced Settings** \> **Conversion and Attribution**.

<Frame>
  ![ALL Interaction Switch After 12 Days](/images/ALL-Interaction-switch-after-12-days.png)
</Frame>

### Switching for the First Time

Say you are on the All Interactions model and switch to the Last Interaction model. The two models attribute conversions differently: All Interactions credits every campaign whose attribution window overlaps with a conversion, while Last Interaction credits only the campaign with the deepest and most recent engagement. Because of this, some campaigns may show a dip in conversion metrics after the switch. Switching in the other direction — from Last Interaction to All Interactions — can cause an apparent inflation in conversion numbers. For more information on how each model works, refer to [All Interactions Model](/user-guide/campaigns-and-channels/campaign-management-and-reports/conversion-and-attribution#all-interactions-model) and [Last Interaction Model](/user-guide/campaigns-and-channels/campaign-management-and-reports/conversion-and-attribution#last-interaction-model).

The first time you switch models, a 12-day comparison window starts automatically. During this window:

- MoEngage stores conversion data for both models simultaneously for the duration of the window.
- You can review and compare data across both models at any point during the 12 days.
- You can switch back to your original model within the window. MoEngage continues storing data for both models throughout the 12-day period.

Once you select a model after the comparison window, MoEngage stores data only for the selected model from that day onward.

<Frame>
  ![First Switch From LAM To ALL](/images/First-switch-from-LAM-to-ALL.png)
</Frame>

### Switching After the Comparison Window

After the 12-day window ends, you can still switch models at any time. However, MoEngage only retains data for the actively selected model. If you switch back to a previously used model, conversion data from the period when that model was inactive will not be available.

For example, if you switch from Last Interaction to All Interactions on day 13 and then switch back to Last Interaction on day 17, conversions from days 13–16 will not be recorded under the Last Interaction model.

<Warning>
Switching models repeatedly after the comparison window ends creates gaps in your conversion data. Once you have decided on a model, MoEngage recommends staying with it.
</Warning>

<Frame>
  ![LAM Switch After 12 Days](/images/LAM-switch-after-12-days.png)
</Frame>

### Impact on Reports and Analytics

When you switch attribution models, conversion data in your campaign and flow analytics changes immediately:

- Conversion data before the switch date shows as zero for the newly selected model. MoEngage starts collecting data for that model from the switch date onward.
- Banners appear on Campaign Analytics pages, Flow pages, and all campaign pages to inform you of the switch date and explain why historical conversion data shows zero.
  <Frame>
    ![Campaign Info Page Banner For First Time ALL Interaction Change From LAM](/images/Campaign-info-page-banner-for-first-time-ALL-interaction-change-from-LAM.png)
  </Frame>
  <Frame>
    ![All Flows Page Banner In Case Of LAM](/images/All-flows-page-banner-in-case-of-LAM.png)
  </Frame>
  <Frame>
    ![ALL Campaign ALL Flows Campaign Info Banner After 12 Days](/images/ALL-campaign_ALL-flows_Camopaign-info-banner-after-12-days.png)

- Existing scheduled and periodic reports that include conversion columns for the previously active model will show zero for any period after the switch date. Update those reports to include the conversion columns for the newly selected model.

The attribution types available change depending on which model is active:

| Model | Attribution Types Available |
| --- | --- |
| All Interactions | View Through Attribution, Click Through Attribution, In Session Attribution |
| Last Interaction | Click Through Conversion, Total Conversions |

## Switching Between Conversion Views

When you switch between conversion views, the metrics are shown for the attribution type selected in the **Default Attribution Type** in the **Campaign Analytics and Info** page changes based on the attribution type chosen.

<Frame>
  ![Switch Conversion View Revamped UI](/images/SwitchConversionView_RevampedUI.gif)
</Frame>

For example, when the attribution model is All Interactions, and the default attribution type is View through attribution, the Campaign Analytics page shows calculated based on view through attribution. When you change the default to click-through, the statistics shown are for click-through attribution.

You can, however, change the attribution type in the **Attribution Type** filter in the Analytics section to see the metrics for other attribution types, as shown in the image below.

# Frequently Asked Questions

<AccordionGroup>
  <Accordion title="What if I switch models by mistake? Can I get my data back?">
    It depends on when you catch the mistake. If you are still within the 12-day comparison window, switch back to your original model — MoEngage has been storing data for both models throughout the window, so no data is lost. If the comparison window has already ended, data from the period when the model was inactive cannot be recovered. Switching back will create a gap for that period.
  </Accordion>

  <Accordion title="Does the attribution model apply to all campaigns and flows, or can I set it per campaign?">
    The attribution model is an account-level setting. All campaigns and flows in your account use the model selected in **Settings** > **Advanced Settings** > **Conversion and Attribution**. You cannot set a different attribution model for individual campaigns or flows.
  </Accordion>

  <Accordion title="Does switching models affect campaigns that are already running?">
    Yes. The change takes effect immediately across your entire account. Active campaigns will show zero conversions under the newly selected model for any period before the switch date. Conversions recorded after the switch date will be attributed under the new model going forward.
  </Accordion>

  <Accordion title="Does the 12-day comparison window restart if I switch models again after it ends?">
    No. The 12-day comparison window only starts the first time you switch to a different model. Once the window ends, any subsequent model switch does not trigger a new comparison window. MoEngage stores data only for the actively selected model from that point, so switching after the window ends will create gaps in your data.
  </Accordion>
</AccordionGroup>
