# Configure Attribution Settings

To configure the attribution settings:

1. Navigate to **Settings** -\> **Advanced settings** -\> **Conversion and attribution**.
2. The following fields are available:
   | Field Name | Description |
   | --- | --- |
   | **Default Attribution Model** | Choose the attribution model for your campaigns and flows by selecting one of the following: <ul><li>**All Interactions**</li><li>**Last Interaction**</li></ul> For more information about how each of these models works, refer to [All Interactions model](/user-guide/campaigns-and-channels/campaign-management-and-reports/conversion-and-attribution#all-interactions-model) and [Last Interaction model](/user-guide/campaigns-and-channels/campaign-management-and-reports/conversion-and-attribution#how-the-all-interactions-model-works). |
   | **Conversion view** | Each attribution model has certain attribution types associated with it. Based on the attribution model chosen, you get the following options in this dropdown list: <ul><li>The All Interactions model has the following options: **View Through Attribution**\*,\* **Click Through Attribution**\*,\* and **In Session Attribution**.</li><li>The Last Interaction model has the following options: **Click Through Conversion** and **Total Conversions**.</li></ul> |
3. Click **Save**. You are prompted for confirmation about changing the attribution model or type. Click **Done**. You get a success message, as shown in the following image. <img src="/images/moengage_cbdc3a.png" alt="SuccessMessage (1).png" />

## Switching Between Attribution Models

You can switch between the All Interactions and Last Interaction attribution models directly in **Settings** > **Advanced Settings** > **Conversion and Attribution**.

<Note>
   The Last Interaction model is the default for any newly created accounts.
</Note>

When you switch between attribution models, the campaign statistics pertaining to conversion are recalculated based on the model chosen.

![Switch\_AttribtuionType\_RevampedUI.gif](/images/moengage_0e587a.png)

For example, if the current setting is the All Interactions Model and there are campaigns with the same conversion goals and overlapping attribution windows, more than one campaign could get the attribution for the conversion. For more information, refer to [All Interactions Model](/user-guide/campaigns-and-channels/campaign-management-and-reports/conversion-and-attribution#all-interactions-model).

However, if the setting is changed to the Last Interaction Model, only the campaign with the deepest and latest attribution gets the attribution for conversion. Thus, some campaigns may experience a dip in conversion and revenue metrics when there is a switch from the All Interactions Model to the Last Interaction Model. Similarly, when there is a switch from the Last Interaction Model to the All Interactions Model, some campaigns may experience inflation in the conversion and revenue metrics.

### 12-Day Comparison Window

The first time you switch to a different attribution model, a 12-day comparison window starts automatically. During this window:

- MoEngage stores conversion data for both the previous and the new model simultaneously.
- You can compare conversion numbers across both models side by side before committing.
- After 12 days, you can review the data and choose which model to continue with.

Once you select a model after the comparison window, MoEngage stops storing data for the other model.

### Switching After the Comparison Window

You can switch attribution models again at any time after the 12-day window ends. However, MoEngage only retains data for the actively selected model. If you switch back to a previously used model, conversion data from the period when that model was inactive will not be available.

For example, if you switch from Last Interaction to All Interactions on day 13 and then switch back to Last Interaction on day 17, conversions from days 13–16 will not be recorded under the Last Interaction model.

<Warning>
Switching models repeatedly after the comparison window ends creates gaps in your conversion data. Once the 12-day window is over and you have decided on a model, MoEngage recommends staying with that model.
</Warning>

### Impact on Reports and Analytics

When you switch attribution models, conversion data in your campaign and flow analytics changes immediately:

- Conversion data before the switch date shows as zero for the newly selected model. MoEngage starts collecting data for that model from the switch date onward.
- Banners appear on Campaign Analytics pages, Flow pages, and all campaign pages to inform you of the switch date and explain why historical conversion data shows zero.
- Existing scheduled and periodic reports that include conversion columns will show zero for the period before the switch. You need to update those reports to include the conversion columns for the new model.

The attribution types available change depending on which model is active:

| Model | Attribution Types Available |
| --- | --- |
| All Interactions | View Through Attribution, Click Through Attribution, In Session Attribution |
| Last Interaction | Click Through Conversion, Total Conversions |

# Adding Conversion Goals to Campaigns

You can define the conversion goals during the third step of campaign creation for all channels. To add a conversion goal to a campaign:

1. Click **+ New goal** in the **Conversion Goals** section.
2. In the **Goal name** field, add a name to the goal.
3. In the **Event Name** dropdown list, select an event to be associated with the goal.
4. In the **Track goals for** section, add the attribution window (the duration from the time user received/clicked the campaign, for which a conversion goal will be tracked.)for the campaign. For more information, refer to \[Attribution Window\]/user-guide/settings/advanced-settings/campaign-attribution-settings#adding-conversion-goals-to-campaigns).
5. You can add multiple conversion goals to a campaign. You can track a maximum of five conversion goals for a campaign.
6. Use the **Primary goal** option to specify any of the goals added as the primary goal for the campaign.

For example, you want to track the number of customers who have purchased an iPhone after you send the campaign. (Event - purchase event, Product - iPhone). Your conversion goal setup might look to the goal in the following image:

![CG\_PrimaryGoal.png](/images/moengage_96bdaa.png)

With multiple conversion goals, you can track more than one business metric or visualize a funnel for your user actions. For example, you can check the following with multiple conversion goals:

- How many users have viewed the product page?
- How many of them added that product to the cart?
- How many of them made a successful order?

## Tracking Revenue Metrics

You can track revenue metrics for the primary conversion goal of a campaign. Enable the Revenue performance toggle for the primary goal.

## Attribution Window

The attribution window is the duration for which the conversion goal is tracked for users from the time they receive or click a communication from a campaign or Flow.

For example, there is an end-of-season sale campaign offering discounts to customers with a conversion goal of purchasing a product (Product Purchase event) and an attribution window of 36 hours. John is a user who receives the campaign at 7 PM today. If John purchases a pair of jeans tomorrow at 10 PM, he is said to have converted. This conversion is attributed to the end-of-season sale campaign because John has achieved the conversion goal within the attribution window specified for the campaign. If John purchases the pair of jeans at 7:30 AM the day after tomorrow, the conversion is not attributed to the campaign.

The attribution windows for various channels and Flows are specified below:

| Channel Name | Maximum Attribution Window |
| --- | --- |
| Push | 36 hrs |
| SMS | 10 days |
| Email | 10 days |
| WhatsApp | 10 days |
| InApp | 36 hrs |
| OSM | 36 hrs |
| Cards | 36 hrs |
| Connectors | 36 hrs |
| Flows | 10 days |

The minimum value for the attribution window is 1 hour for all channels and Flows.
