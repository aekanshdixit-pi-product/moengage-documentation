*Connected App campaigns* allow you to engage users through various third-party platforms integrated with MoEngage. This guide explains how to create and configure a connected app campaign to reach your target audience with relevant content and track your campaign's performance.

<Tip>
- The required connected app must be added to the MoEngage App Marketplace.
- A connector must be configured. For more information, refer to [Create a Channel Connector Config File](https://help.moengage.com/hc/en-us/articles/33116671627924).
- At least one integration for a connector must be added.
- Required connection authorization must be completed.
</Tip>

## Create a Connected App Campaign

To create a connected app campaign:

1. On the left navigation menu in the MoEngage dashboard, click **Engage**, and then click **Campaigns**.
2. On the All campaigns page, click **+ Create campaign**.
3. Under **Connected apps**, click the connected app you want to use:  
    <img src="/images/moengage_8d309a.png" />
4. Choose one of the campaign delivery types:
   - **One Time**
   - **Periodic**
   - **Event Triggered**

For more information on campaign delivery types, refer to [Delivery Types](https://help.moengage.com/hc/en-us/articles/360058616131).

Follow these steps to complete your connected app campaign setup:

1. [Target Users](#step-1-target-users): Define the campaign's target audience.
2. [Content](#step-2-content): Define the content to send.
3. [Scheduling and Goals](#step-3-scheduling-and-goals): Set the campaign's delivery schedule and goals.

### Target Audience

This section provides segmentation filters to help you choose the campaign's target audience. The following options are available:

All Users Filter by User Property Filter by User Behavior Filter by User Affinity Filter by Custom Segment

Use this filter to send the campaign to all app users. You cannot add multiple filters with this option.

<img alt="AllUsers_Segmentation.png" src="/images/moengage_1857d2.png" />

Use this filter to target users based on user properties such as first seen, last seen, city, and so on. Click the **User property** tab, and then do the following:

1. Select the desired attribute in the drop-down list. This shows filters based on the chosen attribute. For example, numeric attributes can have filter conditions such as 'is equal to', 'is not equal to', 'is between'; string attributes have different filters.
2. Enter the filter value(s) to create the desired user segment.
3. To add a filter for nested AND or OR conditions, click **+ Nested Filter**. (This step is optional.)

For more information, refer to [Create Segments](https://help.moengage.com/hc/en-us/articles/206169646-Create-Segments).

<img alt="UserProperty.png" src="/images/moengage_13e813.png" />

Use this filter to target users based on their behavior or events, such as App/Site Opened, Viewed Product, and so on. Click the **User behavior** tab, and then do the following:

1. Select the execution condition and the desired user activity (event).
2. Enter the event frequency and time duration for the event execution.
3. Click **+ Attributes**, and then add an event attribute with a filter condition.
4. Enter the filter value(s) to create a rule.
5. To add aggregation functionality, use **+ Aggregation**. For more information, refer to [Aggregation Operations](https://help.moengage.com/hc/en-us/articles/9722166234004).
6. To add a filter for nested AND or OR conditions, click **+ Nested Filter**. (This step is optional.)

For more information, refer to [Create Segments](https://help.moengage.com/hc/en-us/articles/206169646-Create-Segments).

<img alt="UserBehavior.png" src="/images/moengage_6fe654.png" />

Use this filter to target groups of users based on their actions with specific attribute values. Click the **User affinity** tab, and then do the following:

1. Select the desired user event or attribute.
2. Select the user affinity parameter from the following:
   - **Predominantly**: Select an affinity attribute with a filter condition, and then provide the affinity values.
   - **For a minimum of**: Select an affinity attribute with a filter condition, and then provide the affinity values.
   - **Most no. of times**: Enter a value or use the arrows to provide the top percentage of users.
   - **Least no. of times**: Enter a value or use the arrows to provide the bottom percentage of users.
3. Enter a value or use the arrows to provide the duration of the event execution.
4. To add more attributes to the filter, click **+ Attribute**.
5. To add a filter for nested AND or OR conditions, click **+ Nested Filter**. (This step is optional.)

For more information, refer to [Create Segments](https://help.moengage.com/hc/en-us/articles/206169646-Create-Segments).

<img alt="UserAffinity.png" src="/images/moengage_1f2d47.png" />

Use this filter to select a custom segment as the target audience.

1. Select a custom segment as the target audience from the list of available custom segments.
2. To add a filter for nested AND or OR conditions, click **+ Nested Filter**. (This step is optional.)

For more information, refer to [Custom Segments](https://help.moengage.com/hc/en-us/articles/206528613-Custom-Segments).

<img alt="CustomSegment.png" src="/images/moengage_3e15c8.png" />

After choosing the filter criteria, you can also:

- **Create multiple filters**: Combine multiple user property rules, user activity rules, and custom segment rules to create a complex user segment.
  1. Click **+ Filter**.
  2. Click **AND** or **OR**.
  3. Select the attribute or event, and then define the filter conditions.
- **Exclude users**: Select this check box to define filter criteria for users to exclude from the campaign.
- **Send campaign to the users opted-out in preference management**: Select this check box to include users who opted out of preference management in the campaign. For more information, refer to [Preference Management](https://help.moengage.com/hc/en-us/articles/360037357812).
- **Reset filters**: Click this option to reset the segmentation filters.
- **Show count**: Click this button to display the total number of users currently in the configured segment.

  <img alt="OtherFilterOptions.png" src="/images/moengage_862638.png" />

## Step 2: Content

In this step, define your campaign's content based on your selected connected App.

### Configure

1. **Connector**: Select the connector you want to use for this campaign. The available connectors vary depending on the selected connected app. For example, Telegram's connectors include *Send a message to user or channel*.
2. **Select connection**: Select a connection that you added to the MoEngage App Marketplace for this campaign.

### Content

Configure your campaign with the required information based on your selected connected app. If you created a new connected app, enter the information defined by your app's configuration.

For a list of MoEngage-created connected apps and setup instructions, see [Connected Apps Overview](https://help.moengage.com/hc/en-us/articles/32005939221524-Overview#h_01JCWFF3VB4Y0E1MJ23XRYG7NQ).

- Unique ID
- Email ID
- Mobile Number
- Google Advertising Identifier
- iOS Identifier for Advertising

**When will event-triggered campaigns be sent?**

The campaign will be sent only when:

- The event property/attribute which is being used to create these types of triggers (like the flight\_time example), is in a valid date-time format. If it is not in date-time format, the string should have data in one of the [supported formats](https://help.moengage.com/hc/en-us/articles/206541583#01FX4N0HBQ7721SX2Y6SCSKE9S). If the selected attributes do not contain data in any of these formats, the campaign won't be sent to your users.
- The date/time value from the attribute is greater than the time at which the trigger condition is fulfilled.
- The date/time value from the attribute is less than 30 Days + the time at which the trigger condition is fulfilled e.g. If your trigger condition is fulfilled at date D, your notification will be sent only when the attribute value is less than D+30.
- The intelligent delay optimization (**With Delay**) does not work for campaigns sent on relative timing.

### Conversion Goals

You can add conversion goals to track the business metrics for your campaigns. You can track a maximum of five conversion goals. You can choose any app event as a conversion goal activity and set more than one conversion goal. The first goal will be treated as a primary goal.

For example, if you want to track the number of customers who have purchased an iPhone after your campaign was sent. (Event: Purchase event, Product: iPhone). Your conversion goal setup might be similar to the following image:

*<img alt="ConVErsionGoals.png" src="/images/moengage_b5f5af.png" />*

With multiple conversion goals, you can track more than one business metric or visualize a funnel for your user actions. For example, if you want to check the following, you can do so with multiple conversion goals.

- How many users have viewed the product page?
- How many of them added that product to the cart?
- How many of them made a successful order?
