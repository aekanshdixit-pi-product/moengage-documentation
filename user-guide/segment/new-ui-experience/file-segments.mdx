Growth teams often perform their custom analysis on the users and user data. Post the analysis, growth teams would want to use these users in a campaign or perform further analysis.

These users can be uploaded to MoEnage and used as the File Segment. The File Segment is a list of users. File Segment can be used for MoEngage campaigns or Analytics.

File Segments provide flexibility to create and target user segments that you may have identified through some user analysis at your end. You can import a list of users containing one of the User Attributes that identifies a unique user, and a custom segment of these users gets created.

# Create a File Segment

To create a file segment, perform the following steps:

1. On the sidebar menu in MoEngage, hover over the **Segment** menu item <img alt="Screenshot 2025-06-16 at 3.25.14 PM.png"  src="/images/moengage_a43049.png"  />  . The **Segment** menu appears.
2. Click **+ All Segments**.  
    <img alt="Screenshot 2025-06-17 at 10.54.43 AM.png"  src="/images/moengage_2c3e92.png"  />   
   The All Segments page appears.
3. Click **+ Segment**.  
    <img alt="2025-06-17_12-05-45.png" src="/images/moengage_11f966.png" />   
   The **New Segment** dialog box appears.
4. Click **File Segment**.  
    <img alt="Screenshot 2025-06-17 at 12.08.20 PM.png"  src="/images/moengage_be5dd2.png"  />   
   The **Create file segment** dialog box appears.
5. Perform the following steps:  
    <img alt="File_segment_3.gif"  src="/images/moengage_94aa1c.png"  />
   1. Type a **Custom segment name** and **Segment description**.
   2. Select the file with the user list. You can either upload a CSV or provide a URL to it.
   3. Select the User Attribute that your list is using. For example, if you have a list of the Driving IDs of your users, select Driving ID from the user attribute dropdown.
   4. Provide one or more email IDs where MoEngage can inform, once the custom segment is created.

**Note:** If you don't see the segment after receiving the email, please refresh the page and search.

<Warning>
A user attribute value must uniquely identify a single user. If an attribute value identifies multiple users (e.g. a city attribute with the value 'London' may have many users) these multiple users may get mapped to a single attribute, become part of the segment, and may block other users from being included in the segment.

For example, you upload 5 emails, and the first email matches with 3 users (A, B, and C), and the remaining 4 emails match with one user each (D, E, F, and G). In this case, the system will randomly pick 5 users to be added to the file segment.

The system does the processing in batches of 2,000 attribute values and add maximum of 2,000 users into file segment for each batch. Even with fewer user attribute values, more users could be added to the file segment. For example if 1500 user attribute values results into 2500 users, system will randomly add 2000 of these 2500 users into the file segment.
</Warning>

## File Format

Please check your file before uploading it to MoEngage systems. Ensure that -

- The file does not have duplicate values,
- The file does not have special characters,
- The file does not have double quotes,
- The file does not have empty rows or empty columns
- It should be a CSV file with a single column containing the user attribute.
- The file size should not be more than 64 MB.  
  [Sample file link](https://app-cdn.moengage.com/assets/Sample_GAIDs.csv)

## Custom Segment Availability

As soon as the MoEngage system receives the File Segment request, it creates the custom segment. After creating a custom segment, the system processes the users and adds those users to the custom segment.  
Hence custom segment is **available** when the file is processing and also if the file processing is failed. When the File Segment is processed, its status (either completed or failed) is communicated through an email.

In case the file upload/processing has failed, you can upload the file again using the same custom segment name. The system will add the new user to the provided custom segment. This functionality also enables adding new users to any existing file custom segment.

## Status and Warnings for File Segment Processing

To prevent issues arising from using file segments before they are fully processed, MoEngage provides warnings in the UI. These warnings help you understand if a segment is still being processed and when it's advisable to use it.

<Info>
You will see these warnings on the following pages:

- Create segment page
- **Target audience** sections on all the campaign creation pages
- **Target audience** sections on analytics pages
</Info>

### Warnings During Segment Usage

When you select a file segment that is still being processed for use in creating another segment, while targeting a campaign, or in analytics, a warning message will be displayed. The warning is displayed based on the following conditions:

- **If a single selected segment is being processed**: "Users are being updated in this segment - `<*Custom Segment name*>`. We suggest that you wait until the confirmation email is received to use this segment in a campaign. Keep an eye here for the latest update."
- **If segments nested in another segment are processing**: "Users are being updated in these segments - `<**Custom Segment 1, Custom Segment 2, Custom Segment 3 names**>`. These segments are nested in the parent segment chosen. We suggest that you wait until the confirmation email is received to use the segment in a campaign. Keep an eye here for the latest update."

Refresh the page or choose the segment again to see its latest status. This warning will no longer appear after the last file in the queue is successfully processed.

### Warning in the Segment Detail Page

If you navigate to the detail page of a custom segment that is currently being updated (i.e., a file upload is still processing), a warning message will be displayed as follows: "Users are being updated in this segment - `<*CS name linked*>`. We suggest that you wait until the confirmation email is received to use this segment in a campaign. Keep an eye here for the latest update."

Refresh the page to see the segment's latest status. This warning will no longer appear after the file processing is successfully completed.

## User Availability

File Segments can only have users that are present in MoEngage. If you upload a user, who is not present in MoEngage systems, that user is not processed and can not be part of the File Segment.

To upload new users into the MoEngage and create a File Segment of those users, refer to [Import Users](/user-guide/data/imports/file-based-imports/import-user-data-using-csv).

The total number of users uploaded via the File segment is limited to 2 million per day.  
(This limit is customizable, please get in touch with support to increase the limits.)

# Edit a File Segment

Editing an existing file segment is possible via the MoEngage dashboard. On the MoEngage dashboard, you can perform the following actions on a file segment:

1. Add users: A new list of users can be added to an existing file segment.
2. Remove users: A list of users can be removed from an existing file segment.
3. Replace users: A list of users can be replaced in an existing file segment.

Step 1:

Navigate to the file segment that you would like to edit from the Segment \> All Segments page.

Step 2:

From the Actions tab, click on the More Actions vertical ellipses and choose the Edit option.

Step 3:

1. Choose the Edit action that you want to perform on the file segment among Add users, Remove users and Replace users.
2. Select the file which has the user list to be added, removed, or replaced. You can either upload a CSV or provide a URL to the same.
3. Select the User Attribute that your list is using. For example, if you have a list of the Driving IDs of your users, select Driving ID from the user attribute dropdown.
4. Provide one or more email IDs where MoEngage can inform, once the custom segment is processed.

<img alt="File_segment_4.gif" src="/images/moengage_71d706.png" />

All functionalities of file segments: Create segments, Remove and Replace users from the File Segment can also be accessed using [File Segment API](https://developers.moengage.com/hc/en-us/articles/4405027805844-Custom-File-Segment-API).

# Archive/Unarchive a File Segment

File Segments can be archived from the MoEngage dashboard.

1. File segments that have not run (query not executed) in the last 60 days are automatically archived.
2. All the users from the archived File Custom Segment are dropped after 30 days of archival. Hence after 30 days of archival, the archived custom segment returns 0 users.

<Accordion title="Why Does the User Count Differ Between the File Segment in MoEngage and the Uploaded File?">

## Problem

There is a discrepancy in the user count between the uploaded file and the file custom segment in MoEngage.

</Accordion>
## Explanation

The following are the possible reasons for these differences:

- Zero count
  - **Wrong attribute provided:** The attribute selected during the file segment upload does not match the values provided in the file. For example, if the attribute selected is *age* (double data type) but the file contains values for *name* (string data type), there will be no matching users in MoEngage, resulting in a zero count.
  - **File not downloadable:** If the file URL provided is private or expired, MoEngage cannot download the file, leading to no users being added to the custom segment and a zero count.
  - **Inappropriate CSV content:** Issues such as leading or trailing spaces in values, rows with only spaces, non-UTF-8 characters, values enclosed in double quotes, or values ending with a comma can result in a zero user count.
  - **File processing delay:** After you upload a file, users will not be immediately added to the custom segment. Processing time increases with the number of values in the file. Querying the custom segment immediately after upload may result in a zero count.
- Less count
  - **Less users in MoEngage:** If the values mentioned in the file do not match those in MoEngage, they will not be considered, resulting in a lower user count.
  - **Inappropriate CSV content:** Similar to the zero count scenario, issues with CSV content can lead to a lower user count.
  - **File upload still processing:** A custom segment is created with zero users immediately after the file upload. Users are then added during the file processing. Querying the segment during processing may display a lower user count.
- More count
  - **Attribute value matches multiple users:** If the value provided in the file matches multiple users in MoEngage, all those matched users will be added to the custom segment, resulting in a higher user count.

For more information, read [File Segments](/user-guide/segment/segment-operations/file-segments).

<Accordion title="Why Does CSV Upload to File Segment Fail?">

## Problem

While uploading a CSV file in a file segment, the following error message appears:

There seems to be an error. We are working to fix this.

</Accordion>
## Solution

Perform the following steps:

1. **Verify the File Format**:
   - Ensure the file is in the CSV format.
2. **Check the File Size**:
   - Confirm that the file size does not exceed 64 MB. If the file is larger than the allowed limit, reduce its size before uploading.
3. **Confirm the Unique Identifier Column**:
   - Ensure the file includes only one column designated as the segment's unique identifier.
   - It is recommended to use a unique identifier (UID).
4. **Remove Empty Rows and Junk Data**:
   - Check for empty rows at the end of the file.
   - Remove any unnecessary or duplicate values that might cause issues.
5. **Match Record Values to Attribute Data Types**:
   - Ensure the record values align with the specified data types.
6. **Avoid Unsupported characters**:
   - Remove any unsupported characters in the values. For example, special characters, double quotes (") are not supported.

For more information, read [File Segments](/user-guide/segment/segment-operations/file-segments).

<Accordion title="Why Does Third-Party Link Upload to File Segment Fail?">

## Problem

While uploading a third-party link in a file segment, the following error message appears:

There seems to be an error. We are working to fix this.

</Accordion>
## Solution

Perform the following steps:

1. **Whitelist IPs for private files**:
   - If the file is private, whitelist the relevant IP addresses to grant access exclusively to MoEngage.
2. **Verify the file format**:
   - Ensure the file is in the CSV format.
3. **Check the file size**:
   - Confirm that the file size does not exceed 64 MB. If the file is larger than the allowed limit, reduce its size before uploading.
4. **Confirm the unique identifier column**:
   - The file must contain only one column that serves as the segment's unique identifier.
5. **Remove empty rows and junk data**:
   - Check for empty rows at the end of the file.
   - Remove any unnecessary or duplicate values that might cause issues.
6. **Match record values to attribute data types**:
   - Ensure the record values align with the specified data types.
7. **Avoid unsupported characters**:
   - Remove any unsupported characters in the values. For example, special characters, double quotes (") are not supported.

For more information, read [File Segments](/user-guide/segment/segment-operations/file-segments).
