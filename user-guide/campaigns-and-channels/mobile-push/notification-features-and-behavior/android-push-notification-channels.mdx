Set up [Android notification channels](https://developer.android.com/develop/ui/views/notifications#ManageChannels) (introduced in Android Oreo, API Level 26) to provide users with granular control over their push notification preferences. This guide outlines the prerequisites, configuration steps, and error-handling procedures necessary for a seamless integration.
Android Push Notification Channels provide a unified system that helps users manage their notifications effectively. Instead of globally toggling notifications for a specific application, users can modify preferences, such as opt-outs, sounds, and vibration, for individual notification channels.

<Info>
**Prerequisites**
To successfully use and display notification channels, ensure the following requirements are met:
    * **Target SDK:** Your application must target Android Oreo (Android 8.0 / API level 26) or later.   
    If your app targets an older version but is installed on an Android Oreo device, it will behave as it would on Android 7.1 / API level 25.
    * **MoEngage SDK:** Update the Android SDK version of MoEngage to **v8.0.00** or higher.
</Info>

## Configuration Workflow

Follow these steps to configure and utilize notification channels within your campaigns.

### Step 1: Manage Notification Channels

After updating the SDK, you must map the existing notification channels created by your application developers to the MoEngage dashboard.

1. Navigate to the Content step.
2. Expand the **Basic details** section.
3. Locate the Notification channel drop-down menu.
4. Click **+ Manage notification channel** to open the configuration modal.

[Image: MoEngage dashboard displaying the 'Manage notification channel' drop-down list]  

The modal lists all channels your development team has configured in the app. Each row shows:

- Channel name
- Channel ID (as defined in the app code)
- Importance level — the priority that controls how intrusively the notification appears on the device.

<Tip>
The **Channel ID** and **Importance** values must match what your developers set in the app. Check with them if you're unsure which values to use.
Each channel name and ID must be unique. If you enter a duplicate value, the modal blocks saving and highlights the conflicting field with an error. Correct the value before saving.
[Image: Modal view showing validation errors for duplicate names and IDs ]
</Tip>

### Step 2: Set a Default Channel (Optional)

To streamline future campaign creation, you can designate a specific channel as your default.

1. Once a channel is added and validated, select the **Default** radio button next to the desired channel.  
2. Click **Save**. This channel automatically populates for future campaigns.

[Image: 'Manage notification channel' modal with a default channel selected ]

### Step 3: Select a Channel for Campaign Delivery

Select the most appropriate channel for your current campaign. All notifications routed through a specific channel inherits its configured behavior.  

<Info>
**Fallback Behavior:** If an invalid Channel ID is provided, or if a channel selection is missed, the system automatically utilizes the **General** channel (the designated fallback channel) to ensure successful delivery to users on Android 8.0 (API level 26) and later.
</Info>

## Channel Importance Levels Reference

When selecting a channel, the UI displays a descriptive summary of its behavior based on its assigned importance level.

| Importance Level | System Behavior Description |
| :---- | :---- |
| **Urgent** | Triggers a pop-up banner and plays a sound. These stay visible on the lock screen and status bar. Best for time-sensitive tasks. |
| **High** | Plays a sound and appears on the lock screen, but won't show a pop-up banner. Use this for important updates that don't need to interrupt your work. *(Note: The UI may explicitly list active attributes, such as "Sound on, Vibration on", depending on specific configuration).* |
| **Medium** | Silent alert. Icons appear in the status bar and lock screen, but there is no sound or vibration. Good for general activity feeds. |
| **Low** | Silent and discreet. These won't show in your status bar or play a sound. You'll only see them when you pull down your notification list. |
| **None** | Completely hidden. No sounds, no status bar icons, and no alerts in the notification shade. |

[Image: Dropdown menu states showing definitions for Urgent, High, Medium, Low, and None importance levels ]

## Exception Handling: Unknown or Missing Importance

If a user selects a channel where the importance level is undefined or missing from the API response, the system requires immediate attention to prevent delivery discrepancies.  

**UI Indicators for Unknown Importance:**

1. **Dropdown Status:** The channel will display a status of **Unknown**, accompanied by a yellow warning indicator.  
2. **Inline Warning:** The campaign creation view will display the message: *"Channel importance: Unknown. Ask channel implementer to define the required further behavior for the device."*  
3. **General Error Banner:** A broader system warning may also appear, stating: *"Missing importance breakdown. The behaviour of notification cannot be determined. Explore channel importance for expected behaviour."*

[Image: Campaign creation view displaying warning messages for an 'Unknown' channel importance state ]  

<Info>
 The platform interface does not support the dynamic creation, reading, updating, or deletion of app-level notification channels. Application developers must programmatically manage these lifecycle events within the application codebase.
</Info>

## End-User Experience and Controls

It is important to note that application developers cannot programmatically modify the behavior of a notification channel once it has been created and submitted to the Android Notification Manager. After initialization, the end-user retains complete control over these settings.  
Users can navigate to their device **Settings** or long-press an active notification to modify behaviors or completely block a channel. User modifications to any of the following characteristics will permanently override the application's default settings for that specific channel:

* Importance  
* Sound  
* Lights  
* Vibration  
* Show on lock-screen  
* Override "Do Not Disturb"
