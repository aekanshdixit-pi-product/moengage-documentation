## Overview

You can configure how device data is categorized in your analytics to better align with your reporting needs. These configurations allow you to define specific App Version buckets and prioritize which OEMs (Original Equipment Manufacturers) or browsers appear in your reports.

To access these settings, navigate to **Settings** \> **Channels** \> **Push** and select the **Device analytics** tab.

## App Versions Buckets

You can group distinct app versions into buckets to analyze performance trends across releases (for example, grouping all Beta versions versus stable versions or grouping by every major version release).

- **Bucket allocation:** You can define up to four App Version Buckets.
- **Configuration:** Select specific version numbers from the drop-down list to assign them to a bucket (for example, Bucket 1).

<Info>
- MoEngage recommends combining your Android and iOS versions into the same bucket. Since analytics data is displayed separately for each platform, this approach allows you to effectively utilize the four available buckets for both operating systems.
- When you release a new app version, MoEngage recommends adding it to a bucket in these settings to ensure explicit tracking. Versions not assigned to a bucket are categorized as *Others*.
</Info>

## OEMs & Browsers Selection

Use the **OEM & browser** tab to specify which device manufacturers and web browsers appear in your analytics. You can customize the default list to reflect the devices most used by your subscribers.

<Info>
Changes made to OEM lists, Browser lists, or App Version Buckets apply to campaigns created *after* you save the configuration.

- **New campaigns:** MoEngage applies the new settings to any campaign created after the update.
- **Periodic/Recurring campaigns:** Existing recurring campaigns continue to use the settings that were active when they were created. To apply new settings to a recurring campaign, MoEngage recommends creating a new parent periodic campaign (or duplicating the existing one) after saving your configuration.
</Info>

<img alt="device analytics OEMs &amp; Browsers.png" src="/images/moengage_fe38c6.png" />

### Most Common Browsers (Web Push)

Select the browsers you wish to track for your web subscribers (for example, Google Chrome, Microsoft Edge, Apple Safari).

- **Selection:** You can select up to five specific browsers to track explicitly.
- **Categorization:** Any browser not selected is grouped into the *Others* category.

### Most Common OEMs (Android Push)

Select the Android device manufacturers (OEMs) you wish to track (for example, Samsung, Oppo, Vivo).

- **Selection:** You can select up to five specific OEMs to track explicitly.
- **Categorization:** Any manufacturer not selected is grouped into the *Others* category.
