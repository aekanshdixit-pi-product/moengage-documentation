---
title: "Configure an RCS Sender"
---

RCS Senders of MoEngage refers to the entities authorized to initiate and transmit Rich Communication Services (RCS) messages through the MoEngage platform.    
Configuring an RCS sender is important because it's the authorized entity that MoEngage uses to transmit messages, making it a mandatory prerequisite for running any RCS campaigns. It also establishes the technical integration between your business's RCS provider and the MoEngage platform.  
RCS senders can be added and managed in the **Sender configuration** tab of SMS & RCS settings. You can access **Sender configuration** tab using this path: **Settings** \> **Channels** \> **SMS & RCS** \> **Sender configuration**.

<img alt="RCS 10.png" src="/images/moengage_b856ab.png" />

# Configure Your Sender

Perform the following steps to add an RCS sender:

1. On the **RCS** tab, click **Add RCS Sender**. The Add sender page is displayed.  
    <img alt="RCS 12.png" src="/images/moengage_3adf0e.png" />
2. In step 1, **Service Provider**, select your service provider. For more information on RCS providers, refer [here](https://partners.moengage.com/hc/en-us/categories/35361293352724-RCS-Providers).   
    <img alt="Screenshot 2025-04-09 at 12.24.21 PM.png" src="/images/moengage_68ae8f.png" />
3. In step 2, **Sender Details**, add the required sender details. To complete the sender configuration, refer to the respective [RCS provider articles](https://partners.moengage.com/hc/en-us/categories/35361293352724-RCS-Providers).
4. Click **Save**.
5. After adding the sender, it will be displayed under the **Sender configuration** \> **RCS** tab.  
    <img alt="sender delete (1)mk.png" src="/images/moengage_d2c74c.png" />
6. Click the ellipsis icon ( <img alt="ellipsis.png"  src="/images/moengage_1707c7.png"  /> ) to edit or delete the sender. When you delete a sender, a pop-up appears that displays all the active campaigns associated with it. You must stop each active campaign to delete the sender.

# Other Settings

Before running the RCS campaigns, make sure to validate the following optional configurations:

- Ensure that the attribute that stores the mobile number of users is configured correctly in the [General Settings](https://help.moengage.com/hc/en-us/articles/7455149158676-Configure-SMS-General-Settings).
- Validate the Personally Identifiable Information (PII) Tokenization settings. This is a security measure used to protect sensitive data. PII tokenization allows brands to avoid sending PII data to MoEngage and to fetch it only when necessary to send a message. For more information, refer to [PII Tokenized Sending.](https://help.moengage.com/hc/en-us/articles/9111881294996-PII-Tokenized-Sending)
- Validate the delivery control settings to ensure that you send the optimum number of messages to your users and not during DND hours. For more information, refer to [Frequency Capping (FC)](https://help.moengage.com/hc/en-us/articles/15919660670356) and [Do Not Disturb (DND).](https://help.moengage.com/hc/en-us/articles/15919572705556)
