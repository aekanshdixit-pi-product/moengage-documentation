# Overview

The Campaign Analytics and Info page allows you to analyze campaign performance and access campaign details. This is your central hub for understanding what happened after a campaign was launched. Campaign Analytics helps you prepare performance reports, identify areas for optimization, and inform future campaign strategies.

<Info>
All metrics are calculated and displayed in your app's time zone.
</Info>

You can navigate to it by performing the following steps:

1. On the left navigation menu in the MoEngage UI, click **Engage**, and then click **Campaigns**.
2. On the **All Campaigns** page, click the row that displays information about the campaign you want to analyze. MoEngage displays the **Campaign Analytics and Info** page, which includes the following sections:
   - **Campaign Details**: Contains information about the campaign and its status. For more information, refer to [Campaign Details](#h_01GR88AAMKREGVV9J6D0W4DK4E).
   - **Analytics**: Contains information about the various metrics tracked for the campaign.
   - **Info**: Contains information about the campaign's target segment, channel-specific details, content, preview, and delivery schedule. For more information, refer to [Campaign Info](#01GMA5QTXJWK57Y38HRSXPWFKA).

## Campaign Details

This section contains administrative information about your campaign's status and details. Reviewing this helps you confirm that a campaign is running as expected and provides context for its performance. The following information is included:

- Campaign type
- Campaign ID
- Creation and update details for the campaign
- Team information
- Campaign status  
  A campaign can have the following statuses: Active, Expired, Not Sent, Paused, Scheduled, Sending, Sent, and Stopped. If Campaign Approval Flow (CAF) is enabled, a campaign can also be in the Under Review or Rejected status.

### Actions in the Campaign Details Section

|  |  |
| --- | --- |
| **Action** | **Description** |
| **Refresh** | This action loads the page with the latest campaign details. |
| **Edit** | This action lets you edit the campaign. |
| **Duplicate** | This action lets you duplicate the campaign and opens the campaign editor page with the details of the current campaign. The word *Duplicate* is appended to the duplicate campaign's name. |
| **Stop** | This action is allowed when the campaign is in the Sending state and lets you stop the campaign. |

## Analytics

The Analytics section displays campaign performance and delivery statistics.

Select the **Date range** to view analytics.

<img alt="Image 08-11-24 at 2.08 PM.jpeg"  src="/images/moengage_a065e3.png"  />

Metrics vary for each connected app. By default, MoEngage displays the following metrics:

<img alt="Image 08-11-24 at 2.04 PM.jpeg" src="/images/moengage_dc50df.png" />

- **Attempted**: This is the count of users who qualified for campaigns and were not frequency capped (FC). MoEngage then attempted to send requests for these users.
- **Failed**: This is the number of users for whom the connected app did not return a 200 (OK) status code. This also includes users for whom MoEngage could not find values for user or event attributes used in the connected app request.
- **Successfully Sent**: This is the number of users for whom the connected app received the request and returned a 200 (OK) status code.

Definitions for *Successfully Sent* and *Failed* may vary based on each connected app's requirements.

You can also view a breakdown of failed requests and their reasons:

The **Personalization failed** error occurs when MoEngage cannot find values for one or more event or user attributes used in the request for a specific user. MoEngage displays Error Codes as they are received from the connected app. The count indicates the total number of personalization failures.

### Conversions

If you have attached a conversion goal to your connected app campaign, you can view the following details:

- **Conversion Events**: The number of conversion events users performed within 36 hours of a successful connected app request.
- **Conversions**: The number of unique users who performed the conversion event at least once within 36 hours of a successful connected app request.
