Native In-App templates are pre-built layouts you can customize without writing code. They support the most common in-app messaging use cases — pop-ups, full-screen messages, ratings, and nudges — and are the fastest way to launch a campaign.

# Native template types

| Template type | Description |
| --- | --- |
| **Popup** | Appears in the center of the screen over a transparent overlay. Use it to focus user attention on promotions or important announcements. |
| **Full Screen** | Occupies the entire screen and blocks the app. Use it for critical information, such as mandatory app updates. |
| **Rating** | A pop-up template that lets users provide a 5-star rating within the app. Use it to collect user feedback. |
| **Nudge** | A non-intrusive message that lets users continue interacting with your app. Nudges can appear embedded in the app or at the top or bottom of the screen. Refer to [Nudge templates](#nudge-templates). |
| **Self-Handled** | Renders the message content yourself while using MoEngage delivery, segmentation, and personalization. Refer to [Self-handled In-App template](/user-guide/campaigns-and-channels/in-app-message/templates/self-handled-in-app-template). |

# Customize images

Use high-definition images for native templates, but keep the file size under 2 MB. Before an in-app message is shown, the image is downloaded from a CDN. Larger images can cause delays, especially on slower connections.

## Image aspect ratios

MoEngage recommends the following aspect ratios for each template type:

| Template name | Recommended aspect ratio (width:height) |
| --- | --- |
| **Popup - Image, Text & 3 Buttons** | Images automatically resize. For optimal display, use a 1:1 aspect ratio with at least 50x50 pixels. |
| **Popup - Image, Text & Button** | Images automatically resize. For optimal display, use 1:2, 2:3, 3:2, or 2:1 with a minimum of 450x450 pixels. |
| **Popup - Image, Header, Text & Button** | Images automatically resize. For optimal display, use 1:2, 2:3, 3:2, or 2:1 with a minimum of 450x450 pixels. |
| **Popup - Image & 2 Buttons** | Images automatically resize. For optimal display, use 1:2, 2:3, 3:2, or 2:1 with a minimum of 450x450 pixels. |
| **Popup - Rating with Image** | Images automatically resize. For optimal display, use 1:1 with at least 50x50 pixels. |
| **Popup - Image** | High-resolution images recommended. Max 1200x2000 pixels, min 450x450 pixels. |
| **Popup - Image & Button** | High-resolution images recommended. Max 1200x2000 pixels, min 600x600 pixels. |
| **Full Screen - Image, Text & Button** | Use 3:4 or 3:5 aspect ratio, minimum 450x600 pixels. |
| **Full Screen - Image** | Use 3:4 or 3:5 aspect ratio, minimum 450x600 pixels. |
| **Nudge - Image** | Use wide images with approximate 5:1 or 6:1 aspect ratio. |

## Accepted image types

You can upload `.png`, `.jpeg`, and `.gif` images for any native template.

To use GIFs in Android native templates, complete the integration steps in the [Android In-App NATIV SDK documentation](/developer-guide/android-sdk/in-app-messages/in-app-nativ). The iOS platform supports GIFs by default.

## Image quality and size

Use high-resolution images to prevent distortion on large screens. Large image files may fail to download on devices with unreliable internet connections. Keep images under 2 MB to balance image quality and download time.

## Image scaling for devices

In-app messages display on devices with various screen sizes and DPIs. Images you upload are scaled up or down to fit. For example, if you upload a 600 x 1000 pixel image for a Popup - Image template that displays on a device with a 320-pixel width where the message occupies 400 pixels, the image's width scales down to 300 pixels and its height becomes 500 pixels (preserving the 3:5 aspect ratio).

## Background image

To configure a background image:

1. In **Step 2** of campaign creation (**Content**), click **Template Settings**.
2. Under **Template background image**, select one of the following:
   - **Image URL** — type or paste the image URL.
   - **Upload Image** — drag and drop, or click **Upload from Computer**.

![Options to set a template background image.](/images/moengage_9594e4.png)

<Warning>
  Background images do not scale like other in-app content. The background image stretches to fit the message dimensions calculated from inner widgets.
</Warning>

## Background color

Set a background color for your message, or remove it to make the message transparent.

1. In **Step 2** of campaign creation, click **Template Settings**.
2. Under **Background color**, select a color.
3. Under **Phone background color**, select a color.

![Options to set background color for a template.](/images/moengage_5fc819.png)

# Nudge templates

Nudges are non-intrusive in-app messages that let users keep interacting with the app. MoEngage offers the following native nudge templates:

| Template name | Description |
| --- | --- |
| **Popup - GIF & button** | Background image, GIF, and buttons. Use for announcements. |
| **Nudge - Rating** | Background image, text, ratings, and buttons. Use to collect user feedback. |
| **Nudge - Text & button** | Background image, text, and buttons. Use for offers, promotional messages, and tips. |
| **Nudge - Image & text** | Background image, text, and an image. Use for offers, promotional messages, and announcements. |
| **Nudge - GIF** | Background image and a GIF. Use for announcements. |
| **Nudge - Video** | Background image and a video. Use for announcements. |
| **Popup - Video & button** | Background image, video, and a button. Use for announcements. |
| **Nudge - Image only** | Background image and an image. Use for offers and promotional messages. |
| **Nudge - Text only** | Background image and text. Use for offers, promotional messages, and tips. |
| **Nudge - Image, text & button** | Background image, text, images, and buttons. Use for offers, promotional messages, and tips. |

<Info>
  Nudges are supported from In-App iOS SDK version 4.15.00 and In-App Android SDK version 7.0.0. For more information, refer to [In-App Nativ Nudges](/developer-guide/android-sdk/in-app-messages/in-app-nativ).
</Info>

<Warning>
  Do not select the **On custom event** trigger for nudges. Nudges do not work with event triggers. To ensure iOS nudges remain visible within the app interface, turn on the **iOS Safe Area** toggle in **Template Settings** when creating content for iOS.
</Warning>

## Create a nudge campaign

1. In **Step 2** of In-App campaign creation, type "Nudges" in the **Search templates** box, or select **Nudges** from the **Filter By Type** menu. The template selector displays all nudge templates.
2. Hover over a nudge template, then click **Select**. The template editor displays.
3. Configure the nudge using the editor sections below.
4. Preview the template.
5. Test your campaign. For more information, refer to [Test your In-App campaign](/user-guide/campaigns-and-channels/in-app-message/create/test-your-in-app-campaign).
6. Schedule your campaign in **Step 3**, then publish it.

|  | **Minimised Container Width** | Width of the nudge container. Available only for the minimized display. |
|  | **Nudge Background** | **Upload Image**, **Image URL**, or **Background Color**. |
| **Animation** | **Entry Animation** | Animation when the nudge first appears. |
|  | **Exit Animation** | Animation when the nudge is dismissed or removed. |
| **Close Button** | **Display** | Show or hide the close button. |
|  | **Type** | Button type. Some options are platform-specific (Android-only or iOS-only). |
|  | **Close Button Placement** | **Right** or **Left**. |
| **Margin** | **Margin** | Margin for the nudge container (top, bottom, right, left). |
| **Border** | **Width**, **Radius**, **Border Color** | Border width and radius in pixels, and border color. |

### Image, GIF, or video element

| Section | Field | Description |

# Configure accessibility

Make native In-App messages accessible to all users. For accessibility principles across MoEngage, refer to [Accessible campaigns in MoEngage](/user-guide/campaigns-and-channels/getting-started/campaign-content/accessible-campaigns-in-moengage).

## Add alt text for visuals

Alt text describes images and GIFs for screen reader users.

- **Drag-and-drop editor** — when you add an **Image** or **GIF** element, type a description in the **Alt Text** box.
- **Native templates** — type alt text in the **Accessibility Text** box (for example, on Popup—Image, Text, and Button).
- **Custom HTML** — add the `alt` attribute with a meaningful description to all `<img>` tags directly in your HTML.

Considerations:

- **Describe the image** accurately, including any text in the image. For tips, refer to [Accessible campaigns in MoEngage](/user-guide/campaigns-and-channels/getting-started/campaign-content/accessible-campaigns-in-moengage).
- **Personalization is supported** in alt text using user attributes or event data.
- **Static UI elements** like the **Close** button (X) include default alt text — no manual configuration needed.

## Video accessibility

For video content:

- **Captions and transcripts** — generate them with external tools and link or embed them near the video. MoEngage does not auto-generate transcripts.
- **Video controls** — add **Accessibility Text** to **Play**, **Pause**, **Mute**, and **Unmute** buttons.

## Rating template accessibility

For the **Nudge - Rating** template, the screen reader automatically reads the selected rating followed by the associated text (for example, "4 out of 5 stars, Beyond Expectations").

## Font size and layout

Native In-App messages adapt to user preferences:

- **Font scaling** — text generally respects user font-size settings on Android and iOS.
- **Content scrolling** — Full Screen messages scroll automatically when content overflows. Nudges scroll their content while keeping the nudge anchored.

## Keyboard navigation and focus

The MoEngage SDK manages keyboard navigation and focus:

- **Focus** shifts to the message when it appears and returns to the previous element when it closes. Focus is trapped for intrusive messages.
- **Close button** — the standard **Close** (X) button has an accessible label. If you build a custom close button in HTML, ensure it is keyboard accessible with a proper accessible name.

## Touch target sizes

- **Native templates** are designed to meet accessibility guidelines for touch targets.
- **Custom HTML** — ensure all interactive elements (buttons, links) are large enough for easy tapping. For guidelines, refer to [Accessible campaigns in MoEngage](/user-guide/campaigns-and-channels/getting-started/campaign-content/accessible-campaigns-in-moengage).

## TV considerations

For In-App campaigns on TV, follow the platform's accessibility requirements and remote-control navigation patterns. Refer to [TV templates](/user-guide/campaigns-and-channels/in-app-message/templates/native/tv-templates).
