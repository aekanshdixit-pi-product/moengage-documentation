When you configure a Brand Book for your workspace, you can apply your brand's colors, fonts, typography, button styles, and hyperlink styles to In-App templates. Templates open pre-styled with your brand assets, so you spend less time matching colors and fonts manually.

## Prerequisites

- Your workspace has a published Brand Book. To set up brand assets such as logos, colors, fonts, typography, and buttons, refer to [Brand Book](/user-guide/settings/account/brand-book/brand-book).

## Supported Template Types

In the template selector, the **Filter By Editor** options determine whether Brand Book styles apply. Brand Book styles apply to the following editors:

- **NATIVE**
- **No code gamification**
- **Drag and drop**

Brand Book styles do not apply to:

- **HTML**
- **Self handled**

![Screenshot of the Filter By Editor options in the In-App template selector, showing NATIVE, HTML, Self handled, Drag and drop, and No code gamification.](/images/brand-book-filter-by-editor.png)

## Apply Brand Book Styles to a Template

In **Step 2** (**Content**) of In-App campaign creation, on the **Prebuilt and saved templates** tab, the template gallery has a filter in the top-right corner with these options:

| Option | Behavior |
| --- | --- |
| **All** | Shows all templates in their default colors and styles. You can still apply Brand Book colors manually from the color pickers. |
| **Pre-built** | Shows the templates provided by MoEngage. |
| **Saved templates** | Shows the templates you have saved. |
| **Brand book** | Shows templates pre-styled with your Brand Book colors, fonts, typography, and button styles. |

To open templates with your brand assets applied automatically, select **Brand book**, and then select a template.

![Screenshot of Step 2 of In-App campaign creation, showing the template gallery with the Brand book filter selected in the top-right corner.](/images/brand-book-template-selector.png)

<Info>
  Brand Book styles are starting points, not fixed values. In any color picker or style menu, your Brand Book options appear as selectable choices, but you can choose any color or value instead.
</Info>

## What You Can Customize

The styles that a template inherits from your Brand Book depend on the template type.

### Native Templates

When you open a native template from the **Brand book** tab, the following styles are inherited:

| Section | Inherited from Brand Book |
| --- | --- |
| **Text** | Font size and font color are inherited from your Brand Book typography. The font family (typeface) is not applied, because native templates use the fonts you add to your app. To set a font, select the **Custom font** checkbox. |
| **Buttons** | Font color, button color, and button style are inherited. |
| **Rating** | The **Star color**, rating descriptions, and **Rating description font size** use your Brand Book typography and colors. |
| **Background color** | The template background color is applied from your Brand Book palette. |
| **Image** | The background uses the first color in your Brand Book palette by default. |

In any color box, click the color block to open the picker. Your Brand Book colors appear under **Brand Book Colors**.

In the **Text** section, the **Brand Book typography** drop-down list controls which typography style applies. Native templates use **Paragraph** by default. To apply a different style, select **Header 1**, **Header 2**, **Header 3**, or **Span**.

![Screenshot of the Text section of a native In-App template, showing the Brand Book typography drop-down list set to Paragraph, with font size, font color, and background color inherited from the Brand Book.](/images/brand-book-native-typography.png)

In the **Buttons** section, the **Brand Book button style** drop-down list controls which configured button style applies. Each option combines a tier and a state, such as **primary › default**. Select a different option to apply another Brand Book button style.

![Screenshot of the Buttons section of a native In-App template, showing the Brand Book button style drop-down list set to primary, default, with font color and button color inherited from the Brand Book.](/images/brand-book-native-buttons.png)

The **Rating** section inherits the star color, rating descriptions, and rating description font size from your Brand Book.

![Screenshot of the Rating section of a native In-App template, showing the star color, star rating descriptions, and rating description font size.](/images/brand-book-native-rating.png)

### Drag-and-Drop Editor

Templates built with the drag-and-drop editor inherit the following styles:

| Element | Inherited from Brand Book |
| --- | --- |
| **Headings** | **Header 1**, **Header 2**, and **Header 3** use your configured typography styles. Switch between them directly from the heading drop-down list on the element. |
| **Font family** | All default Brand Book fonts appear in the font family drop-down list. |
| **Colors** | Brand Book colors populate the color picker. |
| **Forms** | Form fields inherit the font family and font size from your Brand Book by default. |

![Screenshot of the Title properties panel in the drag-and-drop editor, showing the heading level set to H1, the font family set to a Brand Book font, and text and link colors inherited from the Brand Book.](/images/brand-book-drag-and-drop.png)

For more information about this editor, refer to [Drag and Drop Editor](/user-guide/campaigns-and-channels/in-app-message/templates/drag-and-drop-editor/drag-and-drop-editor).

### No-Code Gamification Templates

No-code gamification templates inherit the same styles as native templates. In addition:

- Wheel sectors and other elements use your Brand Book colors. Select an element to choose a different Brand Book color from the picker.
- In the color picker, your Brand Book colors appear under **Brand Book Colors**.

![Screenshot of a no-code gamification spin wheel template, showing the template background color picker with Brand Book Colors and the spin wheel preview.](/images/brand-book-gamification.png)

For more information about these templates, refer to [No-Code Gamification Templates](/user-guide/campaigns-and-channels/in-app-message/templates/no-code-gamification-templates).
