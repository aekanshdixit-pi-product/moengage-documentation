---
title: "Why Do Pop-Up and GIF In-App Campaigns Overlap?"
---

## Problem

Both a pop-up message and a GIF-based In-app message are displayed simultaneously, even though they were expected to appear separately.

## Cause

This happens when both campaigns are configured to trigger on the same screen with identical trigger conditions. In such cases, the messages are delivered simultaneously.

## Solution

To prevent overlap between these campaigns, try one of the following:

- Add a delay to one of the campaigns to ensure they don’t trigger at the same moment.
- Use different trigger conditions for each campaign.
- Place the campaigns on separate screens so only one message is displayed per user interaction.
