title: "Configure Web Push Settings"

This article describes the steps to configure the Web Push Settings.

1. Navigate to **Settings \> Channels \> Push \> Web Push**.
2. It consists of two sub-sections, **Configure** and **Notification Opt-in**.

<Tip>
  For a [Portfolio](/user-guide/settings/account/portfolio/portfolio)-enabled workspace, before configuring Web Push settings, you must add at least one project to your MoEngage [Portfolio](/user-guide/settings/account/portfolio/portfolio). For more information on how to add a project, refer [here](/user-guide/settings/account/portfolio/portfolio).
</Tip>

# Project selection

If your portfolio has multiple projects, a dropdown appears at the top of the **Web push** settings tab. Select the project you want to configure from the dropdown menu. The related settings (such as the default notification icon) load automatically. For portfolios with a single project, all settings are appended to that project.

# Notification Opt-in Configuration

You can configure the Web Push Opt-In information in this section. Opt-In refers to how push permission is sought from the users on your webpage.

- Select the preferred project from the **Project**dropdown. This field appears if you have configured more than one project on your portfolio. For more information, refer [here](/user-guide/settings/account/portfolio/portfolio).
- To enable this feature, you must enable the Web Push opt-in under the **Notification opt-in** tab.

![](/images/moengage_7552a0.png)

- MoEngage provides three types of push opt-in:
  - Two-Step Opt-In (Recommended)
  - 1-Click Opt-In
  - Self Handled Opt-in

<Tabs>
  <Tab title="Two-Step Opt-in (Recommended)">
    The Two Step opt-in is the recommended option as it is a soft ask. Soft ask means the option is displayed multiple times even if the user does not allow it and this will maximize your reachable users.

    On the web page, users are asked a "soft ask" that gauges user preference and proceeds to show the main browser permission upon permission.

    The Two Step Opt-in section has the following options:

    | Field | Description |
    | --- | --- |
    | Show Push Permission opt-in after | You can configure the time after which to show the push permission opt-in to the user. |
    | Show Desktop Banner / Mobile Nudge after | You can configure the time after which to show the desktop banner/ mobile nudge to the user. |
    | Opt-In re-appear time | In case the users close the opt-in or banner/nudge, you can configure the time after which to show the opt-in again to the user. |
    | Hide Moengage Branding | You can hide the MoEngage logo appearing on the soft ask and hard ask. |
    | Desktop Banner/Mobile Nudge | <ul><li>Banner/ Nudge Icon</li><li>Accessibility text</li><li>Banner/ Nudge Background Color</li><li>Banner/ Nudge Text</li><li>Banner Text Color</li><li>Allow Button</li><li>Dismiss Button</li><li>**Note**: For more information on Accessibility, refer [here](/user-guide/campaigns-and-channels/getting-started/campaign-content/accessible-campaigns-in-moengage).</li></ul> |

    ![](/images/moengage_45038c.png)
  </Tab>
  <Tab title="1-Click Opt-in">
    The 1-Click Opt-in section has the following options:

    | Field | Description |
    | --- | --- |
    | Opt-In re-appear time | In case the users close the opt-in or banner/nudge, you can configure the time after which to show the opt-in again to the user. |
    | Hide Moengage Branding | You can hide the MoEngage logo appearing on the soft ask and hard ask. |
    | Show Push Permission Opt-in after | You can configure the time after which the users will see the Push permission opt-in. |
    | Use Assistive Overlay to increase opt-ins | This will automatically nudge the users to opt-in for your push notifications. You can also edit the assistive overlay text. |
    | Push Opt-in Preview (Click on text to edit) | You can view and edit the text of the Push opt-in nudge. |

    ![](/images/moengage_15f19e.png)

    <Info>
      For Safari

      - The _Show Push Permission Opt-in after_ value is ignored.
      - The opt-in is triggered on the _click_ event as against page load.
    </Info>

    <Warning>
      One-Click Opt-In

      - does not work on newer versions of Microsoft Edge.
      - does not render the Hard-ask completely on Safari when the user clicks any link on the webpage. This is because the opt-in is triggered on the _click_ event as against page load.
    </Warning>
  </Tab>
  <Tab title="Self-Handled Opt-In">
    The Self Handled Opt-in can be enabled only with Web SDK Integration. For information about configuring Self-Handled Opt-In, refer to [Configure Self-Handled Opt-In](https://developers.moengage.com/hc/en-us/articles/360061219351-Configure-Self-Handled-Opt-In).

    ![](/images/moengage_62943e.png)

    <Warning>
      With Microsoft Edge, Hard-ask on page load does not get rendered using the Self-Handled Opt-In. Hence, it is recommended to invoke Soft-ask first.
    </Warning>
  </Tab>
</Tabs>

<Info>
  **Caching**: All of the above settings take a minimum of 24 hours to reflect on your website.
</Info>

<Info>
  For web push to work on Safari for iOS devices, the following is required:

  - Your website must be a Progressive Website App (PWA).
  - A web app manifest file must be created and linked.
  - The MoEngage service worker file should be imported and placed in the root folder.
  - The PWA must be added to the Home Screen.
  - The user should be prompted for permission to receive web push notifications. We recommend a two-step opt-in process.

  For more information, refer to the [Web Push Integration Steps for Safari on iOS and iPadOS](https://developers.moengage.com/hc/en-us/articles/13906923326100).
</Info>

# Device Analytics

Under the **Device analytics** tab, you can configure your tracking preferences to refine your reporting. This includes:

- **App Version Buckets:** Grouping versions into buckets (e.g., Beta vs. Stable) to analyze adoption trends.
- **OEM & Browser Tracking:** Selecting the top 5 device manufacturers or browsers most relevant to your users, while grouping the rest under "Others."

For detailed steps on configuring these preferences, refer to [Device Analytics Configuration](https://help.moengage.com/hc/en-us/articles/43961409133460).

If your system requires IP whitelisting for using Content API or PII tokenization in Push campaigns, refer [here](/user-guide/settings/account/security/ip-whitelisting-in-moengage).

# Frequently Asked Questions

**Can I choose where web push notifications appear on the screen?**\
The position of web push notifications on the screen varies and depends on the browser and operating system. They may appear in a notification center or as a pop-up and cannot be customized.
