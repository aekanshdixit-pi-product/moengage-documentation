---
title: "What Happens to Hard Bounce, Spam, and Unsubscribe Values When a User's Email ID Is Changed?"
description: "Understand how MoEngage handles the Hard Bounce, Spam, and Unsubscribe suppression values when a user's email ID is updated on their profile."
---
 
When the email ID on a user profile is updated, MoEngage resets the **Hard Bounce** value to its default state (`false`), because the bounce status applied to the user's previous email address.The **Spam** and **Unsubscribe** values are not changed by an email-ID update — they are retained on the profile.
 
For more information on these attributes, refer to [Derived User Attributes](/user-guide/data/user-data/derived-user-attributes).
