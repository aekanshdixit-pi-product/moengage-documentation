# Overview

Deep links are specific URLs that can directly point to specific content within a mobile app or web page. Deep linking enables an app or web page to be accessed and launched from a URL, with behavior potentially varying based on the user's device and the context from which the link is accessed. When a user clicks a deep link, they are directed to a specific location within the app or web page rather than opening it to its homepage. Thus, deep linking provides a more personalized user experience and can guide users toward specific actions, enhancing user engagement and conversion rates.

# Advantages

- **Enhanced user experience**: Deep linking eliminates users' need to navigate multiple screens to find what they are looking for, boosting engagement and satisfaction.
- **Promote app installations**: Deep linking can promote app installations. You can create customized links that take users directly to the app store with pre-populated information, making the installation process smoother and more convenient.
- **Retargeting and re-engagement**: Deep links facilitate retargeting strategies by reminding them of products or services they showed interest in. This helps increase user engagement and conversions.
- **Personalized campaigns**: You can create personalized campaigns by leveraging deep linking. By directing users to specific pages related to their interests or personal preferences, you can tailor campaigns to individual users, increasing the chances of conversion.

# Add Deep Linking Domain to Email Campaign

Afterr configuring the deep linking domain, you can add it to your email campaigns. Perform the following steps to add deep link domains while creating email campaigns:

1. Navigate to [Step 2: Content](/user-guide/campaigns-and-channels/email/create/create-an-email-campaign) of your email campaign.
2. In the **Deeplinking domain** list, select the deep linking domain to be used in this email campaign so that MoEngage wraps the deep links with this domain while sending out the email campaign. Note that any deep links marked as default during configuration are auto-populated in this list.  
    <img alt="deeplink 5 new.png" src="/images/moengage_d78327.png" />   
   <Info>
   When you create the email content, adding the parameter *deep\_link=true* to all deep linking URLs included in the email body is necessary. This allows the MoEngage platform to identify these links as deep links. The system then uses the deep linking domain chosen from the **Deeplinking domain** list to wrap them. All remaining URLs in the email body are wrapped with the selected link branding domain. <img alt="deeplink 6.png" src="/images/moengage_c1ac22.png" />
   </Info>
3. Continue to add the required details and publish the campaign. For more information, refer to [Create an Email Campaign](/user-guide/campaigns-and-channels/email/create/create-an-email-campaign).

# View the Deep Linking Domain Used in a Campaign

Perform the following steps to view the deep linking domain used in a published email campaign:

1. On the left navigation menu in the MoEngage dashboard, click **Engage**, and then click **Campaigns**.
2. On the **All campaigns** page, select the required campaign.
3. On the **Campaign analytics and info** page, click the **Info** tab.
4. In the **Sender details** section, you can view the deep linking domain added to that email campaign.

# Configure Your Own Universal Link

Beyond native integrations with third-party providers, MoEngage also supports universal links. Universal links are ideal if you have a "home-grown" solution where your brand's mobile app handles all deep link routing internally, without relying on a provider like Branch or AppsFlyer.

## How to Implement?

For MoEngage to recognize and correctly process your custom universal link, you must append the `universal-link=true` parameter to the URL within your email's HTML content as shown below:

`__PROTECTED_14__

When an email is sent containing a URL marked with`universal-link=true `, MoEngage handles it differently from standard deep links:

- MoEngage does not wrap this URL with a MoEngage deep linking domain. The link remains exactly as you entered it.
- MoEngage appends a single query parameter to the URL: `moeq`. This parameter contains an encrypted string with all the information MoEngage needs to attribute the click, such as the Workspace ID, Campaign ID, and User ID.

For click attribution to work, your mobile application must be configured to handle the incoming link and the `moeq` parameter. Following is the step-by-step flow that occurs when a user clicks the link:

1. The user's mobile app intercepts the universal link request.
2. Your app's internal logic routes the user to the correct in-app screen or content.
3. Your app must initialize the MoEngage SDK.
4. The MoEngage SDK automatically detects the `moeq` parameter from the incoming link, reads its value, and communicates with the MoEngage Email Tracking API.
5. The click is successfully attributed, and the campaign statistics are updated.
