# Overview

<Tip>
Before starting your journey with content blocks, you must create a content block. For more information, refer to [Content Blocks](https://help.moengage.com/hc/en-us/articles/4414768369172).
</Tip>

Content blocks allow marketers to reuse the same content across multiple campaigns.

# Personalization Attributes Supported in Content Blocks

You can personalize the content block using the following personalization attributes:

| Personalization Type | Supported? | Additional Information |
| --- | --- | --- |
| User attribute | Yes | - |
| Event attribute | Yes | You can use Jinja to personalize content based on event attributes. Ensure the event used for personalization is selected in the trigger criteria of the event-triggered campaign. This ensures the successful personalization of the campaign content using a content block. |
| Product sets | No | - |
| Content API | No | - |
| Content block | Yes | - |
| Auxiliary data | No | - |
| Campaign attribute | Yes | You can use Jinja to personalize content based on the campaign attributes.  **Note**: This is currently available only for Email. |
| Business event attribute | Yes | You can use Jinja to personalize content based on the business event attributes. Ensure you select the same business event whose attribute you used to personalize the content block.  **Note**: This is currently available only for Email, SMS, and Push. |

<Info>
- Content blocks are not supported in the Inbox preview in Email campaigns.
- URLs that are added in content blocks and used in SMS cannot be tracked.
- You cannot archive content blocks used in active event-triggered and periodic campaigns because there is no option to stop these active campaigns. You can archive these content blocks only after the expiry of such campaigns.
</Info>

# Ways to Use Content Blocks

<Info>
When you create a personalization condition or logic using the plain text editor, it appears as a chip. To know more about Chip Actions, refer [here](https://help.moengage.com/hc/en-us/articles/39719238781716-Dynamic-Content-Personalization#h_01JMKXT2S4GE9PS904K9KQS1YH).
</Info>

You can use your content blocks in two ways:

- [Select the desired content block](#h_01J87F9S3JAHD21TFA3JDDWBYA)
- [Insert the block label of your content block directly](#h_01J87NRWH83JD7CX5TSQ6WYYVE)

## Select the Desired Content Block

While Creating a Campaign While Creating an Alert

1. While creating content, type @.
2. On the Personalization pop-up, click the **Content blocks** tab.  
    <img alt="20.png" src="/images/moengage_62254e.png" />
3. In the **Select content block** list, click the desired content block.
4. Turn the **Insert only the content of the content block** toggle on to insert only the content of the content block.  
    <img alt="Screenshot_2022-01-04_at_3.44.38_PM.png"  src="/images/moengage_d20d8b.png"  />   
   This means that only the content will be inserted while detaching itself from the existing content block, and no future updates will be reflected.
5. Click **Done** to insert the content block.

1. While creating content, click **Text personalization**.
2. On the Inform personalization pop-up window, click the **Content block** tab.  
    <img alt="21.png" src="/images/moengage_62be40.png" />
3. In the **Select content block** list, click the desired content block.  
    <img alt="22.png" src="/images/moengage_327116.png" />
4. Turn the **Insert only the content of the content block** toggle on to insert only the content of the content block.  
    <img alt="23.png" src="/images/moengage_22df45.png" />   
   This means that only the content will be inserted while detaching itself from the existing content block, and no future updates will be reflected.
5. Click **Copy & Close** to copy the personalized attribute to the clipboard.
6. Paste the copied value in the message editor using keyboard shortcuts.

## Insert the Block Label of Your Content Block Directly

1. Copy the *Block label* from your Content Block page.  
    <img alt="contentblockkkk.png" src="/images/moengage_7c4e37.png" />
2. Paste the *Block label* into the campaign/template/another content block.
