<Info>
RCS Campaign Info is an add-on feature. Contact your MoEngage CSM (customer success manager) or the [Support team](/user-guide/contact-support/contact-support/raise-a-support-ticket-through-moengage-dashboard) to enable it for your account.
</Info>

Campaign Info displays the details of your RCS campaigns after they are published. You can view this information on the **Info** tab. Perform the following steps to access the **Info** tab:

1. On the left navigation menu in the MoEngage dashboard, click **Engage**, and then click **Campaigns**.
2. On the **All campaigns** page, click the name of the campaign you want to view.
3. On the **Campaign info and analytics** page, click the **Info** tab. The Info tab displays the following details:

Timeline Audience Trigger criteria Scheduling Conversion Goals Preview and Content

This section is available for one-time and periodic campaigns. The timeline shows the different stages of a campaign's publishing process and its current status. This provides complete transparency and insight into the lifecycle of your campaign. It allows you to monitor the campaign's progress from audience selection to final delivery, which is essential for troubleshooting, optimization, and validating performance. Note that the time displayed is based on the App's Timezone. This section contains information about the following:

- **Segmentation query**: Contains information about when the segmentation query was started and completed.
- **Campaign ready**: Contains information about when the campaign was ready to be sent from MoEngage after internal processing. For ASAP campaigns, this is the campaign sending time.
- **First sent**: Contains information about when the first batch of messages was sent from MoEngage and when the first message was sent.
- **Last sent**: Contains information about when the last message was sent for the campaign.

The **Audience** tab contains information about the users who were targeted for the campaign. These users are defined in Step 1 of campaign creation. This tab is useful because it provides a complete and transparent record of a campaign's targeting logic. It serves as a source for validating who a campaign was sent to, which is essential for accurate analysis and compliance. This tab includes the following details:

- ****Segment****: A segment is a group of users defined by specific properties, values, or rules.     
  The **Segment** detail displays the segmentation criteria used to create this campaign. Hence, whenever this segment is executed, all users who fulfill the criteria become part of it and receive the campaign. For more information, refer to [Create Segments](/user-guide/segment/segment-operations/create-segments).
- **Control group**: A [control group](/user-guide/campaigns-and-channels/getting-started/introduction/control-groups) is a set of users added to an exclusion list. These users will not receive the marketing campaign. Control groups help you measure campaign effectiveness by comparing them with your target audience. There are two types: Global Control Group and Campaign Control Group.  
  The **Control group** detail displays whether a control group was enabled during campaign creation.
- **Filter by**: This detail displays the segmentation filters and exclusions applied during campaign creation. Users can be segmented based on all users, property, affinity, behavior, or custom segments that employ complex queries with AND/OR conditions. For more information, refer to [Create Segments](/user-guide/segment/segment-operations/create-segments).

The **Trigger criteria** tab displays the trigger conditions added while creating this campaign. This tab provides a transparent and verifiable record of the specific conditions that launch a campaign. The image below shows an Event Triggered campaign that has an *App/Site Opened* event-based trigger added to it.   
**Note**: This section is available only for [Event Triggered](/user-guide/campaigns-and-channels/getting-started/triggers/event/create-an-event-triggered-campaign) and [Business Event Triggered](/user-guide/campaigns-and-channels/getting-started/triggers/business-event/business-event-triggered-campaigns) campaigns.

The **Scheduling** tab displays information about the campaign’s delivery schedule, delivery controls, and advanced controls configured in Step 3 of campaign creation. This tab provides a complete and verifiable record of a campaign’s timing and delivery controls. The delivery schedule helps you determine when campaigns are sent to your users and how often. This includes the following details:

- **Time selection**: Displays the time when the campaign was sent.
- **Start on**: Displays the date and time when the campaign sending began.
- **Time zone**: Displays the time zone for the campaign's delivery.
- **Best time to send**: Displays the recommended time slot for sending the campaign. For more information, refer to [Best time to send](/user-guide/intelligence/timing/best-time-to-send).
- **Ends on**: Displays the date on which the campaign ends.
- **Repeat:** Displays the periodicity of the campaign. For more information, refer to [Schedule and Goals](/user-guide/campaigns-and-channels/sms-mms-and-rcs/rcs/create-an-rcs-campaign).

Delivery controls help limit the number of campaigns sent to the users using the Frequency capping feature. The delivery control includes the following details:

- **Frequency capping**: This feature helps control the number of messages sent to a user in a day, and the **Frequency capping** detail displays whether this feature is enabled for this campaign. For more information, refer to [Frequency capping](/user-guide/settings/channels/delivery-controls/frequency-capping).
- **Count for FC:** This detail displays whether the **Count for the frequency capping** toggle is turned on. If yes, this campaign is counted for frequency capping.
- **Request limit**: This detail displays the throttle limit of this campaign. Throttling allows you to control the sending rate of a campaign. For more information, refer to [Throttling](/user-guide/settings/channels/delivery-controls/throttling).

The **Conversion goals** tab displays the conversion goals and revenue metrics tracked for the campaign. The primary conversion goal is displayed with a '**P**' icon beside the goal name.

- **Attribution window**: This displays the time or window that is calculated after a user receives or clicks the campaign, for which you want to attribute the conversion goals to the campaign.

The **Control goals** display the following information:

- **Goal Name**: Displays the name of the campaign goal set for the campaign.
- **Event Name**: Displays the event name based on which the campaign goal is configured.
- **Attribute Name**: Displays the attribute name based on which the campaign goal is configured.
- **Attribute Value**: Displays the attribute value based on which the campaign goal is configured.

The Revenue Performace Info Value displays the following information:

- **Event Name**: Displays the event name based on which the campaign revenue information is configured.
- **Attribute Name**: Displays the attribute name based on which the campaign revenue information is configured.
- **Currency**: Displays the currency used to calculate the campaign revenue.

<img alt="Screenshot 2025-04-08 at 6.02.29 PM.png" src="/images/moengage_3029ee.png" />

The **Preview and Content** section contains information about the sender, the message being sent, and a preview of how the message would appear on the user's mobile device. You can view all the different variations and locale combinations configured for the campaign here.

The **Content** section displays the following information about the RCS-only/ RCS with SMS as a Fallback campaign:  

- **Connector**: The configured RCS connector (Service provider).
- **Sender**: The Sender name you provided while configuring the RCS service provider.
- **Template Name**: The name of the template you provided while configuring RCS templates.
- **Template Type**: The type of template. Template type can be **Simple Text** or **Rich Card**.
- **Title Parameters**: The title parameters in the campaign content.
- **Description Parameters**: The description parameters in the campaign content.
- **Suggestion Parameters**: The suggestion parameters in the campaign content.
- **URL Tracking**: URLs in the RCS message used to track RCS clicks.
