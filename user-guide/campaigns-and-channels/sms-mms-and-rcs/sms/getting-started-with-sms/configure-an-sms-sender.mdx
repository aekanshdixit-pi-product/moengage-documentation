---
title: "Configure an SMS Sender"
---

Configuring an SMS sender is important because it's the authorized entity that MoEngage uses to transmit messages, making it a mandatory prerequisite for running any RCS campaigns.

You have to configure senders for integrated or custom connectors(service providers). By having separate threads, you can configure multiple senders for an SMS service provider and streamline your SMS communications for different brands or types. Moreover, you can choose different senders for different SMS campaigns during the creation of SMS campaigns.

SMS senders can be added and managed in the **Sender configuration** tab of SMS, MMS & RCS settings. You can access the **Sender configuration** tab using this path: **Settings** \> **Channels** \> **SMS, MMS & RCS** \> **Sender configuration**.

![CONFIGURE sms.png](/images/moengage_ec5794.png)

# Configure Your Sender

Perform the following steps to add an SMS sender:

1. On the **SMS & MMS** tab, click **Add SMS Sender**. The Add sender page is displayed.

   ![CONFIGURE sms 2.png](/images/moengage_bc3510.png)
2. In step 1, **Service Provider**, select your service provider. You can choose from integrated service providers or add a custom service provider. For more information about adding custom service providers, refer to [Configure a Custom Service Provider](/user-guide/campaigns-and-channels/sms-mms-and-rcs/sms/getting-started-with-sms/configure-custom-connector-for-sms).

   ![Screenshot 2025-09-16 at 6.10.59 PM.png](/images/moengage_49a740.png)
3. In step 2, **Sender Details**, add the sender details of the selected sender. The following are some of the common sender detail fields for integrated service providers:

   | Field       | Description                                                                                                                                                                                                                                                                                                                                         |
   | ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | Sender type | The Sender type denotes whether the sender is used for promotional or transactional messaging. Select **Promotional** or **Transactional** based on your use case.                                                                                                                                                                                  |
   | Sender name | The Sender name is the name that identifies the sender being configured. This name will be available in the Sender selection dropdown while creating campaigns. The default sender configured will be used for sending the SMS campaigns. However, this default setting can be overridden at the campaign level during step 2 of campaign creation. |
   | Sender ID   | The Sender ID is the name or number that is displayed as the sender of the SMS message.                                                                                                                                                                                                                                                             |

   Note that depending on the sender being configured, the fields in the UI vary. For example, some senders may require a username and password, while others may require API endpoint URLs. For more information, refer to [SMS Providers](https://partners.moengage.com/hc/en-us/categories/4701102365972-SMS-Providers) for vendor-specific configuration details.
4. Copy the **Delivery tracking URL** available on the dashboard and paste it into the callback webhook settings of your service provider.
5. Click **Send a test SMS** to test your configuration.
6. Click **Save**.
7. After adding the sender, it will be displayed under the **Sender configuration** \> **SMS & MMS** tab.
8. Click the ellipsis icon ( 

   <Frame>
     ![Ellipsis](/images/ellipsis.png)
   </Frame>
    ) to **Mark as default**, **Edit,** or **Delete** the sender. 

   ![Screenshot 2025-09-17 at 11.26.34 AM.png](/images/moengage_97bd16.png)
   - When a sender is marked as default, this sender is used to send all SMS campaigns, unless a different sender is selected at the campaign level.
   - When you delete a sender, any active campaigns associated with this sender will be send using the default sender. Note that you cannot delete a default sender. To delete a default sender, mark another sender as the default first. This will enable the delete option for the sender you want to delete. You can now proceed to delete the sender.

<Warning>
  - In the **Sender ID** field, ensure that you add the **Header ID** approved under the DLT platform. More details on DLT are captured [here](https://help.moengage.com/hc/en-us/articles/360057032912).
  - If you try to send an SMS via an unapproved header, your SMS is rejected and is not delivered to the users.
</Warning>

# Dynamic Sender Routing

<Info>
  This feature is applicable only for Inform alerts.
</Info>

_Dynamic Sender Routing_ allows you to define custom rules that dynamically assign senders for your SMS alerts. By defining custom routing rules based on user attributes or API attributes, you can:

- Optimize your messaging for better deliverability.
- Manage sending costs.
- Ensure compliance with local regulations.

For more information, refer to [Dynamic Sender Routing](/user-guide/inform/create/dynamic-sender-routing).
