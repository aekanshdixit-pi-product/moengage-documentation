# Introduction

An item catalog is a prerequisite for creating and using recommendations. It is a comprehensive list of all your business offerings with their relevant details, such as products, services, flights, article properties, restaurants, dishes, and social media feeds.

# Catalog

With the Catalog feature, you can create and manage item catalogs. You can connect your item catalog to MoEngage and use the item information in your campaigns.

![Catalogs overview](/images/moengage_0ce5b8.png)

## Create Catalog

To create an item catalog, follow these steps:

1. On the sidebar menu in MoEngage, hover over the **Content** menu item <img alt="Content menu icon" src="/images/moengage_1da6a1.png" style={{ display: "inline-block" }} /> . The **Content** menu appears.
2. Click **Catalogs**. <img src="/images/moengage_f8db06.png" /> The Catalogs page appears.
3. Click **\+ Create Catalog**. The **Create catalog** dialog box appears.
4. In the **Catalog name** box, type a name for the catalog. <img src="/images/moengage_7b3bf3.png" alt="Create catalog dialog box" />
5. Click **Submit**.

After you create a catalog, an option to add a feed appears. Feeds serve as data sources that provide item lists and keep them updated.

![](/images/moengage_7cd387.png)

## Feeds

Feeds act as the data source for item information in the catalog. You can use feeds to upload, update, or sync item information. Each feed contains a set of items from a source.

<Info>
  MoEngage supports both a single feed for all items and multiple feeds for different purposes.
</Info>

### Types of Feeds

Feeds provide detailed attributes of the items, such as ID, title, link, and image link. The following are the feed options:

1. **Manual Upload:** With Manual Upload, you can upload a CSV file containing item listings. Use this method when you want to perform a one-time upload of your item information.
2. **URL Upload:** With URL Upload, you can connect your external catalog to MoEngage by providing the catalog source URL. This keeps your item attribute information synchronized and up to date. You can schedule daily, weekly, or hourly sync intervals to keep your item information current in MoEngage. Use this option for catalogs that require frequent or near real-time updates.
   <Info>
     **Note** If you prefer not to make your catalog feed URL public, import it using SFTP-based ingestion. Whitelist the following MoEngage IPs:
     - DC-01: 52.1.205.204
     - DC-02: 18.195.110.23, 3.77.101.97
     - DC-03: 3.6.251.95
     - DC-04: 18.217.73.1
   </Info>

### Add Feeds to a Catalog

To add feeds to catalogs, follow these steps:

1. On the sidebar menu in MoEngage, hover over the **Content** menu item <img alt="Content menu icon" src="/images/moengage_e053c0.png" style={{ display: "inline-block" }} /> . The **Content** menu appears.
2. Click **Catalogs**. <img src="/images/moengage_502ea4.png" alt="Catalogs page navigation" /> The Catalogs page appears.
3. Click the ellipsis icon for the desired catalog and then click **Add feed**. <img src="/images/moengage_512c31.png" alt="Add feed option in catalog menu" />
4. The **Add feed** page appears.
5. In the **Feed name** box, type a name for the feed (maximum 100 characters).
6. From the **Currency** dropdown, select the currency for your catalog items.
7. Under **Select upload type**, choose one of the following options:
   1. **Manual upload:** Select **Manual upload** to upload a CSV file from your device. In the **Add file** section, drag and drop your file or click **upload from computer** to select it from your computer. <img src="/images/catalog.png" alt="Catalog" />
   2. **URL upload:** Select **URL upload** to import catalog items from an external feed URL. Complete the following fields:
      - **File type:** Select the format of your feed file: CSV, TSV, or XML.
      - **Feed URL:** Enter the URL of your catalog feed. MoEngage supports HTTP, HTTPS, FTP, and SFTP URLs.
      - **Username** and **Password** (optional): Enter your credentials if your feed URL requires authentication.
      - **Upload time:** Select **Schedule** to sync the feed on a recurring basis (daily, weekly, or hourly), or select **One-time upload** to perform a single import. <img src="/images/uploadfeed.png" alt="Uploadfeed" />
   <Info>
     MoEngage expects the following mandatory attributes with every feed:
     - id
     - title
     - link
     - image\_link Click this [sample CSV](https://cdn.moengage.com/static-files/Sample_Catalog.csv) file for a catalog that supports feeds.
   </Info>
8. Review the **File preview** section, which displays the top 5 rows of your feed for validation. Based on the preview results, do one of the following:
   - **If errors are found:** Resolve the issues in your file and re-upload it. Errors block you from proceeding to the next step. <img src="/images/catalog3-2.png" alt="Catalog3 2" />
   - **If warnings are found:** Review the warnings and proceed to the next step. Warnings do not block the upload. Common warnings include:
     - The feed has fewer than five rows of data.
     - The current feed contains more columns than the previous feed.
     - Columns from the previous feed are missing in the current feed. <img src="/images/error.png" alt="Error" />
9. Click **Next**. \
   The **Mappings** page appears.
10. In the **Map columns** section, set the data type for each column in your feed. You can search for a column by name using the search bar. Available data types are **String**, **Numeric**, **Date Time**, **Boolean**, and **Geopoint**. <img src="/images/map.png" alt="Map" />
    <Note>
      - Price and stock/inventory fields must contain numeric values for price drop and back-in-stock alerts to function correctly.
      - By default, MoEngage automatically selects a data type for each column based on the attributes in your feed. You can still change these values from the drop-down list.
        - The data types for the required attributes (id, title, link, and image\_link) are fixed and cannot be changed.
        - If you select a data type that does not match the previously stored type for a column, the system displays a warning. You can still submit, but the mismatch may cause errors during processing.
        - After a data type is set and a feed is processed, it is locked for all future feeds in that catalog, even if the original feed is deleted.
    </Note>
11. Click **Submit**. \
    MoEngage processes your feed.

### Feed Upload States

After you add a feed to a catalog, the feed upload action can have the following states:

- **Success**: The feed uploaded successfully and is available for use in the catalog.
- **Pending**: MoEngage is processing the feed.
- **Processed with Error**: Some items in the feed contain errors. MoEngage partially uploads the feed, skipping items with errors.
- **Error**: The feed upload failed. MoEngage did not upload the feed.

After the feed updates successfully, a summary catalog report appears at the top of the catalog page. This report includes:

- **Total items:** The current total number of items in the catalog.
- **Total feeds:** The number of data source feeds configured to populate and update the catalog.
- **Last updated on:** The date and time of the most recent catalog update. For catalogs with multiple feeds, this reflects the most recent feed update time.
- **Last updated status:** The status of the most recent catalog update. For catalogs with multiple feeds, this shows the status of the most recently processed feed.
- **Next update on:** The next scheduled update for the catalog. For catalogs with multiple feeds, the last and next updates can come from different feeds.

## Items

After successfully uploading the feed(s) to the catalog, navigate to the **Items** tab to view a comprehensive list of items. The **Items** tab shows item details for up to 200 sample items. To find a specific item, use the search bar in the **Items** tab.

![](/images/moengage_f93f69.png)

### Search Items

The **Items** tab includes a search bar that helps you find items quickly. To search for items in your catalog:

1. In the **Items** tab, find the search bar.
2. Type the item's unique ID, title, or description in the search bar.
3. Press Enter on your keyboard.

The list filters immediately to show only matching items. The search checks unique item IDs, titles, and descriptions.

![](/images/moengage_f27862.png)

To view more details, click the item in the list.

![](/images/moengage_2ada31.png)

# Catalog Update Alerts

MoEngage tracks catalog updates and notifies you about their status. It sends notifications for completed updates and error messages for failures. Use these error messages to maintain data quality and resolve issues. Catalog update alerts trigger when feed processing succeeds or fails:

- When MoEngage processes a feed successfully, it sends an email with completion statistics, including the count of newly added, updated, or deleted items. If some items could not be processed, the email also includes a report listing the unprocessed items and the reasons.
- If feed processing fails, MoEngage sends error messages that explain what went wrong so you can fix the issue.

You can subscribe to the catalog update alerts from the **Manage Alerts** option on the catalog page.

![Manage Alerts option on the catalog page](/images/moengage_58b91c.png)

## Manage Alerts

1. On the sidebar menu in MoEngage, hover over the **Content** menu item <img alt="Content menu icon" src="/images/moengage_e053c0.png" style={{ display: "inline-block" }} /> . The **Content** menu appears.
2. Click **Catalogs**. <img src="/images/moengage_502ea4.png" alt="Catalogs page navigation" /> The Catalogs page appears.
3. Click the catalog for which you want to configure update alerts.
4. On the catalog page, click **Manage alerts** in the top section. The Catalog update alerts pop-up window appears.
5. In the **Email address** box, type the email addresses to subscribe to update alerts. Press Enter after adding each address. You can add a maximum of 50 email addresses. <img src="/images/moengage_d4e0e0.png" />
6. Under **Notify on**, select the checkboxes for the alert types you want to receive.
7. Click **Save** to save the subscription preferences.

After you click **Save**, MoEngage sends email alerts to subscribed addresses whenever it processes a feed.

<Info>
  When you create a new catalog, MoEngage automatically subscribes your email address to catalog update alerts. To update your preferences, go to **Manage alerts** on the catalog page.
</Info>

# FAQs

<Accordion title="What is the maximum number of items, catalogs, and feeds an account can have?">
  An account can have a maximum of one million items for a monthly tracked user (MTU) of less than one million or an account can have a maximum of five million items for an MTU of less than five million. An account can have a maximum of 30 catalogs. A catalog can have a maximum of 200 feeds.

  <Note>
    If your catalog size exceeds 5 million items, contact your account manager.
  </Note>
</Accordion>

<Accordion title="Which feed file format types are supported?">
  MoEngage supports feeds in **CSV, TSV, and XML(RSS 2.0-compliant)** formats. The character encoding must be UTF-8.
</Accordion>

<Accordion title="What is the maximum number of columns a catalog can have?">
  MoEngage supports feeds having a maximum of 50 columns.
</Accordion>

<Accordion title="What is the maximum size of a feed that can be imported with Scheduled URL?">
  You can upload a file size of up to 2.5 GB for each scheduled feed either as a one-time scheduled upload or with a daily or weekly refresh frequency. However, for an hourly refresh, MoEngage supports feed files up to 200 MB.
</Accordion>

<Accordion title="What is the maximum size of a feed that can be uploaded manually?">
  MoEngage supports manual feed upload of file sizes up to 64MB.
</Accordion>

<Accordion title="What happens if my feed URL server is down?">
  MoEngage attempts to fetch the feed every 10 minutes and stops after five failed attempts.
</Accordion>

<Accordion title="Is there any limitation with respect to column names?">
  Column names cannot contain a period (.) or dot operator.
</Accordion>

<Accordion title="Which types of feed URLs are supported?">
  MoEngage supports feed upload via **HTTP**, **HTTPS,** **FTP,** or **SFTP** URLs.
</Accordion>

<Accordion title="Can we use Google Sheets to update Feeds?">
  Yes, you can use Google Sheets to update feeds.

  1. Select Feed update through the Scheduled URL option.
  2. In Google Sheets, set the sharing settings to public.
  3. Add the Google Sheet with the URL params as **export?format=csv**. For example: `https://docs.google.com/spreadsheets/d/Google_Sheet_ID/export?format=csv`.
</Accordion>

<Accordion title="How can I import a catalog feed into MoEngage without making the feed URL public?">
  Whitelist the following MoEngage IPs and import your feed via SFTP-based ingestion:

  - DC-01: 52.1.205.204
  - DC-02: 18.195.110.23, 3.77.101.97
  - DC-03: 3.6.251.95
  - DC-04: 18.217.73.1
</Accordion>
