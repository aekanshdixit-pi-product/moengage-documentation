# Introducing Campaign APIs

MoEngage introduced Campaign APIs, which allow you to:

- [Create a campaign](https://developers.moengage.com/hc/en-us/articles/27896715162516-Create-Campaigns) (one-time, periodic, event-triggered, and business event-triggered)
- [Update a campaign](https://developers.moengage.com/hc/en-us/articles/27896779501332-Update-Campaigns)
- [Get campaign details](https://developers.moengage.com/hc/en-us/articles/27896777540756-Get-Campaign-Details) (search for campaigns using multiple filters)
- [Change the status of a campaign](https://developers.moengage.com/hc/en-us/articles/27896790232980-Change-Campaign-Status) (update the status of the campaign)
- [Get all children of a periodic or triggered campaign](https://developers.moengage.com/hc/en-us/articles/27896783707284-Get-Child-Campaigns-of-Periodic-and-Triggered-Campaigns)
- [Test a campaign](https://developers.moengage.com/hc/en-us/articles/27896794400788-Test-a-Campaign)

For now, this is supported only for the Email channel. For more information, refer to [Campaign APIs](https://developers.moengage.com/hc/en-us/sections/27896650217620-Campaign-API).

# Introducing My Studio

MoEngage introduces you to My Studio - your analytics folder!

It's secure, accessible only to you, and saves your queries. Now, you can effortlessly save your analyses and retrieve queries with just one click.

For more information, refer to [My Studio](https://help.moengage.com/hc/en-us/articles/27854721709716-My-Studio).

# Locales for On-Site Messages

Users can now utilize Locale to target audiences in different languages, geographies, devices, and more within a single campaign. No more complicated workarounds, no more hassle.

## How Does It Work?

To send tailored messages, you can include the desired locale in the API call. You can also harness the power of user profiles for intelligent locale selection.

For more information, refer to [Using Locales in On-site Messages](https://help.moengage.com/hc/en-us/articles/28099217510932-Using-Locales-in-On-site-Messages).

# Custom Domains in Emails

MoEngage now allows you to configure a custom domain instead of the standard MoEngage Domain for messages. This feature enables you to use your custom domain in the URLs sent in the message body of SMS, WhatsApp, or Email.

For more information, refer to [Configure Custom Domain](https://help.moengage.com/hc/en-us/articles/13706614569236).

# Enhanced Alert Manager

MoEngage changed how the alert manager is positioned for our different customer plans:

- The **Growth** plan customer is only entitled to system alerts, which MoEngage creates out of the box. System alerts are basically any information that MoEngage wants to send to clients proactively.
- The **Enterprise** plan customer is entitled to both system and custom alerts. These customers can set alerts based on their business threshold to monitor anomalies.

The following are a few UI changes to the Alert Management dashboard:

- **Delivery channel** -\> **Send Alert On**
- **Alert type** -\> **Create Alerts On**
- **Alert the user when** -\> **Send alert when**

For more information, refer to [Alert Management](https://help.moengage.com/hc/en-us/articles/19320181606932) and [Create Alert](https://help.moengage.com/hc/en-us/articles/19094285198484).
