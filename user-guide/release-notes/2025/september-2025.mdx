# Portfolio

The *Portfolio* feature enables you to manage multiple applications and websites as individual *projects* within a single, unified MoEngage workspace.

This feature consolidates user data from all your digital properties into a single customer profile. This unified view allows you to implement cross-brand strategies, streamline campaign management, and analyze performance across your entire portfolio.

Portfolio helps you:

- Consolidate user data from all apps and websites into a single customer profile.
- Build segments and trigger campaigns based on user behavior across your entire portfolio.
- Analyze performance and manage campaigns for all properties from a single dashboard.
- Control communication frequency across all brands to prevent message fatigue.

For more information, refer to the following articles:

- [Portfolio](https://help.moengage.com/hc/en-us/articles/40394603054100)
- [Segmentation in a Multi-Project Portfolio](https://help.moengage.com/hc/en-us/articles/40431846748820)

# Introducing Pre-Built HTML Nudge Templates

MoEngage has introduced a set of pre-built HTML Nudge templates for your In-app Messaging campaigns.

Nudges are non-intrusive messages that appear at the top or bottom of the screen, allowing you to communicate with users without interrupting their app experience.

These templates are fully customizable directly within the In-App HTML editor, enabling you to launch beautiful, interactive campaigns (such as feedback surveys, promotional alerts, and urgent announcements) in minutes. This update helps you save time and boost user interaction without needing to build from scratch.

For more information, refer to [HTML Nudge Templates](https://help.moengage.com/hc/en-us/articles/34206127791380).

# No-Code Gamification Templates for In-App Messages

MoEngage has introduced a suite of no-code gamification templates for your In-app Messaging campaigns, designed to make your app more interactive and engaging than ever before.

You can now effortlessly create and launch a gamified experience using **Spin the Wheel**. This template is fully customizable in a user-friendly editor, allowing you to design compelling campaigns without writing any code or requiring an app SDK update.

This powerful new feature helps you boost user engagement, distribute rewards, generate leads, and create a sense of urgency to drive conversions, all while providing a fun and memorable user experience.

For more information, refer to [No-code Gamification Templates](https://help.moengage.com/hc/en-us/articles/41051898902036-No-code-Gamification-Templates).

# Total Transparency into Your SMS Campaign Setup

The *GET SMS Campaign Details* API provides programmatic access to retrieve the setup and content details for both standalone campaigns and campaigns within Flows.

You can use this API to integrate campaign data with your internal tools, automating governance and reducing manual oversight. This improves compliance checking, centralizes creative assets, and streamlines approval workflows.

The *GET SMS Campaign Details* API helps you:

- Fetch a specific campaign or a filtered list of campaigns.
- Automate content audits for brand and legal compliance.
- Retrieve SMS copy to a central repository for archiving.
- Integrate campaign content with internal approval workflows.

For more information, refer to [Get SMS Campaign Details](https://developers.moengage.com/hc/en-us/articles/40800964029076-Get-SMS-Campaign-Details).

# User Lookup

The *User Lookup* feature is a diagnostic tool within campaign analytics that provides a transparent view into push notification delivery, enabling you to investigate a specific user's journey within a campaign.

This feature empowers you to diagnose specific delivery scenarios with precision. By providing a detailed timeline for any user, you can pinpoint the exact reason for non-delivery or validate audience segmentation. This gives you confidence in your campaign execution and ensures critical messages reach their intended recipients.

*User Lookup* feature helps you:

- Investigate precisely why an individual user did not receive an expected notification.
- Validate why a specific user was included in a campaign's audience to confirm segmentation logic.
- Analyze a detailed, step-by-step delivery timeline from eligibility checks to the final click.

For more information, refer to [User Lookup](https://help.moengage.com/hc/en-us/articles/39219416230676-User-Lookup).

# Dynamic Sender Routing

Dynamic Sender Routing is a customised routing system that automatically selects the most appropriate sender for each API request. By defining custom routing rules based on user attributes or API attributes, you can optimize your messaging for improved deliverability, manage sending costs more effectively, and ensure compliance with local regulations.

For more information, refer to [Dynamic Sender Routing](https://help.moengage.com/hc/en-us/articles/39786984813076-Dynamic-Sender-Routing).

# Teams in Analytics

MoEngage has introduced the Teams feature into the MoEngage Analytics suite.

You can now select a team in Analytics modules like Behavior, Funnels, and Retention, as well as in Custom Dashboards and My Studio. This action automatically filters all data and reports, showing you only the analytics relevant to the users defined in that specific team.

For more information, refer to [Teams 2.0 in MoEngage](https://help.moengage.com/hc/en-us/articles/30310276466964-Teams-2-0-in-MoEngage#h_01J8ABCDE1FGHIJK2LMNOPQRS).

# Improved Table View for Split-By Analytics

MoEngage has released a user experience enhancement to improve the readability of your analytics tables.

Previously, when performing an analysis with multiple "split-by" attributes, the table view would combine all values into a single, comma-separated column. We've fixed this. Now, each attribute you split by gets its own dedicated column, making the data clean, organized, and easy to read.

This update eliminates messy, crunched data, allowing you to analyze results more efficiently without needing to export your data. This improvement is now available in Users, Behaviour, Funnels, Retention, and Session Source analytics.

# Value Suggestions for Object Data Types

MoEngage has introduced support for *Value Suggestions* for object data types.

While Value Suggestions have always helped you by suggesting string and numeric values, we have now extended this capability to your complex object attributes. When creating a segment, you can now search for specific values within an object.

For more information, refer to [Value Suggestions](https://help.moengage.com/hc/en-us/articles/360001062103-Value-Suggestions).
