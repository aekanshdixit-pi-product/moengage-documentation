# Actionable Analytics

*Actionable Analytics* has been updated with new capabilities on chart views. You can now click specific data points, such as a bar or a point on a line chart, to access an expanded action menu.

New options include **View sample users** for immediate data validation and **Export users** for direct data retrieval, alongside existing capabilities such as creating segments and campaigns.

For more information, refer to [Actionable Analytics](/user-guide/analyze/analytics/actionable-analytics).

# Computed Traits

Computed Traits is a new feature that enables you to create dynamic user attributes by applying calculations to existing event data. This capability allows you to define and track complex metrics, such as Lifetime Value (LTV), total orders, or content preferences directly within the platform, eliminating the need for manual data uploads or additional developer resources.

Key Highlights:

- **Flexible computation methods**: Generate traits using standard logic (Count, Aggregation, First/Last Value) or use SQL for advanced, custom calculations.
- **Automated updates**: Schedule computations to run one-time or periodically (weekly or monthly) to ensure user data remains current based on real-time behavior.
- **Seamless integration**: Once created, Computed Traits function as standard user attributes and can be immediately used in segmentation, campaign personalization, and workflows.

For more information, refer to [Computed Traits](/user-guide/data/computed-traits/computed-traits-overview).

# Unified AI Capabilities under Merlin AI

MoEngage has now unified all predictive, generative, and decisioning AI capabilities under a single suite: Merlin AI. This update provides a consistent product identity across the platform, replacing the previous Sherpa AI branding while retaining all existing functionality.

Key Highlights:

- **Predictive insights**. You can access predictive segments, product recommendations, and best-time-to-send features to optimize user paths and improve overall user experience.
- **Generative building tools**. Utilize generative agents, such as Copywriter, Designer, Segment Assist, and Merlin AI Studio, to create campaigns more efficiently.
- **Decisioning agents**. Implement the new offer and campaign decisioning agents to automate strategic campaign choices.
- **Branding updates**. You can view the new Merlin AI branding across the dashboard, product documentation, and the MoEngage website.

# Merlin AI Designer: Omni-Channel Image Support

Merlin AI Designer now includes omni-channel image generation to help you create visuals beyond the previous 21:9 aspect ratio. You can now generate perfectly sized visuals for major marketing and advertisement channels directly within Merlin AI Studio.

Key Highlights:

- **Diverse aspect ratios**: MoEngage added support for 16:9, 4:3, 1:1, 3:4, and 9:16 ratios to ensure your visuals fit seamlessly across email, on-site messages, and social media feeds.
- **Social optimization**: Utilize optimized formats specifically designed for high-performing channels, such as TikTok, Reels, Instagram, and LinkedIn.
- **Channel versatility**: MoEngage retained the 21:9 format for high-impact push notifications while unlocking new portrait and square formats for mobile-first content.

For more information, refer to [Merlin AI Studio](/user-guide/intelligence/merlin-ai/merlin-ai-studio).

# Enhancements to Merlin AI Jinja Assistant

MoEngage has now enhanced Merlin AI Jinja Assistant to simplify the creation of complex logic and personalized content. This enhancement allows you to generate precise Jinja queries for common marketing tasks, such as calculating date differences or encoding URLs, directly within your campaign workflow.

Key Highlights:

- **Advanced date and time logic**: You can now generate efficient Jinja code for date, day, and time formats, including time zone conversions and date differences.
- **Enhanced security and encoding**: The assistant supports SHA 256 encryption, Base64 encoding, and URL encoding to ensure your data is handled securely and correctly.
- **Refined attribute handling**:Use the *in* and *is* operators to query array-type attributes more effectively.
- **Custom formatting and fallbacks**:MoEngagesupports strikethrough text, Indian-style number separators, and specific fallback values for MOE\_NOT\_SEND handling.

For more information, refer to [Merlin AI](https://help.moengage.com/hc/en-us/sections/19096742289300-Merlin-AI).

# Push Device Analytics

The *Device Analytics* feature provides immediate visibility into campaign performance through the lens of specific device attributes. This reporting tool eliminates ambiguity surrounding delivery or click-rate fluctuations by replacing broad assumptions with granular, data-backed insights.

This feature empowers you to slice metrics by critical factors such as OEM, OS Version, App Version, Device Recency, and Browser to diagnose performance issues with precision. By allowing you to cross-reference these factors using Heatmaps, you can determine if low delivery is due to a specific user mix (e.g., dormant users on specific OEMs) or spot integration issues between specific app versions and OS updates. This ensures you can optimize messaging for the right technical environments and verify the ROI of your strategies.

*Device Analytics* helps you:

- Analyze campaign metrics broken down by key dimensions, including OEM, OS Version, App Version, Device Recency (Active vs. Dormant), and Browser.
- Utilize Heatmaps to cross-reference two factors simultaneously, such as checking App Version against OS Version to debug potential SDK or integration issues.
- Determine the precise cause of delivery drops, such as whether high volumes of dormant users on specific OEMs are penalizing your reach.
- Validate the effectiveness of recent optimizations, such as deep-link handling fixes or the impact on conversion rates of iOS Provisional Authorization.

For more information and configuration details, refer to [Push Device Analytics](/user-guide/campaigns-and-channels/mobile-push/analyze/push-device-analytics).
