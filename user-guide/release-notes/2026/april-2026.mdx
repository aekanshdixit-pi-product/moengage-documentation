## Merlin AI Campaign Insight Agent

MoEngage has introduced the Merlin AI Campaign Insight Agent, which translates campaign metrics into plain-language explanations and recommended next steps for **one-time and periodic** Push and Email campaigns. The agent surfaces delivery risk signals, compares current performance against your historical sends, and suggests what to try next—so you spend less time stitching together dashboards and more time acting on the results.

<Note>
Rollout is phased. MFA login is required for early access.
</Note>

For more information, refer to [Campaign Insight Agent](/user-guide/ai-and-intelligence/merlin-ai/campaign-insights-agent).

## MoEngage Go for iOS

MoEngage Go is now available on the **iOS App Store**, allowing you to manage campaigns from your phone. With the app, you can:

- View active and scheduled campaigns.
- Approve or reject items pending review.
- Pause or stop live sends.

Sign-in supports SSO, Google, and email, with the same roles and permissions as the web app. Campaign creation and editing remain on desktop because those workflows require the full interface.

<Note>
Android availability was planned to follow after App Store review completion.
</Note>

## Just-in-Time User Provisioning

MoEngage now supports **Just-in-Time (JIT) user provisioning** for SSO-enabled workspaces. When a team member logs in with SSO for the first time, their workspace account is created or updated automatically from your identity provider—no manual invite required. This keeps access aligned with your corporate identity rules, including time-bound access policies where you configure them.

JIT provisioning is currently in **beta**, with access controlled by phased rollout. SSO must already be configured for your workspace.

For more information, refer to [Just-in-Time User Provisioning](https://help.moengage.com/hc/en-us/articles/47445496296212-Just-In-Time-User-Provisioning-JIT).

## Background Update Push Template

MoEngage has added the **Background Update** push template, which delivers a custom key–value data payload to your app without displaying a visible notification. The template uses the same setup as other push templates and includes a payload preview before you send. It is available for one-time, periodic, and event-triggered campaigns as well as Flows, and it is supported on Android and iOS. Minimum SDK versions apply.

This template is useful for refreshing app state, updating feature flags, or triggering security-related flows—including for users who have opted out of visible push notifications.

For more information, refer to [Push Templates](/user-guide/campaigns-and-channels/mobile-push/create/push-templates#background-update).

## New Connected Sources, Audiences, and Conversion APIs

MoEngage has expanded its partner ecosystem with several additions:

- **Connected sources**: Behamics, Adapty, and Zoho Landing bring intent, subscription, and lead signals into user profiles and journeys.
- **Ad audience connections**: Amazon Ads and Microsoft Ads are now available as audience destinations.
- **Server-side conversion delivery**: Meta, TikTok, Snapchat, and Microsoft Ads now support server-side conversion APIs.
- **Adobe Experience Manager (AEM) Experience Fragments**: Sync multi-locale content from AEM directly as email templates.

Availability varies by data center and Connected Channels packaging. For partner setup guides, refer to [partners.moengage.com](https://partners.moengage.com/).

## Primary Device Per User and Device Opt-Out API

MoEngage has introduced two capabilities for device-level push control:

- **Primary device per user**: Marks the most recently logged-in device for a user so you can restrict sends to that device across Push, Inform, Flows, and APIs. This is useful for sensitive alerts or reducing duplicate sends across a user's tablets and phones. Enablement may require a support-requested flag.
- **Device opt-out API**: Lets you block or unblock a specific device from receiving push through the API. This supports fraud detection and device-loss scenarios.

For more information, refer to [Primary Device Per User](https://help.moengage.com/hc/en-us/articles/33347397582228-Primary-Device-Per-User).

## Rename Custom Segments

You can now rename custom segments from the **All Segments** list or the segment detail view. The segment ID does not change, so existing API calls, automations, and integrations continue to work without modification. Renamed segments are reflected across navigation, recent queries, campaign views, edit history, and audit entries.

<Note>
Renaming is controlled by a feature flag and is not yet supported for all segment sources.
</Note>

## Campaign and Flow List Improvements

MoEngage has updated the **All Campaigns** and **All Flows** list views with the following improvements:

- **Shimmer placeholders** appear while data loads, so the list no longer shows an empty or misleading state mid-fetch.
- A **dedicated empty illustration** displays when no results match your active filters.
- **Search text and filter selections persist** when switching between All, Scheduled, and Active tabs, so you do not need to reapply them.
- **Clicking the search icon** triggers a search, consistent with other search fields across the platform.

Previously, the All Campaigns list could display a "No campaign found" message while results were still loading. The list now waits for data to finish loading before showing the empty state.
