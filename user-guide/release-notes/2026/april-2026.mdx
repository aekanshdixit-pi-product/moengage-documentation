## Merlin AI Campaign Insight Agent

MoEngage has introduced the Merlin AI Campaign Insight Agent, which translates campaign metrics into plain-language explanations and actionable next steps for one-time and periodic Push and Email campaigns. The agent surfaces delivery risk signals, benchmarks current performance against historical sends, and provides recommendations on optimization opportunities—enabling teams to reduce time spent navigating disparate dashboards and focus on acting on insights.

For more information, refer to [Campaign Insight Agent](/user-guide/ai-and-intelligence/merlin-ai/campaign-insights-agent).

## MoEngage Go for iOS

MoEngage Go is now available on the **iOS App Store**, allowing you to manage campaigns from your phone. With the app, you can:

- View active and scheduled campaigns.
- Approve or reject items pending review.
- Pause or stop live sends.

Sign-in supports SSO, Google, and email, with the same roles and permissions as the web app. Campaign creation and editing remain on desktop because those workflows require the full interface.

## Just-in-Time User Provisioning

MoEngage now supports **Just-in-Time (JIT) user provisioning** for SSO-enabled workspaces. When a team member logs in with SSO for the first time, their workspace account is created or updated automatically from your identity provider—no manual invite required. This keeps access aligned with your corporate identity rules, including time-bound access policies where you configure them.

For more information, refer to [Just-in-Time User Provisioning](/user-guide/settings/account/security/just-in-time-user-provisioning-jit).

## Background Update Push Template

MoEngage now supports the _Background Update Push Template_, allowing you to deliver custom key–value data payloads directly to your app without triggering a visible notification.

This template uses standard push campaign configurations and includes a full payload preview before sending. It supports both Android and iOS across one-time, periodic, and event-triggered and Flows campaigns.

**Key Capabilities:**

- **Silent Delivery**: Reach audiences who have opted out of push notifications without interruption.
- **Background Actions**: Seamlessly refresh app states, update feature flags, or trigger security-related workflows in the background.

For more information, refer to [Push Templates](/user-guide/campaigns-and-channels/mobile-push/create/push-templates#background-update).

## New Additions to the Partner Ecosystem

MoEngage has expanded its partner ecosystem with several additions:

- **Connected sources**: Behamics, Adapty, and Zoho Landing bring intent, subscription, and lead signals into user profiles and journeys.
- **Ad audience connections**: Amazon Ads and Microsoft Ads are now available as audience destinations.
- **Server-side conversion delivery**: Meta, TikTok, Snapchat, and Microsoft Ads now support server-side conversion APIs.
- **Adobe Experience Manager (AEM) Experience Fragments**: Sync multi-locale content from AEM directly as email templates.

For partner setup guides, refer to the [Partner Guide](https://partners.moengage.com).

## Primary Device Per User and Device Opt-Out API

MoEngage has introduced two capabilities for device-level push control:

- **Primary device per user**: Marks the most recently logged-in device for a user so you can restrict sends to that device across Push, Inform, Flows, and APIs. This is useful for sensitive alerts or reducing duplicate sends across a user's tablets and phones.
- **Device opt-out API**: Allows you to block or unblock a specific device from receiving push notifications through the [API](https://www.moengage.com/docs/api/device/device-opt-out). This supports fraud detection and device-loss scenarios.

For more information, refer to [Primary Device Per User](/user-guide/campaigns-and-channels/mobile-push/notification-features-and-behavior/primary-device-per-user).

## Rename Custom Segments

You can now rename custom segments from the **All Segments** page or the segment detail view. The segment ID does not change, so existing API calls, automations, and integrations continue to work without modification. Renamed segments are reflected across navigation, recent queries, campaign views, edit history, and audit entries.

For more information, refer to [Renaming a Custom Segment](/user-guide/segment/segment-operations/custom-segments#renaming-a-custom-segment).

## Improvements to All Campaigns and All Flows Pages

MoEngage has updated the **All Campaigns** and **All Flows** pages with the following improvements:

- Shimmer placeholders appear while data loads, so the list no longer shows an empty or misleading state mid-fetch.
- A dedicated empty illustration displays when no results match your active filters.
- Search text and filter selections persist when switching between **All**, **Active**, and **Schedule** tabs.
- The search icon in the search box is now clickable.

Previously, the All Campaigns page could display a "No campaign found" message while results were still loading. The page now waits for data to finish loading before showing the empty state.
