## Decisioning Agent Analytics (Early Access)

MoEngage has introduced the Decisioning Agent Analytics dashboard for Campaign Decisioning. Use it to measure the incremental value generated by your Decisioning Agents by comparing an AI-optimized Agent Treatment Group against a baseline Agent Control Group.

**Key Highlights:**

- **Engagement uplift:** Track the net percentage increase in engagement achieved by the AI engine compared to non-optimized baselines.
- **Operational summary:** View quick cards for Decisions Made, Campaigns Sent, Active Channels, and overall Engagement Uplift.
- **Treatment vs. control comparisons:** Review performance summary tables that show granular event metrics, including positive signals such as clicks and purchases and negative signals such as unsubscribes.
- **Channel and campaign breakdowns:** Identify top-performing assets using channel donut and bar charts, and filter or sort the Campaign Performance Table by Sent, Delivered, or CTR.

This feature is in Early Access. Phase 1 covers Engagement Uplift; conversion metrics and CSV exports are planned for future phases.

For more information, refer to [Decisioning Agent Analytics](/user-guide/decisioning/campaign-decisioning/decisioning-agent-analytics).

## Custom Formula Ranking Strategy for Offer Decisioning

MoEngage has added a Custom Formula ranking strategy to Offer Decisioning. Use it to define which factors influence how offers are ranked for each customer, and how much weight each factor carries, with scores recalculated in real time. This combines manual control with personalization, so rankings stay both commercially intentional and relevant to the individual user.

**Key Highlights:**

- **Fixed value attributes:** Org-level goals stamped on each offer at creation, such as Boost AOV, Improve Retention, or Acquire New Users. The formula weight reflects how much each goal should drive ranking.
- **Dynamic value attributes:** User-level signals, such as Category Affinity, Churn Propensity, or Spend Tier, that shift the rank based on who receives the offer.
- **Real-time scoring:** Fixed and dynamic attributes combine into a per-user score that updates as signals change.

For more information, refer to [Custom Formula Ranking Strategy](/user-guide/decisioning/offer-decisioning/custom-formula-ranking-strategy) and [Create an Offering Attribute](/user-guide/decisioning/offer-decisioning/create-offering-attribute).

## Portfolio Support in Partner Integrations

MoEngage now routes integration events directly to specific projects within multi-app (Portfolio) workspaces. Brand-specific event data lands in the correct project, keeping localized analytics and targeting accurate and enabling cross-brand campaigns without manual workarounds.

**Key Highlights:**

- **Supported CDP partners:** Segment, mParticle, and Adobe.
- **Supported file imports:** CSV uploads, S3, and SFTP.
- **Supported data warehouse imports:** BigQuery, Snowflake, and Databricks.
- **Configuration:** For Segment, enter the target project in the new Project Name field in the Track Call action mapping. For mParticle and other partners, pass the project name as an event attribute named `moe_project_name` in the event payload.

This release is available across all data centers and is backward compatible; non-multi-app workspaces are unchanged. The scope is event routing only. User profile imports and cohort syncs from these partners are not yet supported for Portfolio routing and continue to be ingested at the portfolio root level.

## Enhanced User Profile

MoEngage has revamped the user profile, beginning with availability in DC3. The update adds profile management actions and improves the overall interface.

**Key Highlights:**

- **Delete a user:** Delete a user directly from the UI based on roles and permissions. An audit log entry is generated for each delete action.
- **Email subscription management:** Change a user's email subscription status. The change generates a tracking event, triggers a downstream SendGrid subscription update, and is recorded in the audit log.
- **SMS unsubscribe:** Unsubscribe a user from the SMS channel.
- **Identifier visibility:** In Multi-ID workspaces, any user attribute marked as an identifier appears in the basic details section.
- **Interface improvements:** More discoverable tabs, better event filtering, consistent terminology, and improved readability.

The UI and UX improvements roll out across data centers independently and are currently live in DC3. Delete and subscription functionality is enabled in phases.

For more information, refer to [User Profile](/user-guide/data/user-data/user-profile).

## Native SDK Support for MoEngage Personalize

You can now fetch MoEngage Personalize content directly through the MoEngage SDK using native methods, instead of making direct HTTPS calls to the Personalize Fetch API. Native support is available for Android, iOS, React Native, and Flutter.

**Key Highlights:**

- **No auth token management:** The SDK handles user identity and authentication internally, so you no longer pass credentials with every request.
- **Session and lifecycle alignment:** Because the fetch happens through the initialized SDK, it aligns with the user's session state without syncing user context separately.
- **Structured error handling:** Failure callbacks return standardized reason codes such as `NETWORK_ERROR` and `INVALID_KEY`, making integration issues easier to debug.

For more information, refer to the platform integration guides for [Android](/developer-guide/android-sdk/personalize/personalize-sdk), [iOS](/developer-guide/ios-sdk/personalize/personalize-sdk), [React Native](/developer-guide/react-native-sdk/personalize/personalize-sdk), and [Flutter](/developer-guide/flutter-sdk/personalize/personalize-sdk).

## MoEngage Assist Chrome Extension for Web SDK

MoEngage Assist is a Chrome extension that moves Web SDK debugging into the browser, so internal teams and customers can validate integrations without screen-shares or support tickets.

**Key Highlights:**

- **Automatic setup validation:** Flags common misconfigurations such as an incorrect Service Worker path, initialization errors, and data center validation issues.
- **Campaign inspection:** Inspect Cards, Personalize, and On-site Messaging campaigns to see what is configured, what qualifies, and why a campaign is not firing.
- **Render All preview:** Preview Web Personalization campaigns on any page without affecting live impressions or analytics.
- **Real-time events:** View events firing as you navigate to confirm data is flowing before you publish.
- **Issues view:** See everything tied to a Web SDK integration in one place for triage.

For more information, refer to [MoEngage Assist Chrome Extension](/developer-guide/web-sdk/integration-validation/moengage-assist-chrome-extension).

## Merlin AI In-App Template Generator: Image as Reference

The Merlin AI In-App Template Generator now accepts an image as a reference. Upload a PNG of a design, describe what you want, and Merlin generates a matching HTML template. Generated templates include MoEngage tracking, SDK-optimized rendering, and responsiveness.

**Key Highlights:**

- **Image-based generation:** Use the Add image with your prompt option in the Merlin AI chat, upload a PNG, and prompt Merlin to create a similar template.
- **Design tool workflow:** Export a frame from Figma, Photoshop, Sketch, or any tool that exports to PNG, then upload it to convert the design into a template without a developer handoff.
- **Iterative refinement:** Adjust the result with follow-up prompts, the same as text-only generation.

This feature is available in all databases where the Merlin AI template generator is enabled.

For more information, refer to [Merlin AI In-App Template Generator](/user-guide/ai-and-intelligence/merlin-ai/merlin-ai-in-app-template-generator/merlin-ai-in-app-template-generator#use-a-reference-image).

## All Campaigns Page Enhancements

MoEngage has shipped several usability improvements to campaign management:

- **Copy Link:** Copy a campaign link directly from the All Campaigns page to share with your team.
- **Search term retention on the Drafts tab:** A search term entered on the All Campaigns page now carries over when you switch to the Drafts tab, reducing repeated searches.
- **Alert name wrapping:** Alert names now wrap up to a set limit on the All Alerts page instead of being truncated.
- **Action icons:** Icons have been added to several actions to improve visual clarity and accessibility.
