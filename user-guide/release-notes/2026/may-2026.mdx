## Decisioning Agent Analytics (Early Access)

MoEngage has introduced the Decisioning Agent Analytics dashboard for Campaign Decisioning. Use it to measure the incremental value generated by your Decisioning Agents by comparing an AI-optimized Agent Treatment Group against a baseline Agent Control Group. The dashboard reports engagement uplift, a summary of decisions made, campaigns sent, and active channels, and treatment-versus-control tables that break performance down by channel and campaign. This feature is in Early Access; Phase 1 covers engagement uplift, with conversion metrics and CSV exports planned for later phases.

For more information, refer to [Decisioning Agent Analytics](/user-guide/decisioning/campaign-decisioning/decisioning-agent-analytics).

## Custom Formula Ranking Strategy for Offer Decisioning

MoEngage has added a Custom Formula ranking strategy to Offer Decisioning. Use it to define which factors influence how offers are ranked for each customer and how much weight each factor carries, with scores recalculated in real time. Formulas combine fixed, org-level goals stamped on each offer (such as Boost AOV or Improve Retention) with dynamic, user-level signals (such as Category Affinity, Churn Propensity, or Spend Tier), so rankings stay both commercially intentional and relevant to the individual.

For more information, refer to [Custom Formula Ranking Strategy](/user-guide/decisioning/offer-decisioning/custom-formula-ranking-strategy) and [Create an Offering Attribute](/user-guide/decisioning/offer-decisioning/create-offering-attribute).

## Portfolio Support in Partner Integrations

MoEngage now routes integration events directly to specific projects within multi-app (Portfolio) workspaces, so brand-specific event data lands in the correct project and keeps localized analytics and targeting accurate. For Segment, enter the target project in the new Project Name field in the Track Call action mapping; for mParticle and other partners, pass the project name as an event attribute named `moe_project_name` in the event payload. Support covers the Segment, mParticle, and Adobe CDP partners; CSV, S3, and SFTP file imports; and BigQuery, Snowflake, and Databricks warehouse imports. The release is available across all data centers and is backward compatible, so non-multi-app workspaces are unchanged. Scope is limited to event routing; user profile imports and cohort syncs from these partners continue to be ingested at the portfolio root level.

## Enhanced User Profile

MoEngage has revamped the user profile, starting with availability in DC3. You can now delete a user from the UI based on roles and permissions, change a user's email subscription status (which generates a tracking event, triggers a downstream SendGrid update, and is recorded in the audit log), and unsubscribe a user from the SMS channel. In Multi-ID workspaces, any user attribute marked as an identifier now appears in the basic details section. The release also improves tab discoverability, event filtering, terminology consistency, and readability. UI and UX improvements roll out per data center, while delete and subscription functionality is enabled in phases.

For more information, refer to [User Profile](/user-guide/data/user-data/user-profile).

## Native SDK Support for MoEngage Personalize

You can now fetch MoEngage Personalize content directly through the MoEngage SDK using native methods, instead of making direct HTTPS calls to the Personalize Fetch API. The SDK handles user identity and authentication internally, aligns the fetch with the user's session state, and returns structured failure callbacks with standardized reason codes such as `NETWORK_ERROR` and `INVALID_KEY`. Native support is available for Android, iOS, React Native, and Flutter.

For more information, refer to the integration guides for [Android](/developer-guide/android-sdk/personalize/personalize-sdk), [iOS](/developer-guide/ios-sdk/personalize/personalize-sdk), [React Native](/developer-guide/react-native-sdk/personalize/personalize-sdk), and [Flutter](/developer-guide/flutter-sdk/personalize/personalize-sdk).

## MoEngage Assist Chrome Extension for Web SDK

MoEngage Assist is a Chrome extension that moves Web SDK debugging into the browser, so internal teams and customers can validate integrations without screen-shares or support tickets. It validates setup and flags common misconfigurations such as an incorrect Service Worker path or initialization errors, lets you inspect Cards, Personalize, and On-site Messaging campaigns to see why a campaign is or is not firing, previews Web Personalization campaigns without affecting live impressions, and shows events firing in real time.

For more information, refer to [MoEngage Assist Chrome Extension](/developer-guide/web-sdk/integration-validation/moengage-assist-chrome-extension).

## Web SDK Support for MCP UI Apps

The MoEngage Web SDK now supports MCP UI Apps, the interactive mini-apps that run inside AI chat interfaces such as ChatGPT and Claude and let users take actions without leaving the conversation. Within these apps you can track user behavior in real time, trigger On-site Messages, and serve Cards.

For more information, refer to [Integrate MoEngage With MCP UI Apps](/developer-guide/web-sdk/other-supported-web-sdk-integration/integrate-moengage-with-mcp-ui-apps).

## Merlin AI In-App Template Generator: Image as Reference

The Merlin AI In-App Template Generator now accepts an image as a reference. Upload a PNG of a design exported from Figma, Photoshop, Sketch, or any similar tool, describe what you want, and Merlin generates a matching HTML template that you can refine with follow-up prompts. Generated templates include MoEngage tracking, SDK-optimized rendering, and responsiveness. This feature is available in all databases where the Merlin AI template generator is enabled.

For more information, refer to [Merlin AI In-App Template Generator](/user-guide/ai-and-intelligence/merlin-ai/merlin-ai-in-app-template-generator/merlin-ai-in-app-template-generator#use-a-reference-image).

## MoEngage Go Android App

The MoEngage Go Android app is now available on the Google Play Store, extending the MoEngage Go companion app to Android devices.

## All Campaigns Page Enhancements

MoEngage has shipped several usability improvements to campaign management:

* **Copy Link**: Copy a campaign link directly from the All Campaigns page to share with your team.
* **Search term retention on the Drafts tab**: A search term entered on the All Campaigns page now carries over when you switch to the Drafts tab.
* **Alert name wrapping**: Alert names now wrap up to a set limit on the All Alerts page instead of being truncated.
* **Action icons**: Icons have been added to several actions to improve clarity and accessibility.
