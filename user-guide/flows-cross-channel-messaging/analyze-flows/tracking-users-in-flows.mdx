## Tracking Users Who Entered a Flow

To identify users who have entered a flow, you can segment the users who have performed the "User Entered Flow" event. The event is tracked for each entry done by a user.

<img alt="Screenshot_2022-07-30_at_11.19.24_PM.png" src="/images/moengage_748986.png" />

Below are some examples:

| Segment Entered User | Event Attributes to be Used | Description |
| --- | --- | --- |
| By a flow | Flow Id or Flow Name | By filtering using **Flow Id**, you can find the list of unique users whose conversion was attributed to this flow. |
| By a particular version | Version Name | By filtering using **Version Name**, you can find the list of unique users whose conversion was attributed to this version of the flow. |
| By user type | With control group | By filtering using **With control group =** **True**, you can find the list of users who entered the flow as Control Group (both Global and Flow). By filtering using **With Control Group =** **False**, you an find the list of users who entered the flow as Target Group. |
| By Control Group Type | Control Group Type | By filtering using **Control Type Group = Flow**, you can find the list of users who entered the flow as Control Group. By filtering using **Control Type Group = Global**, you can find the list of users who entered the flow as Global Control Group. |
| By Trigger event type | Trigger Event Met | By filtering using **Trigger Event met** equals to one of your trigger event from a flow. This allows you to identify the users who entered the flow specifically due to a particular trigger event. This is particularly useful when your flows contain multiple primary events. |

<Info>
- The Event time of a User Entered Flow event will show in the User's timezone. If it is unavailable, then the App's timezone is shown. If neither is available, then UTC time is taken as the Event Time.

  **Note:** The displayed event time is not adjusted for local Daylight Saving Time changes.
- You can find the Flow Id for your campaign by opening your flow in a new window and copying the string that follows **/flows/** from URL. For example, 6616a9259670a17eec8b7c65 is the Flow ID in the following image: <img alt="flow ID.png"  src="/images/moengage_bb0719.png"  />
</Info>

## Tracking Users Who Converted

To identify users who have performed the conversion goal event after receiving one of the Flow Action campaigns (that is, converted users), you can segment the users who have performed the "Flow Trip conversion".

<img alt="Screenshot 2024-04-13 at 2.04.50 PM.png" src="/images/moengage_f56f27.png" />   
The event is tracked for each conversion, so if a user performs the conversion multiple times during their flow trip, this event will be raised for each recorded conversion and also for each of the defined goals.

You can identify the users who have converted for a particular flow or a particular action campaign.

Below are some examples:

| Segment Converted User | Event Attribute to be Used | Description |
| --- | --- | --- |
| By a flow | Flow Id or Flow Name | By filtering using **Flow Id**, you can find the list of unique users whose conversion was attributed to this flow. |
| By a particular version | Version Name | By filtering using **Version Name**, you can find the list of unique users whose conversion was attributed to this version of the flow. |
| By a particular flow Action Campaign | Flow Action Campaign Id | By filtering using the **Campaign Id**, you can find the list of unique users whose conversion was attributed to this campaign of the flow. |
| By a particular flow goal event | Flow Conversion Event or Goal Name | By filtering using the **Goal Name**,  you can find the list of unique users who converted by doing this event. |
| By a particular attribution type | Attribution Type | By filtering using the **Attribution Type**, you can find the list of unique users who converted as per the specific attribution logic. |
| For all control group users | User Type | By filtering using the **User Type = Control Group**, you can find the list of unique control group users who converted. |
| For all non-control group users | User Type | By filtering using the **User Type = Targeted**, you can find the list of unique control group users who converted. |

## Tracking Users Who Exited a Flow

To identify users who have exited from a flow, you can segment the users who have performed the "User Exited Flow".

<img alt="Screenshot_2022-09-07_at_2.22.22_PM.png" src="/images/moengage_d29da0.png" />

The event is tracked for each exit done by a user. You can identify a user who has exited a flow with related filters such as exited on conversion, completing a certain flow, and so on.

Below are some examples:

| Segment Exited user | Event Attributes to be Used | Description |
| --- | --- | --- |
| By a flow | Flow Id or Flow Name | By filtering using **Flow Id**, you can find the list of unique users whose conversion was attributed to this flow. |
| By a particular version | Version Name | By filtering using **Version Name**, you can find the list of unique users whose conversion was attributed to this version of the flow. |
| By a particular stage | Stage Id or Stage Name | By filtering using **Stage Id**, you can find the list of unique users who exited the flow through that stage. |
| By a particular exit reason | Exit Reason | By filtering using **Exit Reason**, you can find the list of unique users who exited the flow due to a specific reason. |
| By user type | With control group | By filtering using **With control group =** **True**, you can find the list of users who entered the flow as Control Group (both Global and Flow). By filtering using **With control group =** **False**, you can find the list of users who entered the flow as Target Group. |

<Info>
- Using the "User Exited Flow" event, you can segment how many and which users have exited a specific flow during certain time duration and retarget them through any of the campaigns (Push/Email/SMS/Connector/Flows) depending on your use case.
- The Event time of a User Exited` Flow event will show in the User's timezone. If it is unavailable, then the App's timezone is shown. If neither is available, then UTC time is taken as the Event Time.

  **Note:** The displayed event time is not adjusted for local Daylight Saving Time changes.
</Info>

## Tracking Users Who Advanced Through an Experiment Stage

To identify users who have been allocated to a specific branch of an AB Split or Intelligent Path Optimiser (IPO) stage, you can segment users who have performed the "User Flow Advancement" event. This event is raised each time a user passes through an experiment stage and is assigned to a branch.

<Info>
User Flow Advancement is a [Billable Standard Event](/user-guide/data/event-data/derived-events-and-attributes#billable-standard-events). It is tracked at your request and each occurrence contributes to your billed event count. To get this enabled for your account, contact your MoEngage account manager or raise a [Support Ticket](/user-guide/contact-support/raise-a-support-ticket-through-moengage-dashboard).
</Info>

<img alt="User Flow Advancement event in segmentation" src="/images/User_flow_adv.png" />

Below are some examples:

| Segment Users Who Advanced | Event Attributes to be Used | Description |
| --- | --- | --- |
| By a flow | Flow Id or Flow Name | By filtering using **Flow Id** or **Flow Name**, you can find the list of unique users who were allocated to a branch within that flow. |
| By a particular version | Version Name | By filtering using **Version Name**, you can find the list of unique users who were allocated to a branch in a specific version of the flow. |
| By a particular experiment stage | Stage Id or Stage Name | By filtering using **Stage Id** or **Stage Name**, you can find the list of unique users who passed through a specific experiment stage. |
| By experiment stage type | Stage Type | By filtering using **Stage Type = AB Split**, you can find users who passed through an AB Split stage. By filtering using **Stage Type = IPO**, you can find users who passed through an Intelligent Path Optimiser stage. |
| By a particular branch | Branch Name | By filtering using **Branch Name**, you can find the list of unique users who were allocated to that specific branch. |
| By flow type | Flow Type | By filtering using **Flow Type**, you can find users who advanced within a specific type of flow — Event Triggered, Time Based, or Business Event Triggered. |
| By user type | User Type | By filtering using **User Type = Target Group**, you can find users in the target group. By filtering using **User Type = Flow Control Group** or **User Type = Global Control Group**, you can find users in the respective control groups. |

<Info>
- Using the "User Flow Advancement" event, you can segment users who entered a specific branch and retarget them through any of your channels (Push / Email / SMS / WhatsApp / Flows) depending on your use case.
- Since this event is tracked independently of the flow's attribution window, you can use it to run longer-term analysis of user behavior — conversions, retention, or any other metric — beyond what is available in the flow's built-in reporting.
- By filtering on **Branch Name**, you can compare users who entered different branches of an experiment in MoEngage Analytics or export them to third-party tools for deeper analysis.
- By filtering on **User Type**, you can include or exclude control group users from your analysis to get a clean view of how targeted users in each branch performed.
</Info>
