title: "iOS Push Notifications Integration Validation"
---

This article explains iOS Push Notifications Integration validation manually in a stepwise manner.

# What is to be validated?

To make sure Push notifications are working properly on the device. Please ensure that the .pem/.p8 files are [uploaded to the MoEngage Dashboard](/user-guide/campaigns-and-channels/mobile-push/getting-started-with-mobile-push/configure-mobile-push) before proceeding with the validation.

# How to validate?

## 1. Push Token Generation

- On [your profile](/user-guide/data/user-data/user-profile) page, the IOS push icon has to be green. <img alt="UserProfile_TestOptions.png" src="/images/moengage_8aa84d.png" />

2. Push notification received on the device

- Create a general Push campaign on the Dashboard for your device.
- Test the notification from the test option at the 3rd stage of creation.
- If you get a successful response, create a campaign. If the test has failed, check for the failure reasons. <img alt="download__12_.png" src="/images/moengage_63df2f.png" />
- Check for the icon to be visible on the push notification on your device.

3. Notification Redirection

The following default click actions are supported during campaign creation as shown in the image below:

- Navigate to a screen
- Deeplinking
- Rich landing

<img alt="Notification_Redirection.png" src="/images/moengage_dba2f0.png" />

**Navigate to a screen**

While creating the campaign, select the navigate to Screen option and paste a screen name in the box. On receiving the notification, click on it and the screen selected in the dropdown should open on the app.

<img alt="NavigateToAScreen.png"  src="/images/moengage_6bf763.png"  />

**Deeplinking**

While creating the campaign, select the deeplinking option and add the URL. On clicking the notification, the user has to be redirected to the exact portion of the app intended.

<img alt="Deeplinking.png"  src="/images/moengage_63e380.png"  />

**Rich landing**

While creating the campaign, select the Rich landing option, and add the URL. Click the notification, the user has to be redirected to the web to the provided URL.

<img alt="RichLanding.png"  src="/images/moengage_cbfb0f.png"  />

4. Notification stats

- Once the campaign is sent out, the campaign stats should show the impressions and clicks. <img alt="download__16_.png" src="/images/moengage_37cfcb.png" />

# Success Criteria

1. A push token has to be generated for the users.
2. Notifications should be displayed correctly.
3. The redirection(3 different types) should work as expected.
4. The Impressions and clicks have to be captured.
5. In case of rich notifications(if present), check for the image rendering, notification sound, buttons, their redirections, and so on.

# Potential issues and Solutions

1. A push token has to be generated for the users. To fix Push Token-related issues, please check [iOS](https://developers.moengage.com/hc/en-us/articles/43960004257428-iOS-Push-Integration-Tutorial#h_01K4729FAAANVZVP1NXQWM82GS) docs.
2. Notification should be displayed correctly.
3. The redirection(3 different types) should work as expected.
4. The Impressions and clicks have to be captured. To fix the Impression, or click-related issue, please check [iOS](https://developers.moengage.com/hc/en-us/articles/43960004257428-iOS-Push-Integration-Tutorial#h_01K45ED56HJA1KZP43JP0ZGG1F) docs.
5. In case of rich notifications(if present), check for the image rendering, notification sound, buttons, their redirections, and so on. To fix the Rich Push-related issue, please check the [iOS](https://developers.moengage.com/hc/en-us/articles/43960004257428-iOS-Push-Integration-Tutorial#h_01K45FKD0XDH4X776R47JA04GJ) docs.
6. Note - By default iOS impression tracking was not present in the earlier version but from iOS 10, this can be integrated using [Notification Service extension](https://developers.moengage.com/hc/en-us/articles/43960004257428-iOS-Push-Integration-Tutorial#h_01K45FKD0XDH4X776R47JA04GJ) capabilities.
