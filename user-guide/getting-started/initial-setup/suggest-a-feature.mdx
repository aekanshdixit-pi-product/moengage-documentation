MoEngage provides a feature request system built into the product dashboard so you can suggest a feature. That lets you share ideas directly with the product and leadership team at MoEngage.

## How does this help?

MoEngage values your suggestions and keeps you in the loop when we review or update your ideas or requests. It also allows you to:

- Get timely feedback on requests from product decision-makers
- Gain greater insight into upcoming releases
- Increase your impact on feature development

Each month, the product team and stakeholders at MoEngage review the top 5–10 customer requests and respond to them personally. The ones that qualify become part of our roadmap.

To get your requests into these prioritization meetings, provide all the required details on the **Submit a request** page.

### Create a request

To submit a new feature request or product idea:

1. On the MoEngage Dashboard, click **Need help** on the top navigation bar.
2. Click **Submit product idea**. The Feedback Portal opens.

<Frame>
  ![Submit Product Idea](/images/SubmitProductIdea.png)
</Frame>

3. In the left navigation panel, click **Requests**. A list of all previously submitted requests for your workspace appears.
4. Click **\+ Create**. The **Create a ticket** window appears.

<Frame>
  <img
    src="/images/create-a-feed.png"
    width="896"
    height="1058"
    alt="Create A Feed"
    title="Create A Feed"
  />
</Frame>

5. Fill in the following fields:

| Field | Required | Description |
| --- | --- | --- |
| **Title** | Yes | Enter a concise title that summarizes your request. |
| **Description** | Yes | Provide a detailed description of the feature or idea, including the use case and expected outcome. |
| **Attachment** | No | Click or drag files to upload any supporting documents, screenshots, or reference materials. |
| **Module** | No | Select the relevant product module from the dropdown. Available options include Campaigns, Flows, Push Notifications, Email, SMS, MMS & RCS, WhatsApp, Inbound Channels, Cards, Data, Account Management, Analytics, and more. |
| **Submodule** | No | Select a submodule if applicable. The available submodules depend on the module selected. For example, selecting **SMS, MMS & RCS** shows submodule options such as SMS, MMS, RCS, and TCPA & SMS compliance. |
| **Type** | No | Select the type of request: Feature request, Issue, or Other. |
| **Requester email** | No | The email address of the requester. |
| **Created by** | No | Enter the user ID or name of the person creating the request. |

6. Click **Create ticket**. The ticket is created and appears in the Requests list.

### View and track requests

After submitting a request, you can track it from the **Requests** section. The requests list shows the following details for each entry:

| Column | Description |
| --- | --- |
| **Title** | The title of the submitted request. Click to expand and view the full description. |
| **Created at** | The date when the request was submitted. |
| **Insights** | The number of insights or interactions associated with the request. |
| **Created by** | The name of the user who submitted the request. |

You can use the **All** dropdown, the **Filter** option, or the search icon to narrow down the list of requests.

### Labs

1. Click **Labs** in the left navigation panel to view requests that have been converted into work items.
2. Use the **Active** and **Past** tabs to switch between ongoing and completed work items.
3. You can search for specific items, upvote work items you find valuable, and add comments to share additional context.

### Roadmaps

Click **Roadmaps** in the left navigation panel to view the planned features and enhancements for upcoming quarters.

## Feature status

The following are the statuses you see for every feature request you have submitted.

| Status | Meaning |
| --- | --- |
| Awaiting feedback | Your request stays here while we gauge demand from our other customers. |
| Planned | This request is on our roadmap but we have not started on it yet. |
| Building | This request is currently under development. |
| Released | This request is complete. See the status message for how to use it. |
| Declined | This request was not clear, already exists, or is not aligned with where we are taking our product. See the status message for more information. |

If you have any questions about this process, write to us at [product@moengage.com](mailto:product@moengage.com).
