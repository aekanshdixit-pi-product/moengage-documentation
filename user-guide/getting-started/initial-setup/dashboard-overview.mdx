From the MoEngage dashboard, you can create user segments, send targeted campaigns for your app or website, and access analytics including uninstalls, acquisitions, retention cohorts, campaign performance, and conversion data.

![Dashboard overview](/images/moengage_358a60.png)

# Navigating the Dashboard

The left navigation sidebar gives you access to the main sections of MoEngage:

- **Campaigns & Channels** — Create and manage campaigns across push, email, SMS, in-app, and other channels.
- **Flows** — Build automated multi-step journeys for your users.
- **Analyze** — Access analytics, cohorts, funnels, and campaign performance reports.
- **Segment** — Create and manage user segments.
- **Settings** — Configure your workspace, channels, integrations, and team access.

At the top right you'll find:

- Your **profile icon** — Access My Profile, switch appearance, and manage your workspace settings.
- The **workspace switcher** — Switch between workspaces or add a new one.
- **Need Help** — Access documentation, videos, and support.

# First-Time Setup

The first time when you log in to MoEngage, the Dashboard is available as a Test environment. You can integrate MoEngage with your app or web and get user or event data.

Make your environment Live after integration or tracking of events is complete and your app is released or your website is live.

# Live and Test Environments

Every MoEngage workspace has two environments: **Test** and **Live**. You switch between them using the toggle on the top navigation bar.

![Live and Test environment toggle](/images/moengage_2901a7.png)

Both environments share the same Workspace ID (found under **Settings** \> **Account** \> **General settings**), but all other settings are independent.

|  | Test | Live |
| --- | --- | --- |
| **Purpose** | Integration, debugging, and QA | Production user data and campaigns |
| **Data routing** | SDK sends data when build is DEBUG | SDK sends data when build is RELEASE |
| **Team permissions** | All team members have full feature access | Restricted by role to prevent accidental changes |
| **Data processing** | Faster — helps speed up initial integration testing | Standard |

<Info>
  Apart from Team management, Billing, and your Profile settings, all other settings are separate for the Test and Live environments and available in the sidebar.
</Info>

When you begin integrating any MoEngage SDKs, all your testing of user data or events data is visible in the Test environment. This enables you to test your integration and verify data before you go live or release your app or web.

For more information, refer to [Test Users](/user-guide/getting-started/integration/test-users).

# Workspaces

A workspace is the core unit in MoEngage — it holds your campaigns, user data, analytics, and settings for a single app or business unit. You can have multiple workspaces under one login.

## Switch Between Workspaces

Click the workspace arrow in the top navigation bar to see all workspaces you have access to, then click a workspace name to switch to it.

![Workspace switcher](/images/moengage_f35b96.png)

<Info>
  - You'll only see workspaces you've been invited to. If you're invited to only one workspace, the switcher won't appear.
  - Switching workspaces may log you out if the two workspaces use different login methods or have different 2-step verification settings. For more information, refer to [Automatic Logout](/user-guide/settings/account/security/automatic-logout).
</Info>

## Add a New Workspace

1. Click the workspace arrow and then click **\+ Workspace**. 
2. Enter a workspace name and click **Add**.

<Info>
  Workspace names cannot be changed after creation. Workspaces cannot be deleted.
</Info>

For workspace configuration — time zone, conversion goals, API keys, and more — refer to [Workspace Settings](/user-guide/settings/account/general-settings/workspace-settings).

# Configure Workspace Language

The MoEngage workspace language is set to English by default, but you can change it to any language supported by MoEngage.

<Info>
  MoEngage currently supports Japanese and Brazilian Portuguese.
</Info>

<Note>
  This is an Early Access feature. To enable it for your workspace, contact your CSM or [raise a support ticket](/user-guide/contact-support/contact-support/raise-a-support-ticket-through-moengage-dashboard).
</Note>

Perform the following steps to configure the workspace language:

1. Navigate to **My profile** \> **Preferences** \> **Language preferences**. <img alt="Language preferences navigation" src="/images/moengage_811d12.png" />
2. In the **Workspace language** list, select your preferred language. <img alt="Workspace language selection" src="/images/moengage_7830b8.png" />

Your workspace switches to the selected language.

# Set Up the Dashboard

Configure your workspace by clicking **Settings** in the left navigation sidebar.

![Settings menu in left navigation sidebar](/images/moengage_0a79bb.png)
