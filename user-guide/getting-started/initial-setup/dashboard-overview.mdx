MoEngage Dashboard provides quick access to:

- Create user segments
- Send targeted campaigns for your app or website
- Discover exciting analytics such as uninstalls, acquisitions, retention cohorts, and insights through campaign performance, user conversion data, and more

![dashboard.png](/images/moengage_358a60.png)

# Navigating the Dashboard

The left navigation sidebar gives you access to the main sections of MoEngage:

- **Campaigns & Channels** — Create and manage campaigns across push, email, SMS, in-app, and other channels.
- **Flows** — Build automated multi-step journeys for your users.
- **Analyze** — Access analytics, cohorts, funnels, and campaign performance reports.
- **Segment** — Create and manage user segments.
- **Settings** — Configure your workspace, channels, integrations, and team access.

At the top right you'll find:

- Your **profile icon** — access My Profile, switch appearance, and manage your account settings.
- The **workspace switcher** — switch between workspaces or add a new one.
- **Need Help** — access documentation, videos, and support.

## Live and Test Environments

Every MoEngage workspace has two environments: **Test** and **Live**. You switch between them using the toggle on the top navigation bar.

![Live and Test environment toggle](/images/moengage_2901a7.png)

Both environments share the same Workspace ID (found under **Settings** \> **Account** \> **General settings**), but all other settings are independent.

|  | Test | Live |
| --- | --- | --- |
| **Purpose** | Integration, debugging, and QA | Production user data and campaigns |
| **Data routing** | SDK sends data when build is DEBUG | SDK sends data when build is RELEASE |
| **Team permissions** | All team members have full feature access | Restricted by role to prevent accidental changes |
| **Data processing** | Faster — helps speed up initial integration testing | Standard |

<Info>
  Team management, billing, and profile settings are shared between Test and Live environments.
</Info>

When you first log in, your workspace is in the Test environment. Complete your SDK integration and verify your data in Test before switching to Live.

For more information, refer to [Test Users](/user-guide/getting-started/integration/test-users).

## Workspaces

A workspace is the core unit in MoEngage — it holds your campaigns, user data, analytics, and settings for a single app or business unit. You can have multiple workspaces under one login.

**Switch between workspaces**

Click the workspace arrow in the top navigation bar to see all workspaces you have access to. Click a workspace name to switch to it.

![Workspace switcher](/images/moengage_f35b96.png)

<Info>
  You'll only see workspaces you've been invited to. If you're invited to only one workspace, the switcher won't appear.
</Info>

<Info>
  Switching workspaces may log you out if the two workspaces use different login methods or have different 2-step verification settings. For more information, refer to [Automatic Logout](/user-guide/settings/account/security/automatic-logout).
</Info>

**Add a new workspace**

Click the workspace arrow and then click **\+ Workspace**. Enter a workspace name and click **Add**.

<Info>
  Workspace names cannot be changed after creation. Workspaces cannot be deleted.
</Info>

For workspace configuration — time zone, conversion goals, API keys, and more — refer to [Workspace Settings](/user-guide/settings/account/general-settings/workspace-settings).
