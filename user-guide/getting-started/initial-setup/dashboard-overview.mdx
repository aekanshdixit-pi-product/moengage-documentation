MoEngage Dashboard provides quick access to:

- Create user segments
- Send targeted campaigns for your app or website
- Discover exciting analytics such as uninstalls, acquisitions, retention cohorts, and insights through campaign performance, user conversion data, and more

![Dashboard overview](/images/moengage_358a60.png)

# Navigating the Dashboard

The left navigation sidebar gives you access to the main sections of MoEngage:

- **Campaigns & Channels** — Create and manage campaigns across push, email, SMS, in-app, and other channels.
- **Flows** — Build automated multi-step journeys for your users.
- **Analyze** — Access analytics, cohorts, funnels, and campaign performance reports.
- **Segment** — Create and manage user segments.
- **Settings** — Configure your workspace, channels, integrations, and team access.

At the top right you'll find:

- Your **profile icon** — access My Profile, switch appearance, and manage your workspace settings.
- The **workspace switcher** — switch between workspaces or add a new one.
- **Need Help** — access documentation, videos, and support.

# First-Time Setup

The first time when you log in to MoEngage, the Dashboard is available as a Test environment. You can integrate MoEngage with your app or web and get user or event data.

Make your environment Live after integration or tracking of events is complete and your app is released or your website is live. For more information on the test and live environments, refer to [Live and Test Environments](#live-and-test-environments).

# Live and Test Environments

Every MoEngage workspace has two environments: **Test** and **Live**. You switch between them using the toggle on the top navigation bar.

![Live and Test environment toggle](/images/moengage_2901a7.png)

Both environments share the same Workspace ID (found under **Settings** \> **Account** \> **General settings**), but all other settings are independent.

|  | Test | Live |
| --- | --- | --- |
| **Purpose** | Integration, debugging, and QA | Production user data and campaigns |
| **Data routing** | SDK sends data when build is DEBUG | SDK sends data when build is RELEASE |
| **Team permissions** | All team members have full feature access | Restricted by role to prevent accidental changes |
| **Data processing** | Faster — helps speed up initial integration testing | Standard |

<Info>
  Apart from Team management, Billing, and your Profile settings, all other settings are separate for the Test and Live environments and available in the sidebar.
</Info>

When you begin integrating any MoEngage SDKs, all your testing of user data or events data is visible in the Test environment. This enables you to test your integration and verify data before you go live or release your app or web.

For more information, refer to [Test Users](/user-guide/getting-started/integration/test-users).

# Workspaces

A workspace is the core unit in MoEngage — it holds your campaigns, user data, analytics, and settings for a single app or business unit. You can have multiple workspaces under one login.

**Switch between workspaces**

Click the workspace arrow in the top navigation bar to see all workspaces you have access to, then click a workspace name to switch to it.

![Workspace switcher](/images/moengage_f35b96.png)

<Info>
  You'll only see workspaces you've been invited to. If you're invited to only one workspace, the switcher won't appear.
</Info>

<Info>
  Switching workspaces may log you out if the two workspaces use different login methods or have different 2-step verification settings. For more information, refer to [Automatic Logout](/user-guide/settings/account/security/automatic-logout).
</Info>

**Add a new workspace**

Click the workspace arrow and then click **\+ Workspace**. Enter a workspace name and click **Add**.

<Info>
  Workspace names cannot be changed after creation. Workspaces cannot be deleted.
</Info>

For workspace configuration — time zone, conversion goals, API keys, and more — refer to [Workspace Settings](/user-guide/settings/account/general-settings/workspace-settings).

# Configure Workspace Language

The MoEngage workspace language is set to English by default, but you can change it to any language supported by MoEngage.

<Info>
  MoEngage currently supports Japanese, Brazilian Portuguese, and will soon support German.
</Info>

<Note>
  This is an Early Access feature. To enable it for your workspace, contact your CSM or [raise a support ticket](/user-guide/contact-support/contact-support/raise-a-support-ticket-through-moengage-dashboard).
</Note>

Perform the following steps to configure the workspace language:

1. Navigate to **My profile** \> **Preferences** \> **Language preferences**. <img alt="Language preferences navigation" src="/images/moengage_811d12.png" />
2. In the **Workspace language** list, select your preferred language. <img alt="Workspace language selection" src="/images/moengage_7830b8.png" />

Your workspace switches to the selected language.

# Set Up the Dashboard

Configure the dashboard by clicking the **Settings** menu available on the left navigation sidebar as illustrated in the following images.

<img alt="Settings menu in left navigation sidebar" src="/images/moengage_0a79bb.png" />

You can configure the following settings:

- [App](#app)
- [Channel](#channel)
- [Account](#account)

## App

App settings help in setting the time zone, conversion goal settings, API settings, and analytics settings.

### General

In General settings, you can:

- Find your Workspace ID for integration.
- Set the time zone.
- Update app conversion goal and attribute to see insights on app conversions. Configured attributes are displayed as Key Metrics on the dashboard. For more information, refer to [Key Metrics](/user-guide/analyze/dashboards/key-metrics-old-version).
- Associate or add new locales.

### APIs

In APIs settings, you can:

- Get data API settings such as Data API ID and keys.
- View Transaction Push or Report settings such as API ID and API secret.
- Update, add new, or test content API settings such as Name and URL.

For more information, refer to [MoEngage APIs](https://www.moengage.com/docs/api/introduction).

### Analytics

In Analytics settings, you can configure the following settings:

- Acquisition. For more information, refer to [Acquisition](/user-guide/analyze/analytics/acquisition).
- Uninstalls. For more information, refer to [Uninstall](/user-guide/analyze/analytics/uninstall).
- Reports.

### Control Groups

In Control Groups settings, you can configure the settings for the global control group.

For more information, refer to [Global Control Group](/user-guide/settings/advanced-settings/global-control-group).

## Channel

### Push

You can configure the following settings:

- Web Push. For more information, refer to [Configure Web Push Settings](/user-guide/campaigns-and-channels/web-push/getting-started-with-web-push/configure-web-push-settings).
- Mobile Push.
- Frequency capping, push throttling, and do not disturb the setting. For more information, refer to [Frequency Capping](/user-guide/settings/channels/delivery-controls/frequency-capping), [DND](/user-guide/settings/channels/delivery-controls/do-not-disturb), and [Push Throttling](/user-guide/settings/channels/delivery-controls/throttling).

Based on the platform that your app or website supports and where you want to send push notifications, enter and upload the respective settings under Android, iOS, and Web Push settings.

You can update the settings during SDK integration. For more information on how to update the settings, refer to [SDK integration](/developer-guide/introduction).

### In-App NATIV

You can configure the delay between In-App campaigns.

### Onsite Messaging

You can configure the delay between On-Site messages from different campaigns.

### SMS Connector

You can configure the following settings:

- SMS Connector configuration. For more information, refer to [Configure SMS Connectors](/user-guide/campaigns-and-channels/sms-mms-and-rcs/sms/getting-started-with-sms/configure-an-sms-sender).
- SMS General settings. For more information, refer to [Configure SMS](/user-guide/campaigns-and-channels/sms-mms-and-rcs/other-settings/configure-sms-mms-and-rcs-general-settings).
- SMS Frequency Capping (FC) and Do Not Disturb (DND) setting. For more information, refer to [Frequency Capping](/user-guide/settings/channels/delivery-controls/frequency-capping) and [DND](/user-guide/settings/channels/delivery-controls/do-not-disturb).

### Emails

You must configure Email SMTP settings on the dashboard for Email campaigns. You can configure the SMTP attributes depending on your Email Service provider. For more information, refer to [Email Settings](/user-guide/campaigns-and-channels/email/getting-started-with-email/configure-email-channel).

### Cards

You can configure the following settings:

- URL
- Security Headers

### Preference Management

You can configure opted-out users from future campaigns of SMS, Push, and Email.

## Account

For more information, refer to [Account Management](/user-guide/settings/account).

### Team Management

You can invite team members and change the roles of existing team members. For more information, refer to [Access Roles](/user-guide/settings/account/team-management/access-roles).

### Billing

You can add the payment methods and access the billing history for your MoEngage workspace.
