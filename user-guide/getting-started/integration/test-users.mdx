---
title: "Test Users"
description: "Mark users as test users in MoEngage to validate SDK integration and data tracking in near real time. Organize test groups and review device logs."
---

The _Test Users_ feature allows marketers to build and organize test groups and provide insights into the SDK or API logs available from a test device during SDK integration testing. Once a user is marked as a test user, their data will be processed in near real time to allow you to test your data tracking in Moengage. You can view the data tracked for test users in user profile and recent events.

You can mark up to 1000 users as test users.

# Steps

## Add new test users

1. Go to Test and Debug at the bottom of your sidebar

![Screenshot\_2022-03-07\_at\_9.15.24\_AM.png](/images/moengage_d53ed7.png)

2. Click "\+ Test user " on the top right to mark users as test users.

![Screenshot\_2022-03-07\_at\_9.24.55\_AM.png](/images/moengage_38a5b9.png)

3. You can either mark all the users in the [custom segment](/user-guide/segment/segment-operations/custom-segments) as Test users or select one or more specific users.

![Screenshot\_2022-03-07\_at\_9.25.47\_AM.png](/images/moengage_a30341.png)

3. a) **Select custom segment**\
   i) If you want to mark all users from a custom segment, click on the "Select custom segment" card.
   ii) Select your desired custom segment from the drop-down.
   iii)You can also run a query to see the count of the users in the selected segment.
   iv) Click on _Mark as Test_ to mark these users as test users.

Note: Anonymous users in the custom segment will not get added as test users since they do not have Client ID / UID. You can add anonymous users as test users via GAID

![Screenshot\_2022-03-07\_at\_9.29.21\_AM.png](/images/moengage_ef8a5c.png)

**3. b) Select users**

i) Click the "Select users" card if you want to mark some selected users as test users.

ii) You can select one of the user identifiers from the drop-down.

iii) Enter user detail or a comma-separated list in case of multiple users.

iv) Click on _Mark as Test_ to mark these users as test users.

Note:

1. When we add via **Client id**, the users are added as "Test User", thus their device ID is not shown in the Test Users dashboard ( where all test users are displayed)
2. When we add via **GAID**, the devices are added as "Test Device". On the test users' dashboard, both users and device details are shown.

![Screenshot\_2022-03-07\_at\_9.33.07\_AM.png](/images/moengage_0a374f.png)

4. Once you click on 'Mark as test', users in the selected segments or selected users would be marked as Test users. And you will see the user details in the slide-out.

![Screenshot\_2022-03-07\_at\_9.41.31\_AM.png](/images/moengage_b4e286.png)

Note - In case some of the users failed, you will see the failure details in the same slide-out. <img alt="Screenshot_2022-03-07_at_9.50.17_AM.png" src="/images/moengage_dbf4c5.png" />

## Mark test users from the user profile

If you are already on a user profile and want to mark that user as a Test user, click _mark as a test user_ on the top right.

Note: Only registered users can be added as a "Test User" from the user profile page, not the anonymous user.

![Screenshot\_2022-03-07\_at\_9.45.16\_AM.png](/images/moengage_139a76.png)

![Screenshot\_2022-03-07\_at\_9.48.48\_AM.png](/images/moengage_e43fbe.png)

## Access test users

1. Go to Test and Debug at the bottom of your sidebar
2. The page will show the details of all the test users.

![Screenshot\_2022-03-07\_at\_9.21.35\_AM.png](/images/moengage_95d2a6.png)

### Access user profile

You can click on the MoEngage user id to go to the user profile page.

### View recent events

1. Once you select one or more users, you will see an option (just below the search bar) to see the recent events for the selected users. Click _View recent events_.

Note: We can select up to 5 users and view recent events of those users.

![Screenshot\_2022-03-07\_at\_9.39.04\_AM.png](/images/moengage_077b19.png)

2. The slide-out will show the recent 10 events for the selected users. Note: In the slide-out, we show a maximum of 10 events( of the last 24 hours) for each user.

![Screenshot\_2022-03-07\_at\_9.40.08\_AM.png](/images/moengage_094910.png)
