Custom segments enable you to save user segment queries and provide a name for this query for ease of use. Now this custom segment can be used on multiple pages in the MoEngage dashboard instead of entering the entire query and creating the segment again.

## Creating Custom Segment

In the MoEngage dashboard custom segments can be created in 3 ways -

- Navigate to Segment \> Create Segment, after entering the query, click on the “Create” button and select Create Custom Segment. Enter the custom segment name and save the custom segment. Additionally, in the Recent Query section for a selected query click on the 'Create Custom Segment' icon. Enter the custom segment name and save the custom segment. These sections create a [dynamic custom segment](https://help.moengage.com/hc/en-us/articles/206169646#h_92999542-9291-494d-a673-4465719af0c8). This is explained in the [Create User Segments](https://help.moengage.com/hc/en-us/articles/206169646-Create-User-Segments) article.
- Custom segments can be created while importing users. The users in the segment remain fixed and hence this is a static custom segment. Click [here](https://help.moengage.com/hc/en-us/articles/19192630201620-Import-User-Data-Using-CSV) to know more.
- Custom segments can also be created from Behaviour analysis, Funnel analysis and RFM Segments. These sections create a dynamic custom segment. [Click here to know more](https://help.moengage.com/hc/en-us/articles/360037365132#h_f50fe717-0aed-4bc6-8da1-76facdd6787c).

## Using Custom Segment

On the MoEngage dashboard custom segments can be used in multiple different ways -

- Custom segments can be utilized to create a complex user segment. A custom segment can be used in creating user segments along with user properties and user activities.
- A custom segment can be used while creating other custom segments, hence complex nesting of custom segments is allowed. A custom segment can have up to 3 levels of recursion/nesting i.e. Custom Segment A definition can include Segment B which in turn can include Segment C which in turn can include Segment D. [File Segments](https://help.moengage.com/hc/en-us/articles/207606916) are not counted towards the nesting limit.
- While creating/defining a new custom segment, up to 10 existing custom segments can be used in the new custom segment. [File Segments](https://help.moengage.com/hc/en-us/articles/207606916) are not counted towards this limit.
- Custom Segments can be used to create a target audience while creating a campaign for different channels such as Push, Email, SMS, In-App, Facebook, Flows, etc.

### Custom Segment Analysis

Users of a custom segment can be analyzed on the MoEngage Analytics pages - [Behavior](https://help.moengage.com/hc/en-us/articles/34174962989204-Behavior-Overview), [Funnel](https://help.moengage.com/hc/en-us/articles/34174988942740-Funnels-Overview), [Retention](https://help.moengage.com/hc/en-us/articles/35438085954196-Retention-Overview), [Session & Source](https://help.moengage.com/hc/en-us/articles/35724347745812-Session-and-Source-Overview), [User Paths](https://help.moengage.com/hc/en-us/articles/36048677165332-User-Paths-Overview) & [RFM Segments](https://help.moengage.com/hc/en-us/articles/360037365132-RFM-Segments). By default, all users are analyzed for the above-mentioned analysis. To analyze users of a specific custom segment, select the desired custom segment from the custom segment dropdown. Provide the other required parameter for the respective analysis and hit the 'Apply' button. This provides the desired analysis for a respective custom segment.

## Archive Segment

Navigate to Segment \> All Segments, enter the custom segment name that you want to Archive, and search. For the selected custom segment click on Actions -\> Archive to archive the selected custom segment.

<img alt="Custom_segment_archive.png" src="/images/moengage_052ab2.png" />

For custom segments -

1. Archived custom segments are not available in the custom segments dropdown on the MoEngage dashboard. If an archived [Rule/Filter-based Segment](https://help.moengage.com/hc/en-us/articles/4416971195284-Types-of-Segments#01H8FJPA7DK8GDXG8B3000R6BG) is used in the campaign, the segment gets calculated based on the rule/filters. If an archived [File Segment](https://help.moengage.com/hc/en-us/articles/207606916) is used in a campaign, the segment will return 0 users.
2. Custom segments that have not run (query not executed) in the last 60 days are automatically archived.
3. All the users from the archived [File Custom Segment](https://help.moengage.com/hc/en-us/articles/207606916-Import-User-Segments) are dropped after 30 days of archival. Hence after 30 days of archival, the archived custom segment returns 0 users.
4. You can mark an expiry time for the file custom segments using [Custom File segment API](https://developers.moengage.com/hc/en-us/articles/4405027805844-Custom-File-Segment-API). After the expiry time, all existing users from the custom segment are dropped, and the custom segment is archived.
5. Custom segments cannot have the same name, even when one of the segments is archived.

Navigate to Segment \> All Segments, and click on the 'Archived Segments' toggle to view a list of all archived segments. Search the custom segment that you want to unarchive from the list. For the selected custom segment click on the 'Unarchive' action to unarchive the segment.

<Accordion title="Why Is the Custom Segment Count Less Than the Import User Count?">

## Problem

When you upload a set of users through the **Import users** feature, you may notice that the count of the corresponding custom segment is less than the total number of users uploaded.

</Accordion>
## Explanation

The following reasons typically cause this discrepancy:

- **Duplicate users in the file:** If there are duplicate user IDs in the uploaded file, MoEngage will reject or ignore these duplicates. As a result, the total number of users in the dashboard will be less than the number of users uploaded.
- **Users are not part of the dashboard:** This scenario occurs when you attempt to update existing users through **Import users**. If some users you tried to upload are not included in the custom segment, search for those users on the Create segment page. If you cannot find them, those users are not yet part of the dashboard.

For more information, read [Import User Data Using CSV](https://help.moengage.com/hc/en-us/articles/19192630201620-Import-User-Data-Using-CSV#h_01HB83FB354B1V8VQA9AZF3AQY) and [Custom Segments](https://help.moengage.com/hc/en-us/articles/206528613-Custom-Segments#h_01H9SSGVA9ZRXB5M3RQ3N4P40G).

<Accordion title="Why Do Date Range Filters Fail Sometimes for Custom Attributes in Segmentation?">

## Problem

The data type of a custom attribute changes automatically, which alters the filters available during segmentation.

For example, if the custom attribute **Subscription Date** is ingested with the **datetime** data type, it should allow date-based filters such as **before**, **after**, or **between**. However, sometimes, the **datetime** data type may change to **string**. This results in string-based filters like **does not contain**, **contains**, **is empty**, or **not empty** appearing instead of date filters.

</Accordion>
## Solution

Perform the following steps:

1. **Verify the data type of the custom attribute**: Check if the data type is incorrectly set to **string** instead of **datetime.** If so, change it to **datetime** to avoid causing issues when creating segments based on date ranges.
2. **Report the issue to the admin**: Inform the admin that multiple data types are being ingested for the same attribute. Request them to pin the correct data type (datetime).
3. **Create a new custom attribute**: Create a new attribute with a name similar to the existing one to track the date value.
4. **Import the values with the correct data type**: When importing data for the new attribute, ensure the datetime data type is used.

<Info>
After you create the new attribute and import the date values using the correct data type, you can create segments within a date range based on the custom attribute.
</Info>

<Accordion title="Why Does the Error 'Custom Segment Name Already Taken' Occur?">

## Problem

While creating a custom segment, the following error occurs:

The custom segment name that you have provided is already in use by some other team. Please use a different custom segment name.

</Accordion>
## Solution

Perform the following steps:

- Check if the custom segment is in the Archived list
- Verify if the segment has been archived for more than 180 days.
- Custom segments that have not been run (query not executed) in the last 60 days are automatically archived.
- Custom segments cannot share the same name, even if one is archived.

<Info>
The segmentation backend maintains segment referrals and prevents the creation of segments with the same name, even if the original is archived and not visible in the dashboard.
</Info>
