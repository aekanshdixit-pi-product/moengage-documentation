## Overview

Email Template APIs enable marketers to define, reuse, update, and manage email templates created outside the MoEngage ecosystem with ease. The templates created and updated using these APIs are supported for the Custom HTML Editor (Froala Editor) and not the Drag and Drop Editor.

## Email Template API V1 Endpoints

- [Create Email Template](https://moengage.mintlify.app/api-reference/email-templates/create-email-template)
- [Update a Specific Email Template](https://moengage.mintlify.app/api-reference/email-templates/update-specific-template)
- [Bulk Create/Update Email Template](https://moengage.mintlify.app/api-reference/email-templates/bulk-createupdate-templates)
- [Get a Specific Email Template](https://moengage.mintlify.app/api-reference/email-templates/get-specific-template)
- [Get all Email Templates](https://moengage.mintlify.app/api-reference/email-templates/get-all-templates)

### Update a Specific Email Template

<Accordion title="If I pass don't pass a few fields in the request and the created template has them already, will they be retained post the update?" icon="sparkles">
  The update request will overwrite the existing template with the template shared in the update request. If the update request does not have some of the fields that the created template had, the updated template will not have them too. For example, if you do not pass any attachments in the update request, the updated template will not have any attachments, even if it had attachments while it was created.
</Accordion>

### Bulk Create/Update Email Template

<Accordion title="How is the create request differentiated from an update template request?" icon="sparkles">
  In the case of an update, the template to be updated is identified using its template id. It is mandatory to pass the id field in the required for an update template request. Otherwise, it will be treated as a create template request.
</Accordion>

### Get a Specific Email Template

<Accordion title="What happens when you search for a deleted template?" icon="sparkles">
  The error description contains the following message in such a case: “_Template for given template id is archived_”.
</Accordion>
