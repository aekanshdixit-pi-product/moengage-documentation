## Overview

The MoEngage Coupon Management API allows you to organize and distribute unique, single-use coupon codes at scale. Use these endpoints to automate the lifecycle of coupon lists—from creation and file uploads to activation, archival, and detailed usage reporting.

<Info>
  Authentication is handled via **Basic Auth**. Use your **Workspace ID** as the username and your **API Key** as the password.
</Info>

## Coupon API Endpoints

The Coupon Management API is categorized into three functional areas:

### Coupon Lists (Metadata & Status)

- [Create a Coupon List](#operation/createCouponList) - Define a new category for unique codes.
- [Fetch All Coupon Lists](#operation/fetchAllCouponLists) - Retrieve a list of active or archived coupon containers.
- [Fetch Coupon List Details](#operation/fetchCouponList) - Get real-time counts of available vs. total coupons.
- [Update a Coupon List](#operation/updateCouponList) - Modify names, expiry dates, or alert settings.
- [Activate Coupon List](#operation/activateCouponList) - Reactivate archived lists with a new expiry date.
- [Archive a Coupon List](#operation/archiveCouponList) - Transition a list to inactive and delete associated codes.

### Coupon Files (Inventory Management)

- [Upload Coupon File](#operation/uploadCouponFile) - Populate a list via a hosted file URL.
- [Fetch All Coupon Files](#operation/fetchAllCouponFiles) - Track the status of all uploaded batches.
- [Fetch a Coupon File](#operation/fetchCouponFile) - Check the processing status (e.g., PENDING) of a specific file.
- [Delete a Coupon File](#operation/deleteCouponFile) - Remove specific batches (useful for correcting upload errors).

### Reports

- [Generate Usage Report](#operation/generateUsageReport) - Request a CSV report sent via email detailing which user received which code.

## Rate Limits & Constraints

| Operation Type          | Daily Limit           | Burst Limit        |
| :---------------------- | :-------------------- | :----------------- |
| **Create Coupon List**  | 100 lists / day       | -                  |
| **Fetch/Search Lists**  | 10,000 requests / day | -                  |
| **Upload Coupon Files** | 50 files / day        | 5 files / minute   |
| **Delete Coupon Files** | 50 files / day        | 5 files / minute   |
| **Generate Reports**    | 50 reports / day      | 5 reports / minute |

<Note>
  **Payload Limits:**

  - Manual uploads are limited to **64 MB**.
  - URL-based uploads are limited to **100 MB**.
  - Each coupon list can hold a maximum of **100 million** codes.
</Note>

## Lifecycle Notes

<Note>
  **Initial Setup:** Creating a coupon list only creates the "container." You must use the **Upload Coupon File API** to add actual codes before the list can be used in campaigns.

  **Archival Impact:** When you archive a list, all coupon codes within that list are **permanently deleted**. Campaigns relying on this list will immediately stop being able to allocate codes.
</Note>

## FAQs

### Coupon List Management

<AccordionGroup>
  <Accordion title="Can I reactivate an expired coupon list?">
    Yes. You can use the **Update a Coupon List API** or the **Activate API** to set a future expiry date. If the list has a future date and contains available coupons, it will return to an **ACTIVE** status.
  </Accordion>
  <Accordion title="How do I get notified when coupons are running low?">
    You can configure `alert_conditions` during creation or via an update. Set a `threshold_count` under `coupon_shortage_alert`, and MoEngage will email the specified `email_alert_subscribers`.
  </Accordion>
  <Accordion title="What is the personalization snippet?">
    Each coupon list provides a `personalization_snippet` (e.g., `{{Coupons.CouponListName[label_name]}}`). Copy this snippet into your Push or Email campaign templates to dynamically insert a unique code for each user.
  </Accordion>
</AccordionGroup>

### File Uploads & Processing

<AccordionGroup>
  <Accordion title="Does the Upload API process codes instantly?">
    No. Uploads are processed asynchronously. The initial response will show a status of **PENDING**. You should use a `callback_url` to receive a notification once processing is complete, or poll the **Fetch Coupon File API**.
  </Accordion>
  <Accordion title="What happens if I upload duplicate codes in a new file?">
    The system manages unique code constraints. To ensure accuracy, you can use the **Fetch Coupon File API** after processing to see exactly how many `added_coupons` were successfully ingested from that specific file.
  </Accordion>
  <Accordion title="Can I use a password-protected URL for file uploads?">
    Yes. The Upload API supports `file_url_auth_username` and `file_url_auth_password` for files hosted behind Basic Authentication.
  </Accordion>
</AccordionGroup>

### Reporting

<AccordionGroup>
  <Accordion title="How far back can I generate a usage report?">
    You can generate reports from the `start_date` of the coupon list's creation up to the current date. Note that the `end_date` cannot be in the future.
  </Accordion>
</AccordionGroup>
