> Use the MoEngage Campaigns Draft API to iteratively author, patch, validate, test, and publish your Push and Email campaigns programmatically.

The Campaigns Draft API (V5) empowers you to build campaigns step-by-step rather than requiring a massive, complete payload in a single request. By creating a base draft, you can systematically patch in audience segments, trigger conditions, and rich content. Once the draft is fully assembled, you can validate its structure, test the delivery, and confidently publish it to your live audience.

Additionally, you must include the `X-MOE-Tenant-ID` header (your Workspace ID) and an `Idempotency-Key` (UUID v4) on all POST and PATCH requests to ensure safe retries.


## Endpoints

The Campaigns Draft API consists of the following endpoints to manage your campaign lifecycle:

* [Create Campaign Draft](): Initializes a new Push or Email campaign in a `DRAFT` state using minimal required fields.
* [Get Campaign](): Retrieves a single campaign in its full deparsed form, displaying its current state and configuration.
* [Patch Campaign Draft](): Updates specific components of an existing draft or triggers the formal `PUBLISH` action to launch the campaign.
* [Validate Campaign Draft](): Safely runs a full publish-time validation check without modifying the draft or changing its state.
* [Update Campaign Status](): Applies lifecycle transitions (STOP, PAUSE, RESUME) to campaigns that have already been published.
* [Search Campaign Drafts](): Searches for campaigns using granular filters, with the ability to explicitly include or exclude drafts.
* [Get Campaign Meta](): Fetches lightweight metadata, including daily cached reachability estimates for scheduled campaigns.
* [Test Campaign](): Sends a test push or email using an inline payload or by referencing a saved draft ID.
