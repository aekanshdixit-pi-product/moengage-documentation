> Use the MoEngage Campaigns Draft API to iteratively author, patch, validate, test, and publish your Push and Email campaigns programmatically.

The Campaigns Draft API (V5) empowers you to build campaigns step-by-step rather than requiring a massive, complete payload in a single request. By creating a base draft, you can systematically patch in audience segments, trigger conditions, and rich content. Once the draft is fully assembled, you can validate its structure, test the delivery, and confidently publish it to your live audience.

Additionally, you must include the `X-MOE-Tenant-ID` header (your Workspace ID) and an `Idempotency-Key` (UUID v4) on all POST and PATCH requests to ensure safe retries.

## Campaign versioning

Campaign Versioning (CC-7864) is **opt-in** per workspace. When it is turned on, publishing changes to a campaign that is already live creates a **new campaign document** with an incremented `version_number`. The **`campaign_id`** returned in API responses is the **canonical** identifier: it stays the same across versions so you can correlate drafts, search results, and analytics. Each version still has its own raw **`id`** (24-character ObjectId).

For UI-focused behavior and version history in the dashboard, see [Campaign versioning](/user-guide/campaigns-and-channels/campaign-management-and-reports/campaign-versioning).

## Endpoints

The Campaigns Draft API consists of the following endpoints to manage your campaign lifecycle:

* [Create Campaign Draft](/api/create-campaigns-draft/create-campaign-draft): Initializes a new Push or Email campaign in a `DRAFT` state using minimal required fields.
* [Get Campaign](/api/get-campaign-details-draft/get-campaign): Retrieves a single campaign in its full deparsed form, displaying its current state and configuration.
* [Patch Campaign Draft](/api/update-campaigns-draft/patch-campaign-draft): Updates specific components of an existing draft or triggers the formal `PUBLISH` action to launch the campaign.
* [Validate Campaign Draft](/api/create-campaigns-draft/validate-campaign-draft): Safely runs a full publish-time validation check without modifying the draft or changing its state.
* [Update Campaign Status](/api/update-campaigns-draft/update-campaign-status): Applies lifecycle transitions (STOP, PAUSE, RESUME) to campaigns that have already been published.
* [Search Campaign Drafts](/api/get-campaign-details-draft/search-campaign-draft): Searches for campaigns using granular filters, with the ability to explicitly include or exclude drafts.
* [Get Campaign Meta](/api/get-campaign-details-draft/get-campaign-meta): Fetches lightweight metadata, including daily cached reachability estimates for scheduled campaigns.
* [Test Campaign](/api/test-campaigns-draft/test-campaign): Sends a test push or email using an inline payload or by referencing a saved draft ID.
