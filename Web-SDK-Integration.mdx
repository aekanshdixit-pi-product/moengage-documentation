# Overview

You can integrate the Web SDK for any of the supported frameworks (CDN, NPM, GTM, SPA, AMP, Flutter, and Shopify) using one of the following two methods:

* **Script-based Initialization (Recommended):** Use the form-based interface to generate a validated code snippet or access module-specific configurations. For more information, refer to [Script-based Initialization (Recommended)](#option-1-script-based-initialization-recommended).
* **Manual Integration:** Manually configure the SDK parameters and initialization code. For more information, refer to [Option 2: Manual Integration](#manual-approach).

<Info>
  **Prerequisites**

  You must have your **Workspace ID (App ID)** and **Data Center**. You can retrieve these details by navigating to `Settings > App > General` on the Dashboard.
</Info>

---

# Option 1: Script-based Initialization (Recommended)

To ensure a reliable and streamlined setup, we recommend using the **Script-based initialization** method. This method generates a validated JavaScript snippet, eliminating potential syntax errors and automatically including required flags based on the application type.

<Info>
  **Prerequisites**

  * **Workspace Details:** Ensure you have your **Workspace ID (App ID)** and **Data Center** (found in `Settings > App > General` on the Dashboard).
  * **Service Worker (For Web Push):** If enabling Web Push, create a file named `serviceworker.js` in your **root directory**.
  * **Whitelisting (CSP):** If your site uses a Content Security Policy, whitelist the necessary MoEngage URLs. For more information, refer [here](https://developers.moengage.com/hc/en-us/articles/360060713252-Web-SDK-Integration#h_01J6V629GF3CCYAJRX4ADQ5RGP).
</Info>

Follow these steps to generate your initialization script:

<Steps>
  <Step title="Navigate to the tool">
    Navigate to the [Web SDK Initialization Website](https://app-cdn.moengage.com/sdk/integration/config/index.html).
  </Step>
  <Step title="Configure your application">
    Configure the values based on your application requirements. Refer to the [Configuration Parameters](#configuration-parameters) table below for detailed descriptions of each field.
  </Step>
  <Step title="Generate Code">
    Click **Generate Code** at the bottom of the form to create your validated JavaScript snippet.
  </Step>
  <Step title="Implement the Generated Code">
    Copy the code snippets displayed by the tool and paste them into your project as follows:

    * **Update Service Worker (If Web Push Enabled):** Add the generated `importScripts(...)` line to the top of your `serviceworker.js` file.
    * **HTML File Changes (If Web Personalization Enabled):** Add the generated `<link>` tags and the Personalization `<script>` tag to the top of the `<head>` section of your root HTML file.
    * **Initialize the SDK:**
        * **For CDN:** Paste the initialization script directly into the `<head>` tag.
        * **For NPM:** Paste the `import` statement and the `Moengage.initialize({...})` block into your main JavaScript file.
  </Step>
</Steps>

<Check>
  **Post-requisite**

  After integrating, verify the setup by opening your browser's Developer Tools (Network tab). Reload the page and confirm that requests are successfully reaching MoEngage endpoints (e.g., `cdn.moengage.com` or `sdk-0X.moengage.com`).
</Check>

## Configuration Parameters

The initialization method requires specific inputs to generate the correct code.

<Accordion title="View Detailed Configuration Parameters">
| Section | Field Name | Description |
| :--- | :--- | :--- |
| **Integration Method** | How will you install the SDK? | Determines the installation method (NPM/Yarn, CDN, or specialized platforms like GTM, Shopify, etc.). |
| **Core Configuration** | MoEngage Workspace ID | Unique App ID identifying your app in the MoEngage dashboard. |
| | MoEngage Data Center | Specifies the data center associated with your dashboard. |
| | MoEngage Environment | Sets data reporting to `TEST` or `LIVE`. |
| | Log Level | `0`: No logs; `1`: Critical logs/errors; `2`: Verbose debug logs. |
| | Use Latest SDK Version | Automatically targets the most recent Web SDK version. |
| **Application Profile** | SPA Support | Indicates if your website is a Single Page Application. |
| | Browser Extension | Prevents cookie storage and disables cross-subdomain user sharing if selected. |
| **Feature Configuration** | Enable Web Push | Module for push tokens and browser notifications. |
| | Enable On-Site Messaging | Module for pop-ups and interstitials. |
| | Enable Cards | Enables Notification Inbox and UI customization. |
| | Disable SDK at Init | Initializes in a disabled state; requires `Moengage.enableSdk()`. |
| **Web Push Details** | Service Worker Path | Path to your file (Default: `/serviceworker.js`). |
| **Cards & UI** | CSS Selector | The ID or Class (e.g., `#bell-icon`) that opens the inbox. |
| | Styling | Customize colors, positions, fonts, and z-index for the Inbox and Icons. |
</Accordion>

### Common Configuration Scenarios

| Environment (`env`) | Log Level (`logLevel`) | Data Destination | Console Output |
| :--- | :--- | :--- | :--- |
| `'TEST'` | `0` | Test Dashboard | Clean console (no SDK logs). |
| `'TEST'` | `1` | Test Dashboard | Critical logs and errors (Recommended). |
| `'LIVE'` | `0` | Production Dashboard | Production settings. No logs shown. |
| `'LIVE'` | `1` | Production Dashboard | Enables error logs for troubleshooting. |

# Option 2: Manual Integration

You can manually integrate the SDK by configuring the initialization parameters directly in your codebase.

### Supported Platforms
* [CDN](#method-1-moengage-cdn)
* [NPM](#method-2-using-npm)
* [Google Tag Manager (GTM)](https://developers.moengage.com/hc/en-us/articles/17390387490324-Google-Tag-Manager-GTM-Templates)
* [Single Page App (SPA) Support](https://developers.moengage.com/hc/en-us/articles/360062008652-Single-Page-App-SPA-Support)
* [Flutter SDK](https://developers.moengage.com/hc/en-us/articles/20661421128084-Getting-Started-with-Flutter-SDK)
* [Shopify App](https://developers.moengage.com/hc/en-us/articles/18058275070612-Shopify-2-0)

<Note>
  For any manual integration, ensure you specify the MoEngage Data Center and complete URL Whitelisting.
</Note>

## Method 2: Using NPM

Add the package to your project:

```bash
npm install --save @moengage/web-sdk
# or
yarn add @moengage/web-sdk
```

Initialize the SDK in your main JavaScript file:

```javascript
import moengage from "@moengage/web-sdk";

moengage.initialize({
  app_id: 'XXXXXXXXXXXXXXXX',
  env: 'LIVE',
  logLevel: 0
});

// Example usage
moengage.add_email('abc@xyz.com');
```

# Whitelisting URLs

If your website uses a Content Security Policy (CSP), ensure you whitelist the following endpoints:

* **script-src:** `https://cdn.moengage.com/`, `https://app-cdn.moengage.com/`
* **connect-src:** `https://sdk-01.moengage.com/` through `https://sdk-06.moengage.com/`
* **image-src:** `https://moe-email-campaigns.s3.amazonaws.com/`, `https://image.moengage.com/` (and DC-specific image URLs).
* **style-src / font-src:** `https://fonts.bunny.net/`

# Troubleshooting

### Not able to debug SDK?
Check that the `logLevel` is set to `1` (or `2` for verbose) while initializing.

### What is the difference between TEST & LIVE?
* `env: 'TEST'`: Routes data to your MoEngage Test dashboard (Staging).
* `env: 'LIVE'`: Routes data to your production dashboard.
