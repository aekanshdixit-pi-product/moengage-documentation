[Wootag](https://www.wootag.com/) is a signal orchestration platform that triggers personalized MoEngage campaigns based on real-time contextual data such as weather, sports scores, and stock prices. By monitoring live feeds, Wootag fires Business Events to MoEngage when specific conditions are met, enabling relevant, contextual customer engagement.

# MoEngage × Wootag

This integration enables an automated data flow from Wootag to MoEngage, triggering campaigns that you can personalize using dynamic event attributes.

## Use Cases

Integrating Wootag with MoEngage helps you address the following use cases:

- **Real-time signal monitoring**: Wootag continuously monitors live contextual signals such as temperature or sports scores.
- **Automated event triggers**: Automatically fire MoEngage Business Events when rule conditions (for example, temperature above 40°C) are met.
- **Dynamic personalization**: Use Jinja templating to inject event data such as the city name or signal values directly into campaign content.
- **Contextual lifecycle engagement**: Deliver relevant offers or alerts based on a user's current environment or interests.

## Supported Moment Categories

The following table describes the signal categories available for triggering campaigns through Wootag:

| Moment Type | Description |
| --- | --- |
| **Weather** | Triggers campaigns based on local conditions such as temperature, humidity, air quality, or rain. |
| **Sports & Stocks** | Engages users with real-time updates on live sports scores or stock price fluctuations. |
| **Crypto & Forex** | Monitors digital currency and foreign exchange rates to trigger financial alerts. |
| **Traffic & Commodities** | Tracks live traffic conditions or commodity prices (for example, metals) for contextual triggers. |

# Integration

<Info>
  **Prerequisites**

  - **MoEngage Business Events**: Your MoEngage Customer Success Manager must enable this feature before you proceed.
  - **Account access**: Ensure you have active login credentials for both the MoEngage and Wootag dashboards.
  - **Moment Rules**: You must have at least one Wootag Moment Rule (Weather, Sports, and so on) configured.
</Info>

## Step 1: Get Your MoEngage API Credentials

To get your MoEngage API credentials, perform the following steps:

1. On the left navigation menu in the MoEngage dashboard, click **Settings** \> **Account** \> **APIs**.
2. On the **APIs** page, copy the following credentials:
   - **Workspace ID** (earlier App ID): Your unique MoEngage workspace identifier.
   - **Event Key**: The Campaign/Business Events API key.
3. Find your data center code from your dashboard's URL. The following table describes the data center names and URLs:
   | Data Center Name | Dashboard URL |
   | --- | --- |
   | DC-01 | `https://dashboard-01.moengage.com` |
   | DC-02 | `https://dashboard-02.moengage.com` |
   | DC-03 | `https://dashboard-03.moengage.com` |
   | DC-04 | `https://dashboard-04.moengage.com` |
   | DC-05 | `https://dashboard-05.moengage.com` |
   | DC-06 | `https://dashboard-06.moengage.com` |
   | For more information, refer to the [Data Centers in MoEngage](/user-guide/data/key-concepts/data-centers-in-moengage) guide. |  |

## Step 2: Install the MoEngage App on Wootag

To connect MoEngage in the Wootag Apps Marketplace and enable triggers, perform the following steps:

1. Navigate to the Wootag Apps dashboard and log in.
2. Click the **Apps** icon in the sidebar, and then locate the **Moengage** tile.
3. Click **Install**. <img src="/images/wootag-install-moengage.png" alt="Wootag Apps marketplace with the MoEngage tile highlighted" />
4. Enter the following details in the credentials form:
   | Field | Required | Description |
   | --- | --- | --- |
   | **Workspace ID** | Yes | Your unique MoEngage workspace identifier. |
   | **Event Key** | Yes | The Campaign/Business Events API key. |
   | **Data Center** | Yes | Select your data center (for example, DC-01) from the list. |
5. Click **Proceed**. A success message confirms the MoEngage installation. <img src="/images/wootag-moengage-credentials.png" alt="Wootag MoEngage credentials form with Workspace ID, Event Key, and Data Center fields" />

## Step 3: Configure Wootag Moment Rules

Define the signal conditions that trigger your MoEngage campaigns:

1. Go to the **Rules** section in the left sidebar.
2. Create or edit a rule: define a rule name and select the Moment type (for example, Weather).
3. Configure triggers: select conditions such as temperature above 40°C and set the city.
4. Preview attributes: review the event data payload and Jinja syntax for personalization.
5. Copy the JSON: use the Preview modal to copy the event data JSON for campaign setup.

For more information, refer to the [Wootag platform documentation](https://www.wootag.com/).

<Warning>
  **Business Events activation**

  If you do not see Business Events in your MoEngage campaign creation flow, the feature is likely not enabled for your workspace. Contact your MoEngage Customer Success Manager to request activation before you run campaigns.
</Warning>

## Step 4: Create a Campaign in MoEngage

Wootag captures contextual live moments (such as extreme weather shifts, real-time sports updates, or market drops) and forwards them dynamically to MoEngage as trigger Business Events. After your rules are established, you can configure and launch your MoEngage campaigns using the following methods:

- **Directly from Wootag**: Trigger and provision the MoEngage campaign inside the Wootag interface using native action tools.
- **Via MoEngage**: Construct and manage the event-triggered campaign inside the MoEngage Campaign dashboard, referencing the event parameters sent from Wootag.

To configure your triggers, perform the following steps:

1. In Wootag, click the **Campaign** dropdown on your rule, and then select **MoEngage**. <img src="/images/wootag-campaign-dropdown.png" alt="Wootag rule list with the Campaign dropdown open and MoEngage selected" />
2. Click **Create MoEngage Campaign** to set up a new triggered campaign. <img src="/images/wootag-create-campaign.png" alt="Wootag MoEngage Campaign modal with the Create MoEngage Campaign option highlighted" />
3. Enter the campaign name, date range, and trigger interval to prevent over-messaging.
4. In MoEngage, go to **Campaigns** \> **Email/Push (Business Event Triggered)** to verify the campaign exists.
5. Select the appropriate Business Event (for example, Lucknow) created by Wootag.
6. In the **Content** section, use the Jinja editor to personalize messages with event data. Reference attributes using the syntax `{{ BusinessEventAttribute['name'] }}`. For example:
   ```text wrap
   Alert: {{ BusinessEventAttribute['name'] }} detected in {{ BusinessEventAttribute['source'] }}.
   ```
7. Test your setup to confirm that users receive relevant contextual messages when rules fire.
8. Preview the custom attributes payload in Wootag to confirm the JSON keys.
9. Publish the campaign, and then monitor the Business Event analytics in MoEngage.

<Info>
  **Runtime flow**

  When a rule is met (for example, temperature rises above 40°C), Wootag initiates the MoEngage [Trigger Business Event API](/api/business-events/trigger-business-event), delivering a push or email personalized with live signal data. Verify rate limits (default 200 triggers per day) with your Customer Success Manager for high-frequency signals.
</Info>
