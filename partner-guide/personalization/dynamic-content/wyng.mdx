# Introduction

[Wyng](https://www.wyng.com/) makes it easy to build interactive digital experiences (that is, quizzes, preference centers, promotions) that engage consumers at the right moments, collect preferences and other zero-party data, and personalize in real-time.

# MoEngage × Wyng

The MoEngage and Wyng integration allows you to leverage zero-party data earned via Wyng experiences to personalize interactions in MoEngage Campaigns and Flows. Wyng can also power a preference center, so consumers can control the data and preferences (including communication preferences) they share with your brand.

# Integration

<Info>
**Prerequisites**

* Ensure you have access to your Wyng account
</Info>

## Step 1: Get your MoEngage endpoint

Speak to your MoEngage Customer Success Manager to help you get a dedicated endpoint. You can define your own mapper using MoEngage Connected Sources. 

## Step 2: Set up the webhook in Wyng

Follow the steps to create webhook in Wyng. Refer to [Wyng documen](https://help.wyng.com/s/article/Send-form-entries-through-webhook-integration)t for more details. On your Wyng dashboard,

  1. Go to _More_ >> Integrations 
  2. Click Add Integrations  
![Screenshot 2025-02-14 at 3.52.52 PM.png](/images/partner_34807488534164.png)
  3. Click Webhooks  
![Screenshot 2025-02-14 at 3.44.28 PM.png](/images/partner_34807516389012.png)
  4. Enter the webhook details 

| Field | Description |
| --- | --- |
| Webhook name | Enter the webhook name, you can name it MoEngage |
| Data source | Select one your existing data sources in Wyng |
| Data destination | Select URL endpoint as the destination |
| Choose experience | Select one of the existing forms/experience data that you want to share with MoEngage |
| URL | Enter the MoEngage Endpoint provided to you by your Customer Success Manager Custom Headers** |
| ** | Add the following custom headers: | Header(s) | Value |
| --- | --- |
| Content-Type | application/json |
| Authorization | Get your Basic Token by going to any online [Basic Auth Header Generator](https://www.debugbear.com/basic-auth-header-generator) tool. Enter your Workspace ID as username and Data API Key as password. Your final token should look like this (MoEngage Dashboard >> Settings >> APIs): `Basic Qk5CR1NNNUR4eHh4eHhWQ1hSRFJEWTdSOlFsWHVoTHVaeHh4eHh4QWQrQnBYUEotNw==` Paste this as the value here. |
| Request Body | Define your request body. Add the fields that you want to send to MoEngage as user or event attributes. ![Screenshot 2025-02-14 at 3.48.27 PM.png](/images/partner_34807488535188.png) |
  
  4. Click on "Save" to add the webhook.



## Step 3: Send targeted campaigns on MoEngage

Your users and events should start showing up in MoEngage. You can now run event-triggered campaigns to your users.
