# Introduction

[Livingdocs](https://livingdocs.io/) is a modern digital content creation and publishing system in use at a wide variety of media companies. 

# MoEngage × Livingdocs

With the MoEngage and Livingdocs integration, you can edit and manage all your templates inside Livingdocs and pull them dynamically inside MoEngage campaigns to send out up-to-date articles to your customers.

# Integration

<Info>
**Prerequisites**

* You would need your [Access Tokens](https://docs.livingdocs.io/reference/public-api/get-started/#authorization) to make use of Livingdocs APIs.
</Info>

## Embed Livingdocs templates inside MoEngage Campaigns

Using MoEngage's Content APIs, you can dynamically pull the latest copy of your templates from inside Livingdocs and send them to your customers. 

### Step 1: Set up a MoEngage Content API

  1. Navigate to **Settings** > **Advanced settings** > **Content API**. Click **\+ Add content API** at the top right corner of the Content API screen.![ContentAPI_Navogation.png](/images/partner_21979775680020.png)
  2. Add your Livingdocs API to the endpoint URL. Depending on your use case, the endpoint will change. ![](/images/partner_19878525089556.png)  
  


We will make use of Livingdocs' [Composition API](https://docs.livingdocs.io/reference/public-api/composition-api/) to fetch the HTML content of your templates.

  3. While setting up your content API, insert the following URL:
         
         https://server.livingdocs.io/api/beta/composition/&lt;document_id&gt;

  4. To find your document ID, open your document inside LivingDocs and look at the URL:  
  
![](/images/partner_19878750341652.png)  
Under Headers, add the following:

     1. `Authorization`: `Bearer &lt;your-access-token&gt;`
     2. `Content-Type`: `application/json; charset=utf-8`
     3. `Accept`: `application/json`
  5. Select the method as POST.
  6. Under Body, type the following:  



```json
&#123;
    "fields": [
        "systemdata",
        "content",
        "metadata",
        "includes",
        "html"
    ],
    "resolveIncludes": true
&#125;
```

  7. Click on "Save" and then "Done" once you see a successful response.
  8. Note - You can also test your APIs. To test your APIs, from the Content API's action menu, click on "Test". You can your API response and validate it. 



### Step 2: Display your template inside MoEngage campaigns

To use this API in your MoEngage campaigns,

  1. On step 2 on your campaign creation, enter @. 
  2. In the pop up, you can select the Livingdocs Content API configured in Step 1.
  3. You can use [MoEngage Templating Language](/user-guide/campaigns-and-channels/getting-started/message-personalization/jinja-templating-language#content-api-0-3) to parse through the API response and use the content in your campaigns:  


JINJA
         
         &#123;% set response = ContentApi.LivingdocsIntegration2(&#123;(&#123;"params":&#123;&#125;,"static_params":&#123;&#125;,"dynamic_params":&#123;&#125;,"request_body":&#123;"fields":["systemdata","content","metadata","includes","html"],"resolveIncludes":"true"&#125;&#125;)&#125;) %&#125;
         
         &#123;% for block in response.content[0].containers.main %&#125;
         
         &#123;% if block.component == "free-html" %&#125;
         
         &#123;&#123; block.content['free-html'].html &#125;&#125;
         
         &#123;% endif %&#125;
         
         &#123;% endfor %&#125;

  
![](/images/partner_19879027390356.png)
  4. You can click on the Preview tab to view your campaign in real-time personalization:  
![](/images/partner_19879067572372.png)



## Using Dynamic Blocks inside MoEngage Campaigns

At times, you might want to use a dynamic piece of code that remains the same throughout MoEngage campaigns across templates. You can achieve this with MoEngage Content Blocks:

  1. On your MoEngage Dashboard side navigation menu, go to _Content_ >> _Content Blocks_.
  2. Click on "_Create content block_ ".
  3. Create your content block. It can either be of type Plain text or HTML:  
![](/images/partner_19879758142996.png)
  4. Once you are ready, copy the block label. You will need to type this inside Livingdocs template editor.



Next, go to the Livingdocs template editor, and paste the block code:

![](/images/partner_19879866340500.png)

That's it! Your content block will automatically be added to the place where you inserted this code:

![](/images/partner_19879891001876.png)
