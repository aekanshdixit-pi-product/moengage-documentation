## Introduction

This guide provides detailed instructions for creating a MoEngage Push Campaign custom activity in SFMC Journey Builder.

<Info>
  **Prerequisites**

  - Salesforce Marketing Cloud Access: Ensure you have access to SFMC Journey Builder.
</Info>

## Steps

### Configure MoEngage Push in Journey Builder

1. In your SFMC journey, drag and drop the "MoEngage Push" activity.

   ![Drag and drop MoEngage Push activity in SFMC Journey Builder](/images/partner_36576366510356.png)

2. Click on "MoEngage Push".
3. Fill in the relevant details for your campaign in the three step process. You can refer to the [MoEngage Push API document](/api/campaigns/push-api) for details on each of these fields.

**Important note**

| Context | Value |
| --- | --- |
| Workspace Name | Enter your workspace name. It is present in the header on your MoEngage dashboard. See the screenshot below. *Note that the Workspace name must be the same as the one entered on the "MoEngage User Authentication Page" CloudPage. To verify the Workspace name, go to the Data Extension titled "MoEngage Credentials DE" and open "MoEngage Credentials".* |
| Personalise campaign content | To personalize the message in the Message Field, you can use the Data Extension (DE) field name in mustache syntax, formatted as `` `{{Field Name}}` ``. Ensure that the field name exactly matches the target DE. If the field name is correct, it will populate with the corresponding value from the target DE; otherwise, it will display the placeholder `` `{{Field Name}}` `` as is. |

![Workspace name displayed in the MoEngage dashboard header](/images/partner_36576366510484.png)

Once the custom activity is set up, you can continue with your Journey in the Journey Builder.

### View campaign on MoEngage

You can navigate to the All Campaigns page and view all the created campaigns. These will be created as API triggered Push Campaigns on MoEngage.
