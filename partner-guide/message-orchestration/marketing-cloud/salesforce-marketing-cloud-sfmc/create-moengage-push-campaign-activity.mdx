## Introduction

This guide provides detailed instructions for creating a MoEngage Push Campaign custom activity in SFMC.

<Info>
  **Prerequisites**

  - Salesforce Marketing Cloud Access: Ensure you have access to SFMC.
</Info>

## Steps

### Configure MoEngage Push

1. In your SFMC journey, drag and drop the "MoEngage Push" activity.

   ![Drag and drop MoEngage Push activity in SFMC](/images/partner_36576366510356.png)

2. Click on "MoEngage Push".
3. Fill in the relevant details for your campaign in the three step process. You can refer to the [MoEngage Push API document](/api/campaigns/push-api) for details on each of these fields.

**Important note**

| Context | Value |
| --- | --- |
| Workspace Name | Enter your workspace name. It is present in the header on your MoEngage dashboard.<br /><br /><img src="/images/partner_36576366510484.png" alt="Workspace name displayed in the MoEngage dashboard header" /><br /><br /><em>Note that the Workspace name must be the same as the one entered on the "MoEngage User Authentication Page" CloudPage. To verify the Workspace name, go to the Data Extension titled "MoEngage Credentials DE" and open "MoEngage Credentials".</em> |
| Personalise campaign content | To personalize the message in the Message Field, you can use the Data Extension (DE) field name in mustache syntax, formatted as <code>&#123;&#123;Field Name&#125;&#125;</code>.<br /><br />Ensure that the field name exactly matches the target DE. If the field name is correct, it will populate with the corresponding value from the target DE; otherwise, it will display the placeholder <code>&#123;&#123;Field Name&#125;&#125;</code> as is. |

Once the custom activity is set up, you can continue with your Journey.

### View campaign on MoEngage

You can navigate to the All Campaigns page and view all the created campaigns. These will be created as API triggered Push Campaigns on MoEngage.
