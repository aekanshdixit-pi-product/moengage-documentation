# Introduction

This guide provides detailed instructions for creating a MoEngage Push Campaign custom activity in SFMC Journey Builder.

<Info>
**Prerequisites**

* Salesforce Marketing Cloud Access: Ensure you have access to SFMC journey builder.
</Info>

# 

# Steps

## Configure MoEngage Push in journey builder 

  1. In your SFMC journey, drag and drop "MoEngage Push" activity.   
![Screenshot 2025-04-14 at 2.48.03 PM.png](/images/partner_36576366510356.png)
  2. Click on "MoEngage Push"
  3. Fill in the relevant details for your campaign in the three step process. You can refer to the [MoEngage document](https://developers.moengage.com/hc/en-us/articles/4404548252820-Push-API#01GKF5C20A5XZXR93HFZ7EMGFP) for details on each of these fields. 



**Important Note-**

| Context | Value

| --- | --- |  
| Workspace Name  |  Enter your workspace name. Its present in the header on your MoEngage dashboard.   
![Screenshot 2025-04-14 at 2.33.43 PM.png](/images/partner_36576366510484.png) _Note that the Workspace name must be the same as the one entered on the "MoEngage User Authentication Page" CloudPage. To verify the Workspace name, go to the Data Extension titled "MoEngage Credentials DE" and open "MoEngage Credentials"._  
| Personalise campaign content  | To personalize the message in the Message Field, you can use the Data Extension (DE) field name in mustache syntax, formatted as _&#123;&#123;Field Name&#125;&#125;_.  
  
Ensure that the field name exactly matches the target DE. If the field name is correct, it will populate with the corresponding value from the target DE; otherwise, it will display the placeholder &#123;&#123;Field Name&#125;&#125; as is.   
  
Once the custom activity is setup, you can continue with your Journey in the Journey Builder. 

## View campaign on MoEngage 

You can navigate to All Campaigns Page and view all the created campaigns. These will be created as API triggered Push Campaigns on MoEngage.
