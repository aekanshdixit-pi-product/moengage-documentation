#  Introduction

[Google Cloud Storage](https://cloud.google.com/storage) is an enterprise public cloud storage platform that can house large unstructured data sets.

# MoEngage × GCS

The MoEngage and GCS integration exports Campaign Interaction Events to your Google Cloud Storage buckets. You can then import these files into BigQuery for further processing and analysis.

# Integration 

<Info>
* Ensure you have a Google Cloud Platform account with permission to create a new bucket and a service account.
  * Exports to GCS are part of the Streams add-on. Contact your dedicated MoEngage CSM (customer success manager) to enable it for your account.
</Info>
## Step 1 - Create a dedicated service account for MoEngage

On your Google Cloud Platform console, go to the IAM & Admin console.

  1. From the sidebar, open "Service Accounts".



![serviceaccountshome.png](/images/partner_12062501879700.png)

  2. Click on the "+ Create Service Account" button.
  3. Give this service account a name, and an optional description.
  4. Click on "Create and continue".



![newserviceaccount.png](/images/partner_12062703174548.png)

  5. Then add the role of "Basic >> Browser" to this service account.



![browserrole.png](/images/partner_12062802345876.png)

  6. Click "Continue". Optionally, add users' access to this service account.
  7. Click "Done".
  8. Open the service account from the list of service accounts.
  9. Go to the "Key" tab, and then click on "Add Key". Select "Create new key".



![newkey.png](/images/partner_12062893736724.png)

  10. Select the Key Type as JSON. Then click "Create".



![jsonkey.png](/images/partner_12062926508052.png)

  11. Your new key will be automatically downloaded in JSON format. Make sure to store this safely. You will need to upload this later on the MoEngage dashboard.



## Step 2: Grant this service account permissions to your bucket

  1. On your service account details page, copy the email of your Service Account.



![serviceaccountemail.png](/images/partner_12063141549076.png)

  2. Go to the Cloud Storage Console, and open the bucket in which you want MoEngage to export the events.
  3. Under the Permissions tab, click on "Grant Access".



![GrantBucketAccess.png](/images/partner_12063166880276.png)

  4. Paste and then select the email of the service account in the "New Principals" field.
  5. Assign the roles of "Cloud Storage >> Storage Admin".



![bucketpermission.png](/images/partner_12063216822036.png)

  6. Click "Save".



## Step 3: Enable APIs & Services in your GCP Account

  1. In your GCP Console, go to "APIs & Services", then click on "+ Enable APIs And Services".



![EnableAPIs.png](/images/partner_12063295895828.png)

  2. Search for "Cloud Storage" and open the API details page.
  3. Click on Enable to enable it.



![enablecloudstorageapi.png](/images/partner_12063394282388.png)

## Step 4: Setup the GCS Integration on the App Marketplace

  1. On your MoEngage Dashboard, go to the App Marketplace.
  2. Search for "Google Cloud Storage".
  3. Go to the "Integrate" tab, and click on "+ Add Integration".



![addintegration.png](/images/partner_12063471722644.png)

  4. Give your connection an identifiable name.
  5. Enter your bucket name.
  6. Optionally, select your bucket region. You can find this on your Bucket Details page.
  7. Upload the Service Account Credential JSON file that you downloaded earlier. You can verify your Client ID and Client Email upon upload.
  8. Select which events you want to export. By default, each event will be exported with standard attributes.



![integration1.png](/images/partner_12064408178068.png)

  9. Optionally, you can choose to test the connection. MoEngage will attempt to verify if the details you entered are correct. If there is an issue, an error will be shown that you can rectify.



## Note on required permissions

To perform end-to-end testing, MoEngage requires a few additional permissions to be configured. If your IT Policy does not allow for these permissions, you can skip them. However, it might not be possible for you to test the connection from our app marketplace. Please note that skipping these optional permissions will not impact the actual data export in any way.

Permission

|  Required |  Description <br/>- |-

| --| |

---  
`resourcemanagement.projects.get`

| No |  Allows the user to view metadata for a project in Google Cloud Resource Manager. MoEngage uses this permission to access your service account during testing. <br/>- `resourcemanagement.projects.getIampolicy` | 

| No | |

  Allows the user to view the IAM policy for a project in Google Cloud Resource Manager. MoEngage uses this permission to check whether you have assigned the right policy to your bucket.  
`storage.bucket.get`

|  No |  Allows the user to view metadata for a Google Cloud Storage bucket MoEngage uses this permission to check whether the bucket that you have provided during configuration actually exists or not. <br/>- `storage.object.create` | 

| Yes | |

  Allows the user to create objects (files) in a Google Cloud Storage bucket. MoEngage requires this permission to create files in your bucket.  
`storage.object.get`

|  Yes |  Allows the user to read the contents of objects (files) in a Google Cloud Storage bucket. MoEngage requires this permission to verify if the files are correctly created in your bucket. <br/>- `storage.object.update` | 

| Yes | |

  Allows the user to modify the contents of objects (files) in a Google Cloud Storage bucket. MoEngage requires this permission to update files in case of any discrepancies found.  
  
Feel free to create a custom role that incorporates the mandatory permissions required by MoEngage and other optional permissions to aid you in testing as cleared by your IT Policies.

# Verify if you are receiving the data correctly

MoEngage will upload files in the following structure:
    
    
    gs://&lt;your-bucket-name&gt;/event-exports/&lt;your_app_name&gt;/&lt;connection_id&gt;/export_day=YYYY-MM-DD/export_hour=HH/&lt;file-name&gt;.json.gz

Go to your bucket, and verify if the files are present:

# Export Frequency

We will dump the data at hourly intervals.

# Frequently Asked Questions

<AccordionGroup>
<Accordion title="How soon will I start getting the data?">
It typically takes up to 30 minutes for the automated data exports to be set up for your account and for you to start seeing data there.
</Accordion>

<Accordion title="Can I export additional events at a later time?">
Yes, if you need to export new events at a later time, you can always edit your configuration on the App Marketplace and get the new events added to your exports.
</Accordion>

<Accordion title="Can I export historical data using Automated Data Exports?">
As of now, you cannot export historical data using Automated Data Exports. Once configured, you will start seeing data for each event from the time you enabled your exports.
</Accordion>

<Accordion title="Is there any pricing involved for Automated Data Exports?">
If your billing plan does not cover this already and you need to get this enabled at a later time, you would need to work with your Customer Success Manager to get this included in your billing plan. Based on the amount of data that MoEngage will place in your Data warehouse/Partner/API on a monthly basis, this could lead to additional charges.
</Accordion>
</AccordionGroup>
