# Introduction

[Zapier](https://zapier.com/) is an online platform that helps you automate workflows by connecting your apps and services you use. Using the MoEngage <> Zapier integration, you can create custom workflow automations for more than 5,000+ apps on Zapier. 

The action "Add or Remove User From Cohort (Audience)" helps to update a cohort (Audience) in MoEngage. You can either choose to add a user, or remove a user from the cohort. If a user already exists in the cohort, the action does nothing. 

# Use Cases

With this action, you can update cohorts from multiple sources in near real-time using Zapier. Some ideas to get you started - 

  1. Add new GoTo Webinar registrants into a cohort in MoEngage.
  2. Remove users from a cohort in MoEngage when a new row is added in Google Sheets.
  3. Add a user to a cohort in MoEngage when a Google Form is submitted.
  4. Add new Facebook Lead Ad leads to a cohort inside MoEngage.
  5. Remove user from a cohort when a new Typeform entry is created.<Info>
**Prerequisites**
Ensure you have access to your Zapier account and have connected it to your MoEngage account. You can [follow the steps here to connect Zapier to MoEngage. ](https://partners.moengage.com/hc/en-us/articles/7542196666388)
</Info># Integration

The action makes use of MoEngage [Cohort/ Audience APIs](https://developers.moengage.com/hc/en-us/articles/4426761481108-Cohort-Audience-). For this tutorial, we will pick up Use Case #1: Add new GoTo Webinar registrants into a cohort in MoEngage. On your [Zapier Dashboard](https://zapier.com/app/dashboard), and click on "+ Create Zap" on the top-left corner. 

## Step 1: Select GoTo Webinar as the Trigger App

For the trigger, type "GoTo Webinar" and select it. This will be the trigger that will start your zap.

![Action2-Step1.png](https://partners.moengage.com/hc/article_attachments/7899336232340/Action2-Step1.png)

## Step 2: Setup the trigger app

  1. Select the event from the dropdown: "New Registrant". Click continue.
  2. Connect your GoTo Webinar Account, if not done already. Click "Allow" to connect with Zapier. Once selected, click continue to move to the next step.



![Action2-Step2.png](https://partners.moengage.com/hc/article_attachments/7900142277396/Action2-Step2.png)

  3. Under the "Set up trigger" step, select the Webinar from the dropdown. Click continue.
  4. Click on "Test Trigger" to pull in a recent registrant as sample data. If no data is present, you can create a new registrant from your GoTo Webinar Dashboard.
  5. Once you are satisfied with this data, click continue. 



![Action-TestTrigger-1.png](https://partners.moengage.com/hc/article_attachments/7900176305812/Action-TestTrigger-1.png)

![Action-TestTrigger-2.png](https://partners.moengage.com/hc/article_attachments/7900165890964/Action-TestTrigger-2.png)

## Step 3: Add MoEngage as the Action app

Click on "+" to add a new step. Then, search for "MoEngage" from the apps and select it.

![Step8.png](https://partners.moengage.com/hc/article_attachments/7568867646612/Step8.png)

## Step 4: Setup MoEngage Action

  1. From the event dropdown, select: "Add or Remove User From Cohort (Audience)". Click continue.
  2. If you have followed the prerequisites, you should see your MoEngage account already listed. You can either choose that or connect a new one. Click continue once done.
  3. In the next step "Set up action", you will see a form for you to either fill-in or map from the previous step. [Click here to know more](https://developers.moengage.com/hc/en-us/articles/4426761481108-Cohort-Audience-).



![Action2-Step3-1.png](https://partners.moengage.com/hc/article_attachments/7899503804948/Action2-Step3-1.png)

![Action2-Step3-2.png](https://partners.moengage.com/hc/article_attachments/7899479160340/Action2-Step3-2.png)

Select if you want to add or remove user. Fill in all the mandatory fields and you will then be presented an option to test this setup. Do make sure you fill in at least one of "User ID" or "Anonymous ID". To know more, read the [Cohort API docs](https://developers.moengage.com/hc/en-us/articles/4426761481108-Cohort-Audience-#request-0-7).

You can also choose to add custom attributes in the form of key-value pairs and map them to inputs from the previous trigger app:

![Action3.png](https://partners.moengage.com/hc/article_attachments/7899927753748/Action3.png)

Once you have selected your mapping data from the previous step, it should look like this:

![Action4.png](https://partners.moengage.com/hc/article_attachments/7899920176020/Action4.png)

  4. For our setup, we have linked the following: 
     1. `Cohort Name` -> `"goto_sample_webinar"`
     2. `Cohort ID` -> `"goto_sample_webinar"`
     3. `User ID` -> `GoTo Webinar (Email)`
     4. `Anonymous ID` -> `GoTo Webinar (Email)`
     5. Once you are satisfied with your mappings, click on "Continue".
  5. You will now be presented with an option to test this setup.



![Action5.png](https://partners.moengage.com/hc/article_attachments/7899999901716/Action5.png)

  6. Click on "Test & continue" to test this setup.
  7. If the API call was successful, you should see a "success" return.



![Action6.png](https://partners.moengage.com/hc/article_attachments/7900012847764/Action6.png)

You can also verify this in your MoEngage dashboard. Go to your MoEngage Dashboard >> Segment >> Create segment. Filter users by "Custom Segment" and you should see your newly created cohort here. This confirms that your Zap was successful.

![Action7.png](https://partners.moengage.com/hc/article_attachments/7900058612756/Action7.png)

  8. Once you are satisfied that everything works, click on "Publish Zap" and it will run every time a new registrant is added to your Webinar.
