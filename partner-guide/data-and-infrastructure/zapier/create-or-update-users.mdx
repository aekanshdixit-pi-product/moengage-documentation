# Introduction

[Zapier](https://zapier.com/) is an online platform that helps you automate workflows by connecting your apps and services you use. Using the MoEngage <> Zapier integration, you can create custom workflow automations for more than 5,000+ apps on Zapier. 

The action "Create or Update User" helps to add a user into MoEngage. If the user already exists, the action updates the user’s attributes.

# Use Cases

With this action, you can create or update users from multiple sources in near real-time using Zapier. Some ideas to get you started - 

  1. Update user attribute in MoEngage when a Google Form is submitted.
  2. Create or update users in MoEngage with new Facebook Lead Ad leads.
  3. Create or update users in MoEngage from HubSpot.
  4. Add new users in MoEngage from Typeform entries.
  5. Create or update users in MoEngage from rows in Google Sheets.<Info>
**Prerequisites**
Ensure you have access to your Zapier account and have connected it to your MoEngage account. You can [follow the steps here to connect Zapier to MoEngage. ](https://partners.moengage.com/hc/en-us/articles/7542196666388)
</Info># Integration

The action makes use of MoEngage [User APIs](https://developers.moengage.com/hc/en-us/articles/4413167462804-User-). For this tutorial, we will pick up Use Case #1: Update a user attribute in MoEngage as soon as a Google Form is submitted. On your [Zapier Dashboard](https://zapier.com/app/dashboard), and click on "+ Create Zap" on the top-left corner. 

## Step 1: Select Google Forms as the Trigger App

For the trigger, type "Google Forms" and select it. This will be the trigger that will start your zap.

![Action1-Step1.png](https://partners.moengage.com/hc/article_attachments/7721767045524/Action1-Step1.png)

## Step 2: Setup the trigger app

  1. Select the event from the dropdown: "New Form Response". Click continue.
  2. Connect your Google Account, if not done already. Once selected, click continue.
  3. Under the "Set up trigger" step, select the form from the dropdown. Click continue.
  4. Click on "Test Trigger" to pull in a recent form submission as sample data. If no data is present, it will be auto-populated.
  5. Once you are satisfied with this data, click continue. 



![Action1-Step2.png](https://partners.moengage.com/hc/article_attachments/7721915281300/Action1-Step2.png)

![Action1-Step2-2.png](https://partners.moengage.com/hc/article_attachments/7721915254548/Action1-Step2-2.png)

## Step 3: Add MoEngage as the Action app

Search for "MoEngage" from the apps and select it.

![Step8.png](https://partners.moengage.com/hc/article_attachments/7568867646612/Step8.png)

## Step 4: Setup MoEngage Action

  1. From the event dropdown, select: "Create or Update User". Click continue.
  2. If you have followed the prerequisites, you should see your MoEngage account already listed. You can either choose that or connect a new one. Click continue once done.
  3. In the next step "Set up action", you will see a list of attributes and user properties for you to either fill-in or map from the previous step. [Click here to know more](https://developers.moengage.com/hc/en-us/articles/4413167462804-User-#request-body-0-1).



![Action1-Step3-1.png](https://partners.moengage.com/hc/article_attachments/7722082187540/Action1-Step3-1.png)

![Action1-Step3-2.png](https://partners.moengage.com/hc/article_attachments/7722082759956/Action1-Step3-2.png)

![Action1-Step3-3.png](https://partners.moengage.com/hc/article_attachments/7722116477332/Action1-Step3-3.png)

![Action1-Step3-4.png](https://partners.moengage.com/hc/article_attachments/7722116440084/Action1-Step3-4.png)

![Action1-Step3-5.png](https://partners.moengage.com/hc/article_attachments/7722087108116/Action1-Step3-5.png)

Fill in all the mandatory fields and you will then be presented an option to test this setup. You can also choose to add custom attributes in the form of key-value pairs and map them to inputs from the previous trigger app:

![Action1-Step4.png](https://partners.moengage.com/hc/article_attachments/7722184370324/Action1-Step4.png)

Once you have selected your mapping data from the previous step, it should look like this:

![mceclip0.png](https://partners.moengage.com/hc/article_attachments/7722189009172/mceclip0.png)

  4. For our setup, we have linked the following: 
     1. `Customer ID` -> `Google Forms (Email Address)`
     2. `Other Attributes` -> `form_submitted_on` -> `Google Forms (Last Submitted TIme)`
     3. `Other Attributes` -> `another_attribute` -> `"I can also write my own text"`
     4. Once you are satisfied with your mappings, click on "Continue".
  5. You will now be presented with an option to test this setup.



![Action1-Step5-1.png](https://partners.moengage.com/hc/article_attachments/7722308968724/Action1-Step5-1.png)![Action1-Step5-2.png](https://partners.moengage.com/hc/article_attachments/7722346499092/Action1-Step5-2.png)

  6. Click on "Test & continue" to test this setup.
  7. If the API call was successful, you should see a "success" return.



![mceclip1.png](https://partners.moengage.com/hc/article_attachments/7722381695764/mceclip1.png)

You can also verify this in your MoEngage dashboard. Go to your test user's profile and look under custom attributes:

![mceclip2.png](https://partners.moengage.com/hc/article_attachments/7722405725332/mceclip2.png)

  8. Once you are satisfied that everything works, click on "Publish Zap" and it will run every time a new response is added to your Google Form.
