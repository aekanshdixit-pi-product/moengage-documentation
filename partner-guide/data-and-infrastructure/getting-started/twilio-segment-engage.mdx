# Introduction

| info |  **Information** The Twilio Segment integration does not currently support [Unified Identity (Identity Resolution)](https://help.moengage.com/hc/en-us/articles/24050999467284-Unified-Identity-Identity-Resolution) or [Portfolio](https://help.moengage.com/hc/en-us/articles/40394603054100-Portfolio).

| --- | --- |  
  
[Twilio Segment ](https://segment.com/) is a customer data platform that helps you collect, clean, and activate your customer data.

This article will give an overview of the connection between MoEngage and Twilio Segment Engage (earlier Segment Personas) and list a step-by-step approach to enabling the integration. 

# Scope

The MoEngage and Twilio Segment integration allows you to use [Engage](https://segment.com/docs/personas/), Twilio Segment built-in audience builder, to create segments of users based on data you have already collected across various sources. These users will then be assigned user attributes or custom events that can be used to create MoEngage segments for use in MoEngage campaigns and flows.

# Integration

| library_add_check |  **Prerequisites**

  * Login credentials for MoEngage and Segment.
  * Segment Engage must be enabled for your segment account. You can validate this by navigating to Engage from the sidebar on your Segment dashboard.

| --- | --- |  
  
[Computed traits](https://segment.com/docs/personas/computed-traits/) and [audiences](https://segment.com/docs/personas/audiences/) can be sent to MoEngage as custom attributes or custom events.

  * Traits and audiences sent using the _identify_ call will appear in MoEngage as _Tracked Custom Attributes_ with a value of _True._
  * Traits and audiences sent using the _track_ call will appear in MoEngage as _Tracked User Events_.  
  
When you connect the computed trait/audience to the MoEngage destination, you can choose your preferred method (or choose to use both).



## Sync time

The default integration for the MoEngage × Segment Engage connection is **Real Time.** But there are some filters that will disqualify the persona from syncing in real-time, including some time-based filters that restrict your audience’s size at the time of message send.

## Step 1: Create a Segment computed trait/audience 

### Computed Trait 

  1. In Segment, navigate to the  _Computed Traits_ in _Engage._
  2. Click _New Computed Trait._
  3. Create your computed trait or audience. A lightning bolt in the top corner of the page will indicate if the computation updates in real-time.  


![](/images/partner_44425531631764.png)

  4. Next, select  _MoEngage_ as your destination.  


![](/images/partner_44425508047380.png)

  5. Preview by clicking _Review & Create_.  
_Note-_ By default, Segment queries all historical data to set the current value of the computed trait and audience. To omit this data, uncheck _Historical Backfill_.   


![](/images/partner_44425531632532.png)

  6. In the computed trait, adjust the connection settings based on how you would like your data sent to MoEngage.



### Audience 

  1. In Segment, navigate to the  _Audience_ in _Engage._
  2. Click __New  
__

![Screenshot_2022-04-27_at_12.22.46_PM.png](/images/partner_5873711417108.png)

  3. Create your audience. A lightning bolt in the top corner of the page will indicate if the computation updates in real-time.  


![Screenshot_2022-04-27_at_12.25.56_PM.png](/images/partner_5873719003924.png)

  4. Next, select  _MoEngage_ as your destination.  


![](/images/partner_44425508048148.png)

  5. Preview your audience by clicking  _Review & Create_.  
_Note-_ By default, Segment queries all historical data to set the current value of the audience. To omit this data, uncheck _Historical Backfill_.   


![Screenshot_2022-04-27_at_12.27.04_PM.png](/images/partner_5873702066324.png)

  6. In the computed trait or audience settings, adjust the connection settings based on how you would like your data sent to MoEngage.  


![Screenshot_2022-04-27_at_12.26.38_PM.png](/images/partner_5873748655892.png)




## Step 2: Segment users in MoEngage

To create a segment of these users, navigate to _Segments >> Create Segment._ Next, based on which call you used:

  * **Identify** : Select  _User Property_ and select the specific attribute.  


![Screenshot_2022-04-27_at_12.55.14_PM.png](/images/partner_5873977244052.png)

  * **Track** : Select  _User Behaviour_ and select the specific event.  


![Screenshot_2022-04-27_at_12.45.55_PM.png](/images/partner_5873934199572.png)
