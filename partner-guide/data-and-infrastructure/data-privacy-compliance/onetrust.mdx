# Introduction

[OneTrust](https://www.onetrust.com/) is a privacy and security software provider providing the visibility you need to better understand your trust landscape, action to leverage powerful insights, and automation to keep you elevated from the competition.

# MoEngage × OneTrust

The MoEngage and OneTrust integration allows you to use the OneTrust workflow builder to create security workflows for your product.

# Integration

<Info>
**Prerequisites**

* Ensure you have access to your OneTrust account.
</Info>

The integration allows you to update user consent in MoEngage from the OneTrust workflows. 

## Step 1: Configure MoEngage 

On your OneTrust dashboard, 

  1. Navigate to Integrations > Credentials in OneTrust.
  2. Add new credentials to allow OneTrust to authenticate with the MoEngage. 



## Step 2: Create an integration workflow

  1. Navigate to Integrations > Connections.
  2. Click Add New to create a new integration workflow.
  3. Enter the details- 
     * Name: _MoEngage_
     * Description:_Update user consent_ and action
  4. Click add a _Custom System_



## Step 3: Add workflow trigger

On the Workflow Builder screen,

  1. Select the Add OneTrust Trigger block.
  2. Choose the appropriate trigger based on when you want the workflow to initiate. This could be a module-specific event (e.g., assessment submission, data subject request stage change), or a webhook if an external system will initiate the workflow.   
![onetrust2.png](/images/partner_42483215032084.png)
  3. Click Done. 



## Step 4: Add action

  1. Click Add Action.
  2. Select MoEngage (that has been configured in step 2)
  3. Enter the following details 
     * Endpoint:  _&lt; Your MoEngage connected source endpoint&gt;. _You can reach out to your dedicated customer success manager for the details
     * Method: POST
     * Headers: 
       * Content-Type: application/json
       * Authorization: Generate a Basic Authentication Token using an online Basic Auth Header Generator tool. You must enter your Workspace ID as username and Data API Key as password. To find your credentials, perform the following steps:   
On the MoEngage UI, navigate to Settings > Account > APIs. Copy the ID under Workspace ID (earlier app id). Copy the Data under API keys. Your Basic Authentication Token is a combination of Workspace ID and Data.

  4. Click _Finish_ and then  _Activate_ to enable the workflow.
