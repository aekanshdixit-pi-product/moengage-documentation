The integration of Wisely and WhatsApp with MoEngage allows you to send your templated messages using a registered number powered by Wisely.

## Sender Configuration

To configure a WhatsApp number in the MoEngage Dashboard, perform the following steps:

  1. Go to **Settings** > **WhatsApp** > **Sender configuration**.
  2. Choose **Wisely** from the left menu and click **\+ Sender**.  
![7a.png](/images/partner_31645896642964.png)
3. Enter the following details:

| Field Name | Description |
| --- | --- |
| Sender name | This field denotes the name you want to provide to the sender profile. This would be available in the sender so that you can recognize it easily while using it to create a campaign inside MoEngage. The sender name should be between five and 50 characters. This is a mandatory field. |
| WhatsApp business number | This field denotes the phone number registered with WhatsApp (directly or through Wisely) using which you want to send out WhatsApp messages to your users. On Wisely, you can find sender details at **Channels** _ > _**WhatsApp** _ > _**Business Profiles** _._ This is a mandatory field. |
| API URL | The base endpoint URL of WhatsApp. This is a mandatory field. The API URL for Wisely is: [https://moeng.wiselyapi.com/moeng/whatsapp/message](https://moeng.wiselyapi.com/moeng/whatsapp/message) |
| apikey | This field indicates the API key generated from the Wisely account and is specific to the WhatsApp number. This is a mandatory field. Refer to the following points to view the API key in the Wisely account: 1. [Log in to the Wisely account](https://wisely.tanla.com/) with your valid credentials. 2. Navigate to the API icon from the left menu. The Access account page is displayed. 3. Click the key icon next right to your access account. Your API key is displayed on the screen. 4. Copy and paste in your API header. |



## Delivery Tracking

To track the delivery of your WhatsApp messages inside MoEngage, you must copy the MoEngage Delivery Tracking URL, and share it with Wisely, and get it updated on their side. You should be able to see the URL as shown below:  
![wisely.png](/images/partner_31132607916308.png)

In the **Delivery tracking URL** field, enter the following URL:

[https://api-0X.moengage.com/whatsapp/wisely/dlr/](https://api-0X.moengage.com/whatsapp/wisely/dlr/)

where X is the number of the data center (DC).

### Add MoEngage Webhook in Wisely

  1. [Log in to your Wisely account](https://wiseauth.tanla.com/realms/enterprise/protocol/openid-connect/auth?client_id=tan_ent_portal_ui&redirect_uri=https%3A%2F%2Fwisely.tanla.com%2Fdashboard&state=7529d2b9-e2b0-4f50-a522-1ead88b367c8&response_mode=fragment&response_type=code&scope=openid&nonce=c8454ec3-e7f8-45e6-8d82-86a4c94f81cf) with valid credentials.
  2. Click **API** from the left menu.
  3. Locate and select the **Access Account** where you want to place the webhook URL.
  4. Click the stencil icon (![stencil.png](/images/partner_31132652062484.png)) next to the access account.
  5. Select **Webhook for Outgoing Status Events** and place the delivery tracking URL of MoEngage in the **HTTPS Webhook URL** field.  


<Info>
Add the MoEngage webhook to an access account with WhatsApp configured as one of the channels.
</Info>
  6. Click **Update**.  
![Image 08-10-24 at 3.48 PM.jpeg](/images/partner_31068380500244.png)  
Upon successfully adding webhook, you will see the "Access Account updated successfully" message.



### Add Delivery Tracking for User-Initiated Messages

  1. [Log in to the Wisely platform](https://wiseauth.tanla.com/realms/enterprise/protocol/openid-connect/auth?client_id=tan_ent_portal_ui&redirect_uri=https%3A%2F%2Fwisely.tanla.com%2Fdashboard&state=7529d2b9-e2b0-4f50-a522-1ead88b367c8&response_mode=fragment&response_type=code&scope=openid&nonce=c8454ec3-e7f8-45e6-8d82-86a4c94f81cf) with valid credentials.
  2. Click API from the left menu.
  3. Locate and select the **Access Account** where you want to place the webhook URL.
  4. Click the stencil icon ( ![stencil.png](/images/partner_31132652062484.png)) next to the access account.
  5. Select the **Webhook for User Initiated Messages** and place the MoEngage webhook URL in the **HTTPS Webhook URL** field.  


<Info>
Webhooks for user-initiated messages can only be added to access accounts that are of the Conversational type.
</Info>
  6. Click **Update**.  
![Image 08-10-24 at 3.55 PM.jpeg](/images/partner_31068460945684.png)  
Upon successfully adding webhook, you will see the "Access Account updated successfully" message.
