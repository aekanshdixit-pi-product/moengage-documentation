# Introduction

[Times Mobile](https://timesmobile.in/) (Part of Times Internet Limited) offers enterprise communication services including SMS, Voice, Email, RCS & WhatsApp Business Messaging. With the MoEngage and Times Mobile SMS Integration, you can communicate with your end users for the following:

* Bulk SMS communication of different categories – Promotional, Service Implicit, and Service explicit.
* Real-Time Delivery updates

In addition, you can also automate sales and marketing engagements by sending personalized texts through a secured SMS Platform.

# Integration 

<Info>
* You must have an SMS account with respective SMS Push API details from TIL.
* You must have DLT-approved details i.e., templates if the region of your account is in India.
</Info>

## Configure Times Mobile as a Custom SMS Connector

Log in to the MoEngage dashboard:

1. Navigate to Settings > Channel > SMS & Connectors.   
   **![](/images/partner_22513369490836.png)**
2. Click the _SMS Connector Config tab_.
3. Under _Custom Connecters_ , click _+CREATE._ ![](/images/partner_22513369500052.png)
4. Add the following details:

| Field | Description |
| --- | --- |
| **Sender Type** | Select Sender Type - Promotional/Transactional based on SMS account at TIL. |
| **Connector Name** | Enter the Connector Name for the SMS Connector to uniquely identify it. Example: "Times Mobile" |
| **Sender Name** | Enter Sender Name (DLT Approved Header) |
| **API URL** | Select the **GET** or **POST** option depending on which HTTP Endpoint you have from TIL. Now Enter the TIL SMS Push Endpoint (GET/POST) with respective credentials/details:<br/><ul><li><b>GET API:</b><ul><li>Endpoint: <code>https://bsms.timesapi.in/ct/api/v1/send?username=&amp;password=&amp;unicode=false&amp;from=&amp;to=Moesms_destination&amp;text=Moesms_message&amp;dltContentId=Moesms_dltTemplateId&amp;corelationid=Moesms_campaignId</code></li><li>Key-value pair: Enter your keys and values.</li></ul></li><li><b>POST API:</b><ul><li>Endpoint: <code>https://bsms.timesapi.in/ct/api/v1/message</code></li><li>Enter the JSON payload structure supported by the POST Endpoint.</li></ul></li></ul> |
| **Parameter Details** | Key value pair / Json Parameter details:<br/><ul><li>username=<code>{'<******>'}</code> [TIL api username]</li><li>password=<code>{'<******>'}</code> [TIL api password]</li><li>unicode= [by default “false” for normal English SMS text and “true” for Unicode SMS text]</li><li>from= [DLT approved Header]</li><li>to=<code>Moesms_message</code> [Mobile no. on which SMS to be sent]</li><li>text=<code>Moesms_destination</code> [DLT approved message text]</li><li>dltContentId=<code>Moesms_dltTemplateId</code> [DLT Template ID for message text]</li><li>corelationid=<code>Moesms_campaignId</code> [Parameter for MoEngage Campaign Id]</li></ul> |
| **Headers** | For POST Endpoint, please enter below key-value pairs:<br/><ol><li>Key: <code>Content-Type</code> &#124; Value: <code>application/json</code></li><li>Key: <code>Accept</code> &#124; Value: <code>application/json</code></li><li>Key: <code>Authorization</code> &#124; Value: <code>Basic</code></li></ol> |
| **Body Type (Only for POST)** | For POST Endpoint, select “Raw” and Enter the entire JSON payload structure:<br/><br/><pre><code>{'{'}<br/>  "extra": {'{'}<br/>    "corelationid": "Moesms_campaignld",<br/>    "dltContentId": "Moesms_dltTemplateld"<br/>  {'}'},<br/>  "message": {'{'}<br/>    "recipient": "Moesms_destination",<br/>    "text": "Moesms_message"<br/>  {'}'},<br/>  "sender": "",<br/>  "unicode": "false"<br/>{'}'}</code></pre> |
| **Delivery tracking** | For Delivery tracking, after saving the “Custom Connector” edit it. Now select the option “Map fields manually” and do the below mapping:<br/><ul><li>Unique sent ID field: <code>uuid</code></li><li>Unique delivery ID field {'*'}: <code>uuid</code></li><li>Delivery status field {'*'}: <code>status</code></li><li>Success values for delivery status {'*'}: <code>DELIVERY_SUCCESS</code></li><li>Failure reason field {'*'}: <code>description</code></li></ul><br/>Please share SMS delivery tracking URL with Times for configuration. |

5. Click Save. The settings have been added.

## Troubleshooting

If any issue occurs while sending test message or saving configuration follow the below steps:

* Ensure that the TIL Endpoint and respective parameters/json structure are added correctly as they are case-sensitive.
* If the issue persists, raise a Support Request to MoEngage to get your error code.
* Connect with the TIL support team to get the issue resolved after you receive the error code from the MoEngage Support team. Support Email: [support@timesmobile.in](mailto:support@timesmobile.in)
