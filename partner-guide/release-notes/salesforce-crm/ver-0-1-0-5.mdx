This release introduces enhancements and improvements focused on historical sync with adjustable batch sizes, Apex Flex Queue verification, and updated permission sets with new user and admin permissions. 

# What's New?

The following are the new features and improvements introduced in this version:

### Enhanced batch size options for historical sync

  * Introduced additional batch size options for historical sync functionality.
  * New batch sizes are now available with options: 5, 10, 15, 20, 25, 50, 75, 100, 125, 150, 175, and 200.



### Apex Flex Queue logic check for historical sync

Enhanced historical sync execution logic to check Apex Flex Queue availability. The Apex Flex Queue is deemed complete if there are more than 50 pending jobs.

  * Improved historical sync execution logic to validate Apex Flex Queue availability.
  * The Apex Flex Queue is considered complete if there are more than 50 pending jobs.



**If the queue is full** :

  * The **Sync** button will be disabled.
  * This message will be displayed: Salesforce Apex Flex Queue is complete. Please try again later.



**If the user refreshes the page** :

  * The **Sync** button will be re-enabled.
  * Clicking the button will trigger the Apex Flex Queue full check again.



### Permission set enhancements

  * Renamed the existing**MoEngage Sync Configuration Permission** to **MoEngage Admin Permission**.
  * Introduced a new permission set: **MoEngage User Permission** , which provides access to: 
    * Package custom objects and fields
    * Custom settings and metadata
    * Apex class access
