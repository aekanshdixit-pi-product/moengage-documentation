This release includes refinements to labeling and configuration behaviors to enhance clarity and reduce manual setup effort during deployment.

# What's new?

## (UI Update) Field Label Change: Exception Object

  * The field previously labelled “**Detailed Error Message** ” on the Exception object (API Name: `moengage__Full_Error_Message__c`) is now relabelled as “**MoEngage Error Message** ”.
  * This update aims to provide a clearer context specific to the integration with MoEngage.



## Platform Behavior Update

### Lightning Record Page: Sync Queue

  * The “**Sync Queue Record Page** ” will now be **auto-activated** org-wide upon both installation and upgrades of the package.
  * This ensures a consistent user experience and removes the need for manual page activation.
