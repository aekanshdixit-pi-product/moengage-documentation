## What's New?

### Trigger Handling Process Improvements

This version introduces smarter management for organizations dealing with expired or suspended package licenses. The primary goal is to ensure that MoEngage logic does not block standard Salesforce operations on your mapped objects.

- **License Status Verification:** The system now proactively checks the status of the installed package license (Active, Expired, or Suspended).
- **Automated Logic Bypass:** If the system identifies an inactive license, it automatically skips the packaged code execution. This ensures that standard Salesforce triggers, workflows, and validation rules continue to function without interference from the package.

## Issue Fixes

### Graceful Handling of Expired and Suspended Licenses

- **Problem:** Previously, if a MoEngage package license expired or was suspended, the package triggers could inadvertently hinder normal operations on mapped Salesforce objects, potentially causing errors or blocking record updates.
- **Solution:** We have implemented a license-aware trigger logic. The package now detects its own license state; if the license is not in good standing, the trigger logic bypasses the MoEngage-specific code path, allowing your organization to function normally.
