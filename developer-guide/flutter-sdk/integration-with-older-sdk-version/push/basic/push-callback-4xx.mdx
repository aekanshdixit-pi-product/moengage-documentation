# Configuring Callbacks in Flutter

MoEngage's Flutter plugin optionally provides a callback on push click. To register for the callback call theô `setUpPushCallbacks()`ô on theô `MoEngageFlutter`ô object in your dart code.  
This API takes a method as input with whoseô `typedef`ô isô `PushCallbackHandler(PushCampaign pushCampaign)`

<CodeGroup>
``` Dart
import 'package:moengage_flutter/moengage_flutter.dart';
  void _onPushClick(PushCampaign message) {
    print("This is a push click callback from native to flutter. Payload " +
        message.toString());
  }
final MoEngageFlutter _moengagePlugin = MoEngageFlutter();
_moengagePlugin.setUpPushCallbacks(_onPushClick);
```

</CodeGroup>

Make sure this callback is set as soon as the application is initialized. Preferably in theô `initState()` of your application widget.

Make sure the callback is set before calling the `initialise()` of the MoEngage Plugin.

## Payload

NotificationPayload received in the callback is aô `PushCampaign`ô instance with the following definition:

<CodeGroup>
``` Dart
class PushCampaign {
  String platform;
  bool isDefaultAction;
  Map<String, dynamic> clickedAction;
  Map<String, dynamic> payload;
}
```

</CodeGroup>

Payload Structure forô `clickedAction`ô Map

<CodeGroup>
```json JSON
{
  "clickedAction": {
    "type": "navigation/customAction",
    "payload": {
      "type": "screenName/deepLink/richLanding",
      "value": "",
      "kvPair": {
        "key1": "value1",
        "key2": "value2",
        ...
      }
    }
  }
}
```

</CodeGroup>

`platform`ô - Native platform from which callback is triggered. Possible values -ô `android`,ô `ios`  
`isDefaultAction`ô - This key is present only for the Android Platform. It's a boolean value indicating if the user clicked on the default content or not. true if the user clicks on the default content else false.  
`clickedAction`ô - Action to be performed on notification click.  
`clickedAction.type`ô - Type of click action. Possible valuesô `navigation`ô andô `customAction`. Currently,ô `customAction`ô is supported only on Android.  
`clickAction.payload`ô - Action payload for the clicked action.  
`clickedAction.payload.type`ô - Type of navigation action defined. Possible valuesô `screenName`,ô `deepLink`,ô `richLanding`. Currently, in the case of iOS, rich landing and deep-link URLs are processed internally by the SDK and not passed in this callback therefore possible value in the case of iOS is only `screenName`.  
`clickAction.value`ô - value entered for navigation action or custom payload.  
`clickAction.kvPair`ô - Custom key-value pair entered on the MoEngage Platform.  
`payload`ô - Complete campaign payload.

## Android Payload

If the user clicks on the default content of the notification the key-value pair and campaign payload can be found inside theô `payload`ô key. If the user clicks on the action button or a push template action the action payload would be found insideô `clickedAction`.  
You can use theô `isDefaultAction`ô key to check whether the user clicked on the default content or not and then parse the payload accordingly.

## iOS Payload

In the case of iOS, you would always receive the key-value pairs with respect to clicked action inô `clickedAction`ô property. Refer to thisô [link](https://developers.moengage.com/hc/en-us/articles/4564174542228)ô for knowing the iOS notification payload structure.
