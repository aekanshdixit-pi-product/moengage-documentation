By default, we use HTTPS protocol for all requests made from the SDK, HTTPS encrypts the requests by default. MoEngage SDK optionally adds another layer of encryption apart from the encryption done by HTTPS.  
To enable this additional encryption you need to

* Call the [*enableEncryption()*](https://moengage.github.io/android-api-reference/core/com.moengage.core/-mo-engage/-builder/enable-encryption.html) on the [*MoEngage.Builder*](https://moengage.github.io/android-api-reference/core/com.moengage.core/-mo-engage/-builder/index.html) object while initializing the SDK
* Add the security module. as a dependency in your *app/build.gradle* file.

# SDK Installation

## Installing Using BOM

Integration using BOM is the recommended way of integration, refer to the [Install Using BOM](https://developers.moengage.com/hc/en-us/articles/44622788607892) document to configure BOM if not done already. Once you have configured the BOM, add the dependency in the ***app/build.gradle*** file as shown below

<CodeGroup>
```auto build.gradle
dependencies {
    ...
    implementation("com.moengage.security")
}
```

</CodeGroup>
Alternatively, you can add the dependency directly as shown below.

## Installing using Artifact Id

![](https://img.shields.io/maven-central/v/com.moengage/security)

Add the following dependency in the ***app/build.gradle*** file.

<CodeGroup>
```auto build.gradle
dependencies {
    ...
    implementation("com.moengage:security:$sdkVersion")
}
```

</CodeGroup>
replace **$sdkVersion** with the appropriate SDK version

<Info>
Adding the above dependency and enabling flag isn't enough for this feature to work; there is some additional configuration required on our side to completely enable this feature. In case you want to use this feature, contact your account manager or the MoEngage Support team.
</Info>
# Version Compatability

The ***security*** artifact/module is dependent on the **Core SDK** or ***moe-android-sdk*** artifact/module.  
Based on the version of the Core SDK you are using, choose the right version of the Security module.

| Core SDK Version | Security |
| --- | --- |
| 12.0.00 and above | 2.0.0 and above |
