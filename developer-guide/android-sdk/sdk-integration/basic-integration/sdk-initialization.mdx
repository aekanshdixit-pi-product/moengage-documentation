# SDK Initialization

Get the Workspace ID from the Settings page on the MoEngage dashboard at **Settings** --\> **App** --\> **General**, then initialize the MoEngage SDK in your Application class `onCreate()`.

<Info>
  **Note** Initialize the SDK on the main thread inside `onCreate()`. Do not create a worker thread and initialize the SDK on that thread.
</Info>

Replace `Your_Workspace_ID` with the Workspace ID from your dashboard and replace `DATA_CENTER_1` with the [data center](/developer-guide/android-sdk/sdk-integration/basic-integration/data-center) that matches your dashboard host.

<CodeGroup>

```kotlin Kotlin wrap
import com.moengage.core.DataCenter
import com.moengage.core.MoEngage

val moEngage = MoEngage.Builder(this, "Your_Workspace_ID", DataCenter.DATA_CENTER_1)
                     .build()
MoEngage.initialiseDefaultInstance(moEngage)
```

```java Java
import com.moengage.core.DataCenter;
import com.moengage.core.MoEngage;

MoEngage moEngage = new MoEngage.Builder(this, "Your_Workspace_ID", DataCenter.DATA_CENTER_1)
                     .build();
MoEngage.initialiseDefaultInstance(moEngage);
```

</CodeGroup>

## Portfolio (Optional)

If your MoEngage account is configured with multiple projects in a [portfolio](https://www.moengage.com/docs/user-guide/settings/account/portfolio/portfolio), use the methods shown below.

<CodeGroup>

```kotlin Kotlin wrap
import com.moengage.core.DataCenter
import com.moengage.core.MoEngage
import com.moengage.core.config.ProjectConfig

val moEngage = MoEngage.Builder(this, "Your_Workspace_ID", DataCenter.DATA_CENTER_1)
              .configureProject(ProjectConfig("<YOUR ProjectID>"))
              .build()
MoEngage.initialiseDefaultInstance(moEngage)
```

```java Java wrap
import com.moengage.core.DataCenter;
import com.moengage.core.MoEngage;
import com.moengage.core.config.ProjectConfig;

MoEngage moEngage = new MoEngage.Builder(this, "Your_Workspace_ID", DataCenter.DATA_CENTER_1)
                     .configureProject(new ProjectConfig("<YOUR ProjectID>"))
                     .build();
MoEngage.initialiseDefaultInstance(moEngage);
```

</CodeGroup>

For more information about the detailed list of possible configurations, refer to the [API reference](https://moengage.github.io/android-api-reference/core/com.moengage.core/-mo-engage/-builder/index.html).

<Warning>
  **Critical** All configurations must be added to the builder before initialization. If you call initialize at multiple places, ensure that all required flags and configurations are set each time you initialize to maintain consistency in behavior.
</Warning>

# Data Flow

The SDK detects the build type of the installed application and uses it to decide whether data should be sent to the test or live environment of the MoEngage platform.

- Unsigned/Debug build — Data flows to the test environment.
- Signed/Live build — Data flows to the live environment.
