# Configuring Build Settings

## Prerequisites

Before configuring the Android build settings, complete the following:

- Install the MoEngage React Native SDK in your project.
- Open the `android/` directory of your React Native project in Android Studio, or prepare to edit Gradle files under `android/app/` directly.
- Confirm whether your project uses the Groovy DSL (`build.gradle`) or the Kotlin DSL (`build.gradle.kts`) for Gradle configuration.

## Add AndroidX Libraries

<Warning>
  The MoEngage SDK depends on the `lifecycle-process` library for a few key features to work, and the latest version of `lifecycle-process` depends on the `androidx.startup:startup-runtime` library. Retain the `InitializationProvider` component in the manifest. When you add other Initializers using `startup-runtime`, also add the Initializer for the `lifecycle-process` library. Refer to the [AndroidX Lifecycle documentation](https://developer.android.com/jetpack/androidx/releases/lifecycle#2.4.0) to learn how to add the Initializer.
</Warning>

The SDK also depends on a few AndroidX libraries for its functioning. You can add these libraries through one of two approaches: enable the required flag in the `package.json` file, or declare the dependencies directly in your application's `build.gradle` file. Select the approach that best fits your project, and apply a single approach to maintain a clean dependency declaration and consistent build behavior.

### Configure in the Package JSON file

<Info>
  - Setting `includeAndroidXRequiredLibraries` to `true` automatically adds the AndroidX libraries used by the SDK as dependencies.
  - **Required AndroidX Libraries:** The MoEngage SDK requires the following core AndroidX dependencies:
    - `androidx.core:core` 
    - `androidx.appcompat:appcompat` 
    - `androidx.lifecycle:lifecycle-process` 
  - Before setting `includeAndroidXRequiredLibraries` to `true`, verify whether your native Android project's `build.gradle` already declares compatible versions of the AndroidX libraries listed above (`androidx.core:core`, `androidx.appcompat:appcompat`, and `androidx.lifecycle:lifecycle-process`). If these dependencies are already declared, set `includeAndroidXRequiredLibraries` to `false` to prevent version conflicts and duplicate dependency errors at build time.
</Info>

File - **_package.json_**

<CodeGroup>

```json JSON
{
    "moengage": {
        "includeAndroidXRequiredLibraries": true
    }
}
```

</CodeGroup>

### Configure in the Gradle file

Skip this step if you set `includeAndroidXRequiredLibraries` to `true` in the previous step.

Edit the following file based on your Gradle DSL:

- Groovy DSL: `android/app/build.gradle`
- Kotlin DSL: `android/app/build.gradle.kts`

<CodeGroup>

```groovy Groovy

    dependencies {
    ...
      implementation("androidx.core:core:1.9.0")
      implementation("androidx.appcompat:appcompat:1.4.2")
      implementation("androidx.lifecycle:lifecycle-process:2.7.0")
    }
```

</CodeGroup>

## Feature Modules (optional)

To include optional modules from the MoEngage SDK based on your feature requirements, you can use the provided flags. By default, these modules are not included in your project.

You can configure this by adding a new "moengage" key to the root level of your project's package.json file.

Inside this object, you can pass specific feature flags:
- true: Instructs the build system to download and link the required native dependencies for that specific module.
- false (or omitting the key): Excludes the module from your project. By default, all optional modules are excluded to help keep your overall app size optimized.

File - **_package.json_**

<CodeGroup>

```json JSON
{
    "moengage": {
        "richNotification": true,
        //Required for Push Templates
       
        "encryption": true,
       //Required for Add-On-Security
      
        "pushAmp": true,
        //Required for Push Amplification
      
        "hmsPushkit": true,
        //Required for HMS PushKit
      
        "deviceTrigger": true
       //Required for Device Triggered
    }
}
```

</CodeGroup>
